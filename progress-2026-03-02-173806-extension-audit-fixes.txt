## Codebase Patterns
- `chrome.scripting.executeScript` `func` runs in the browser context (isolated from Node.js module scope). To pass module-level constants into an injected script, add them to the `args` array — plain objects are JSON-serializable and can be received as extra function parameters.
- `vi.hoisted()` is the required pattern for creating mock function references used in `vi.mock()` factories — avoids `@typescript-eslint/unbound-method` ESLint errors when accessing methods on mocked objects
- Test mock function types in `vi.hoisted`: use `vi.fn<(arg?: unknown) => ReturnType>(() => default)` to properly type parameter and return shapes, enabling `mock.calls[0]?.[0]` access without TS errors
- For object method mocks, always create standalone `vi.hoisted` references rather than accessing `vi.mocked(obj.method)` — the latter triggers unbound-method lint errors

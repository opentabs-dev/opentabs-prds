## Codebase Patterns
- chrome.storage interactions follow `.then(success, failure)` pattern (not `.catch()`), see `hooks/useTheme.ts`
- Use `chrome.storage.session` for UI state that should persist within a browser session but reset on Chrome quit
- Use `chrome.storage.local` for state that should persist across browser restarts (e.g., theme)
- React 19 `useRef` requires an explicit initial value argument (e.g., `useRef<T | undefined>(undefined)`)
- Do NOT debounce chrome.storage writes that need to survive page close — chrome.storage.session.set() is async and cannot complete during unmount/beforeunload. Write immediately instead.

## Codebase Patterns
- Shared types live in platform/shared/src/index.ts; Config*Params interfaces are grouped together around line 295
- ConfigSet*Params naming convention: singular (one tool) → subset (tools[]) → all (no tools param)
- Build order: tsc --build → browser-tools-catalog → extension bundle → side-panel → icons → install-extension
- Extension message flow: define interface in extension-messages.ts → add to InternalMessage union → add handler in background-message-handlers.ts → register in backgroundHandlers map + EXTENSION_ONLY_TYPES set → add bridge function in side-panel/bridge.ts
- addPendingPluginAllToolsUpdate/removePendingPluginAllToolsUpdate accept an array of tool names — reusable for both all-tools and subset-of-tools operations
- E2E selectors for tool names in the side panel must use `{ exact: true }` with getByText because ToolRow renders descriptions inline (visible)

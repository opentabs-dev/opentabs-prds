## Codebase Patterns
- `platform/shared/src/constants.ts` holds all path helpers (`getConfigDir`, `getConfigPath`, `getExtensionDir`, `getLogFilePath`). New path helpers go here, then exported from `platform/shared/src/index.ts`.
- `platform/cli/src/config.ts` imports path helpers from `@opentabs-dev/shared` and re-exports them for CLI commands to consume.
- The `handleStart` function in `platform/cli/src/commands/start.ts` handles all initialization (autoInitialize, ensureAuthSecret, port check, banner) before branching on mode (background vs foreground). Background mode uses `spawn` with `detached: true` + `child.unref()`.
- The `StartOptions` interface in start.ts holds all `--flag` options. The `registerStartCommand` function uses Commander's `.option()` to register flags.
- `child.pid` from `spawn` is `number | undefined` — must narrow before using in template literals (ESLint `@typescript-eslint/restrict-template-expressions`).
- New CLI commands follow the pattern: `commands/<name>.ts` exports `register<Name>Command`, which is re-exported from `commands/index.ts` and called in `cli.ts`.
- `process.kill(pid, 0)` checks process existence without sending a signal — throws if dead (used in stop command and can be used elsewhere for PID checks).

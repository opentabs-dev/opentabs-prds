## Codebase Patterns
- The root CLAUDE.md should remain plugin-agnostic — describe patterns and conventions, not inventories of specific plugins
- MCP workflow triggers table in CLAUDE.md mirrors the SERVER_INSTRUCTIONS Workflow Triggers table from `platform/mcp-server/src/mcp-setup.ts` (lines 353-378)
- MCP resource URIs follow the `opentabs://` scheme with `guide/` and `reference/` path prefixes
- Platform tools are raw JSON Schema objects in `PLATFORM_TOOLS` array, auto-indexed in `PLATFORM_TOOL_NAMES`. Tests use `.size` for dynamic count assertions
- `resolvePrompt` in `mcp-prompts.ts` returns `PromptResult` with `messages: PromptMessage[]` where each message has a single `content` (TextContent or EmbeddedResourceContent)
- The `buildConfigStatePayload` exclusion test explicitly lists platform tool names — must be updated when adding new platform tools
- Browser tool description changes require updating both the source definition AND `platform/shared/src/generated/browser-tools-catalog.ts` — the barrel validates at import time
- E2E tests `config-watcher.e2e.ts` and `hot-reload-dynamic.e2e.ts` hardcode `builtInToolSet` with platform tool names — update when adding new platform tools

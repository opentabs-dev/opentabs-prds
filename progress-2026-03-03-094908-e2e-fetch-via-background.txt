## Codebase Patterns
- E2E test servers follow the pattern in `e2e/test-server.ts`: listen on PORT=0, print "Listening on http://localhost:<port>" to stdout (parsed by `startServerProcess` in fixtures.ts)
- E2E fixtures use `startServerProcess(entryFile, label)` factory to spawn test server subprocesses
- The e2e-test plugin uses Zod v4 where `z.record()` requires two arguments: `z.record(keySchema, valueSchema)` — e.g., `z.record(z.string(), z.string())`
- `ToolHandlerContext.fetchViaBackground(url, init?)` is injected by the adapter runtime — tool handlers must check `context?.fetchViaBackground` before using it
- `FetchProxyResponse` type is exported from `@opentabs-dev/plugin-sdk` with fields: `status`, `statusText`, `headers`, `body`, `ok`

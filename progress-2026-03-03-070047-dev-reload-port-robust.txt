## Codebase Patterns
- Dev reload server is in `scripts/dev-reload-server.ts`, constants in `scripts/dev-reload-constants.ts`
- The dev orchestrator (`scripts/dev.ts`) is the only caller of `startDevReloadServer`
- `DevReloadServer` interface is the public API for the relay server
- Build scripts (`build-side-panel.ts`, `build-extension.ts`) are in `platform/browser-extension/`
- Port placeholder `__DEV_RELOAD_PORT__` is used in dev reload client JS files and replaced at build time
- Build scripts read the dev reload port from `process.env.OPENTABS_DEV_RELOAD_PORT` (set by `dev.ts` after binding)
- `dev.ts` starts the relay server BEFORE the extension build so the env var is available at build time

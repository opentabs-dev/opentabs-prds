## Codebase Patterns
- `writeAndWaitForWatcher(server, writeFile: (attempt: number) => void, expectedLog, outerTimeoutMs?, innerTimeoutMs?)` in `e2e/helpers.ts` — wraps a file write and polls server logs for the watcher to detect it; retries on FSEvents miss. Callback receives `attempt` (0-indexed) so callers can produce slightly different content on retry.
- `writeAndWaitForWatcher` internally calls `waitForLog(server, 'File watcher: Watching', 10_000)` at the start — do NOT clear `server.logs` immediately before calling it or that wait will timeout. Clear logs before any hot-reload trigger phase, not right before a write.
- For `writeToolsToManifest` retries, append `' '.repeat(attempt)` to a description field to guarantee FSEvents sees a content change on retry.
- Prefer `mcpClient.listTools()` over `fs.readFileSync(configPath)` for verifying config persistence in E2E tests — avoids JSON.parse failures from partial writes, and `listTools()` confirming the state is functionally equivalent since the MCP server only reflects config after writing to disk.

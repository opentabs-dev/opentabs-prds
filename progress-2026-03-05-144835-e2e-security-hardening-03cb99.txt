## Codebase Patterns
- E2E tests use `createTestConfigDir()` / `cleanupTestConfigDir()` / `startMcpServer()` from `e2e/fixtures.ts` for server-only tests
- Node.js `fetch` treats Host as a forbidden header and silently ignores custom Host values — use `node:http` request for Host header override tests
- The `/health` endpoint returns 200 with `{ status: 'ok' }` for unauthenticated requests (not 401)
- ALLOWED_HOSTS set: `localhost`, `127.0.0.1`, `::1`, `::ffff:127.0.0.1` — all with optional port suffix
- Server-only E2E tests use `beforeEach`/`afterEach` for config dir lifecycle instead of Playwright fixtures
- Plugin tool tests require full browser fixtures (mcpServer, testServer, extensionContext, mcpClient) + `setupToolTest` helper
- `slow_with_progress` tool is useful for concurrency/timing tests — `durationMs` controls how long dispatch slots stay occupied
- Biome rejects non-null assertions (`!`) — use optional chaining (`?.`) instead
- Tests needing 'ask' permission flow require a custom fixture with `OPENTABS_DANGEROUSLY_SKIP_PERMISSIONS: ''` — use `base.extend<>()` pattern from permissions.e2e.ts
- WebSocket steal pattern disconnects the extension without destroying BrowserContext — connect fake WS, wait for "Closing previous", close fake, wait for disconnected
- Confirmation dialog uses `[role="dialog"]` locator — use it as synchronization signal since no server log is emitted for confirmation.request
- The /health endpoint returns 200 for both unauthenticated and invalid-auth requests (minimal response) — it never returns 401. Version header is `x-opentabs-version`.
- `extension_get_state` returns `networkCaptures` array with `{ tabId, requestCount, isCapturing }` — use with `waitFor` polling to verify async capture cleanup after tab close
- `config.setToolPermission` is a WebSocket JSON-RPC method — persistence is async (fire-and-forget via configWriteMutex), poll config.json with `waitFor` to verify writes

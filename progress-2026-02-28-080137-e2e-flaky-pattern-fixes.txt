## Codebase Patterns
- `waitFor` in e2e/helpers.ts is the canonical polling helper — takes (predicate, timeoutMs, intervalMs, label) and throws a descriptive error on timeout
- E2E helper imports live in e2e/helpers.ts; fixture helpers (startMcpServer, createMcpClient, etc.) in e2e/fixtures.ts
- ESLint `arrow-body-style` rule requires `() => expr` instead of `() => { return expr; }` for single-return arrow functions
- `testServer.invocations()` returns all recorded API calls — poll it with `waitFor` to deterministically confirm a request arrived (replaces fixed sleeps)
- `waitForExtensionDisconnected` polls the server health endpoint — cannot be used against a killed server (use fixed sleep instead)

## Codebase Patterns
- sanitize-error.ts: URL regex must run before path regexes. The Windows path regex `[a-z]:[/\\]` matches the `s:/` in `https://` and `p:/` in `http://`, consuming the trailing part of the protocol and producing `htt[PATH]` instead of `[URL]`.
- Vitest runs TypeScript source directly (no build step needed for tests).
- The pre-commit hook (lint-staged) runs prettier and eslint --fix automatically on staged files.
- detectPrimaryApiBaseUrl: prefix counts must be based on distinct endpoint count (+1 per endpoint, not +callCount), and minCount threshold uses endpoints.length (not sum of callCounts), so that the threshold genuinely requires multiple distinct endpoints to share a prefix.

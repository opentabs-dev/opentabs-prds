## Codebase Patterns
- CSS animations are defined in `platform/browser-extension/src/side-panel/styles.css` alongside Tailwind theme variables
- Tailwind CSS v4 is used; custom animations use `@keyframes` + utility class pattern
- The `animate-activity-flash` and `animate-activity-fade-out` classes are now the standard for tool activity indication
- **ESLint rule `react-hooks/set-state-in-effect`**: Never call setState synchronously in `useEffect` body — wrap ALL setState calls in callbacks (e.g., `setTimeout(..., 0)` for immediate triggers). Direct effect-body setState causes a lint error.
- **TypeScript `noImplicitReturns`**: If any code path in a function has an explicit `return <expression>`, ALL paths need explicit returns (or the function must be typed as returning `void`). Use `return;` for early exits.
- For `useEffect` fade-out patterns: use two timers — `setTimeout(() => setFadingOut(true), 0)` and `setTimeout(() => setFadingOut(false), 500)` — both in callbacks. Return cleanup that clears both timers.
- React components that use hooks (useState, useEffect, useRef) must place all hook calls BEFORE any early returns.

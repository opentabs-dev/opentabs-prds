## Codebase Patterns
- `restoreWsConnectedState` in `background-message-handlers.ts` is called once from `background.ts` at service worker startup. Pattern for awaitable one-shot async init: store the promise in a module-level variable, export `waitForXxx()` that returns it (or `Promise.resolve()` if not started).
- `handleBgGetFullState` uses an async IIFE pattern: `(async () => { ... })().catch(...)`. Await calls go inside the IIFE.
- Module-level state (`wsConnected`, `wsConnectedRestorePromise`) persists across tests in the same module import. The `beforeEach` resets `wsConnected` via `handleWsState({ connected: false })`.
- When adding a new export to `background-message-handlers.ts`, add it to both the `export { }` block at the bottom AND the destructured import in the test file.
- `server-state-cache.ts` uses pending optimistic update Maps (`pendingPluginToolUpdates`, `pendingBrowserToolUpdates`) that are re-applied after every `updateServerStateCache` call. This protects in-flight toggle states from `plugins.changed` overwrites. Pattern: register before the optimistic cache update, remove on server response (success or failure), then revert on failure.
- When adding new exports from `server-state-cache.ts`, update both the `export { }` block AND the `vi.mock('./server-state-cache.js')` in `background-message-handlers.test.ts` (which has a separate `vi.hoisted()` block + destructured const for mock variables).
- When adding new mocks to `message-router.test.ts`, update FOUR places: `vi.hoisted()` return object, destructured `const { }`, `vi.mock()` factory, AND `resetRoutingMocks()`. Also add `mockResolvedValue` in `beforeEach` for async mocks that need a default. Missing `resetRoutingMocks` causes call-count pollution across tests.
- `wrapAsync` in `message-router.ts` fires `fn(params, id).catch(...)` without returning the promise. Tests that check side effects after an async handler must `await Promise.resolve()` (twice) to flush microtasks after calling `handleServerMessage`.
- For `offscreen/index.ts` tests: mock Chrome APIs, WebSocket (as a class), and fetch via direct `globalThis` assignment BEFORE `await import('./index.js')`. Use `await new Promise(r => setTimeout(r, 0))` after import to let the async IIFE (bootstrapFromAuthFile + connect) complete. Use a class-based mock for WebSocket â€” `vi.fn().mockImplementation(factory)` is unreliable as a constructor mock for overriding Node.js built-ins.

## Codebase Patterns
- sdk-version.ts uses top-level await (ES module with top-level async), so cannot import logger.ts ‚Äî use console.error for logging in this file
- Prettier config requires long template literals in function calls to be on their own line with trailing comma
- Pre-commit hook runs prettier --write + eslint --fix automatically on staged files
- Import order in test files: local regular imports first, then external packages (vitest), then node built-ins, then type-only local imports last
- ESLint uses `import-x/consistent-type-specifier-style: prefer-top-level` ‚Äî always use separate `import type` statements, not inline `type` specifiers
- When adding a type-only import from the same module as an existing value import, it must be a separate `import type` statement placed AFTER the value import and after external package imports
- globalThis-based persistence pattern: use `__opentabs_<name>__` keys on `(globalThis as Record<string, unknown>)` for state that must survive hot reloads; always reset ALL globalThis keys in the `_reset*` test helper, not just one

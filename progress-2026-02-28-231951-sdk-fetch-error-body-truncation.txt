## Codebase Patterns
- The plugin-sdk fetch.ts test server is defined inline in the `beforeAll` block — add new route handlers there for new test scenarios
- For testing fetch internals that need mock responses (e.g., `.text()` rejection), use `vi.spyOn(globalThis, 'fetch')` — remember to add `vi` to the vitest imports and call `fetchSpy.mockRestore()` in `finally`
- ReadableStream with `controller.error()` in the `start` hook creates a response whose body stream errors immediately, causing `.text()` to reject — useful for testing error fallback paths
- Module-level constants in fetch.ts follow pattern `const CONSTANT_NAME = value;` just after the imports section

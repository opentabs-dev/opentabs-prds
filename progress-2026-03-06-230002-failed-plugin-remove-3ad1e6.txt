## Codebase Patterns
- Plugin management functions in plugin-management.ts use a configWriteMutex for serialized read-modify-write to config.json, with atomicWrite for safe file writes
- Extension handlers in extension-handlers.ts follow a consistent pattern: validate params, call business logic, trigger onReload, send plugins.changed notification, send success result
- Dispatch routes in extension-protocol.ts are simple if/return blocks matching method name + id presence
- Failed plugins only have their raw specifier string from config.json (not a plugin name), since they never enter the registry
- New bg: message types require changes in 4 files: extension-messages.ts (type + InternalMessage union), background-message-handlers.ts (handler + map + EXTENSION_ONLY_TYPES + export), offscreen/index.ts (relay switch case), bridge.ts (function + export)
- Storybook test utilities (fn, expect, etc.) import from `storybook/test`, not `@storybook/test`
- Biome enforces CSS class sorting via nursery/useSortedClasses — check ordering when combining Tailwind utilities

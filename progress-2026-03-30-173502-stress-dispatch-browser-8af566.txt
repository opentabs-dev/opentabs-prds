## Codebase Patterns
- E2E stress tests go in `test.describe('stress')` blocks at the end of existing E2E files
- Use `setupToolTest` for standard fixture setup (waits for extension connected, tab sync, opens test app tab, polls until tool is callable)
- `mcpServer.health()` returns a HealthResponse with `status` field for health checks
- Navigate to non-matching URL with `testServer.url.replace('localhost', '127.0.0.1')` pattern
- `parseToolResult` parses JSON string content from tool results into objects
- MCP protocol returns errors as fulfilled promises with `isError: true`, not rejected promises
- `Promise.allSettled` is preferred for concurrent calls where some may fail

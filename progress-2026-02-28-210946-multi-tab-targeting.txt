## Codebase Patterns
- Shared wire types live in `platform/shared/src/index.ts` and are imported by both mcp-server and browser-extension
- `TabMapping` in `platform/mcp-server/src/state.ts` mirrors the wire format and must stay in sync with shared types
- Wire parsing in `extension-handlers.ts` uses `WireTabMapping` / `WirePluginTabInfo` interfaces for defensive parsing (all fields optional, validated before use)
- Test files that set `state.tabMapping` must use the `TabMapping` shape from state.ts — multiple test files reference it: `extension-protocol.test.ts`, `http-routes.test.ts`, `reload.test.ts`
- The `/health` endpoint in `http-routes.ts` exposes per-plugin tab info from `state.tabMapping`
- The `config.getState` handler in `extension-handlers.ts` also reads `state.tabMapping` for tabState
- `PluginTabStateInfo` in `extension-messages.ts` has shape `{ state: TabState, tabs: PluginTabInfo[] }` — the old `{ state, tabId, url }` shape is gone
- `lastKnownState` cache in `tab-state.ts` stores serialized JSON strings (not `TabState`) — use `getAggregateState()` to extract the aggregate state from a cache entry
- `updateLastKnownState` takes `PluginTabStateInfo` (not `TabState`) — callers pass the full computed state
- `findAllMatchingTabs` in `tab-matching.ts` returns `chrome.tabs.Tab[]` with `.title` — pass it through to `PluginTabInfo.title`
- Extension test mocks for `tab-state.js` must include `getAggregateState` alongside `getLastKnownStates`
- `dispatchToTargetedTab` in `dispatch-helpers.ts` handles targeted dispatch (tab exists + URL pattern check + no fallback). `dispatchWithTabFallback` handles auto-select dispatch. Callers choose based on `tabId` presence.
- JSON-RPC error codes: -32001 = no usable tab, -32002 = adapter not ready, -32003 = tab URL not matched (security). All in `json-rpc-errors.ts`
- `handleToolDispatch`, `handleResourceRead`, and `handlePromptGet` all extract optional `tabId` from params and route to targeted vs fallback dispatch
- Browser tool registration requires 4 files: `browser-tools/<name>.ts` (definition), `browser-tools/index.ts` (barrel), `browser-tool-names.ts` (static list), `permissions.ts` (tier). Also update `e2e/helpers.ts` BROWSER_TOOL_NAMES.
- `defineBrowserTool` handler returns `Promise<unknown>` — for sync handlers use `Promise.resolve()` (ESLint rejects async without await)

## Codebase Patterns
- The shared package barrel is at platform/shared/src/index.ts; re-exports use `.js` extension (e.g., `'./generated/browser-tools-catalog.js'`)
- The tsconfig for shared (`platform/shared/tsconfig.json`) uses `"include": ["src"]` which covers `src/generated/` automatically
- Scripts in `scripts/` are type-checked by `tsconfig.scripts.json` which references `platform/shared`
- The browser tools barrel at `platform/mcp-server/src/browser-tools/index.ts` exports 40 tools as `browserTools: BrowserToolDefinition[]`
- `BrowserToolDefinition.icon` is optional (`icon?: string`); all current tools have explicit icons but the generator defaults to `'globe'`
- Generated files should be run through prettier (via `execFileSync` with `--parser typescript` and stdin input) to match project formatting conventions
- The root build script chains: `tsc --build && generate:browser-tools-catalog && build:bundle && build:side-panel && generate-icons && install-extension && npm rebuild`
- To use `@opentabs-dev/shared` in `e2e/*.ts`, add `"@opentabs-dev/shared": "*"` to root `package.json` devDependencies and run `npm install`; the e2e tsconfig resolves it via moduleResolution `bundler` through node_modules without needing a project reference
- Never put non-import code between import blocks — ESLint's import-x plugin will error with "Import in body of module; reorder to top"
- ESLint import order groups: `['index', 'sibling', 'parent', 'internal', 'external', 'builtin', 'object', 'type']`; scoped packages (`@*/**`) are "internal" and come before external packages (react, lucide-react)

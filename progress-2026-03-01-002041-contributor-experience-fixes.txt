## Codebase Patterns
- React Compiler handles memoization: do NOT use useCallback/useMemo/React.memo
- For useEffect exhaustive-deps: if a function is only called inside the effect, move it inside the effect (safe when it uses only refs and stable state setters)
- For functions needed both in JSX and in an effect (e.g., handleSearchChange), inline equivalent logic in the effect instead of adding the function to the dependency array
- State setters (setFoo) and refs (fooRef.current) are always stable — safe to use inside effects without listing in deps

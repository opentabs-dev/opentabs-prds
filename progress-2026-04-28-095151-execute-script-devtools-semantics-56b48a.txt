## Codebase Patterns
- writeExecFile in adapter-files.ts uses server-side vm.Script to detect expression vs statement code, then inlines the appropriate async wrapper (no eval/new Function in browser). This is critical for strict-CSP compatibility.
- eval() and new Function() inside browser-executed code ARE subject to page CSP even when the wrapper file is injected via chrome.scripting.executeScript. File injection bypasses CSP for the FILE itself, but not for dynamic code execution within it.
- The generated browser-tools-catalog.ts is auto-regenerated by `npm run generate:browser-tools-catalog` as part of `npm run build` — no need to manually update it.
- adapter-files.test.ts uses OPENTABS_CONFIG_DIR env var to isolate tests to a temp directory; each test suite sets this globally with afterAll cleanup.
- The pre-commit hook runs biome-check-ts on staged .ts files — changes must pass Biome lint + format before committing.
- If E2E tests fail with "Executable doesn't exist at /ms-playwright/chromium-XXXX", run `npx playwright install chromium` to install the matching Chromium version. This happens when Playwright npm version is updated but Docker image still has old binaries.

## Codebase Patterns
- Server-only fields (source, reviewed, npmPackageName, sdkVersion, update) come from the MCP server, not the plugin adapter. They are extracted from raw WebSocket JSON with runtime type validation in extractServerOnlyFields.
- handleBgGetFullState spreads serverPlugin to inherit all server-sourced fields automatically — new fields added to ConfigStatePlugin don't need manual enumeration here.
- The spread order in handleBgGetFullState matters: metaFields → serverOnlyDefaults → serverPlugin → explicit overrides (permission, tools, tabState, tabs). Later spreads override earlier ones.

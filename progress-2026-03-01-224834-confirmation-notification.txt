## Codebase Patterns
- Side panel open/closed state is tracked per-window in `side-panel-toggle.ts` via `chrome.sidePanel.onOpened/onClosed` (Chrome 141+). The `openWindows` Set is persisted to `chrome.storage.session`.
- Confirmation badge logic lives in `confirmation-badge.ts` — manages badge counts, background timeouts, and Chrome desktop notifications for confirmation requests.
- Module-level state in Chrome MV3 service workers is volatile (wiped on suspend/wake). State that needs to survive is persisted to `chrome.storage.session` and restored on wake.
- Tests for Chrome extension modules use dynamic `await import()` after setting up `globalThis.chrome` mocks. Use `vi.hoisted` + `vi.mock` for module-level mocking.

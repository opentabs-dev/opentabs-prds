## Codebase Patterns
- `pluginCacheKey` in `platform/plugin-tools/src/commands/build.ts` (line 47) is a module-scope counter used to bust the Node.js ESM module cache via `?t=N` query param on dynamic imports
- Node.js ESM module cache is keyed by the full URL (including query string), so each unique `?t=N` gets a fresh module load
- plugin-tools build.ts uses spawnSync for running tsc; on Windows, `shell: true` is required to resolve `.cmd` wrappers in node_modules/.bin
- `fetchJSONImpl` in `platform/plugin-sdk/src/fetch.ts` handles 204 No Content by returning `undefined` (no schema) or throwing `ToolError.validation` with a clear "no body to validate" message (with schema)

## Codebase Patterns
- executeInTab is a local helper in analyze-site/index.ts that wraps dispatchToExtension for browser.executeScript — it unwraps nested results and returns unknown
- Chrome can truncate large JSON payloads (>50KB) to strings instead of parsed objects — guard with typeof + first-char check
- PR #38 established the Array.isArray() validation pattern for executeInTab call sites expecting arrays
- The pageTitle call at line 954 legitimately returns a string (document.title) — truncation guard only triggers for '{' or '[' prefixed strings
- dispatchToAllConnections returns `{ connectionId: string; result: unknown }[]` — always validate result before spreading or accessing properties
- Standard object-type dispatch result guard: `x !== null && typeof x === 'object' && !Array.isArray(x)` then cast to `Record<string, unknown>`
- For numeric properties from dispatch results, validate with `typeof x === 'number'` before arithmetic to prevent NaN

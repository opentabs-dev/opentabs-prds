## Codebase Patterns
- Telemetry system lives in `platform/mcp-server/src/telemetry.ts` — exports `trackEvent(event, properties?)` and `getSessionId()`
- Events are fire-and-forget, no PII. Import as `import { getSessionId, trackEvent } from './telemetry.js';`
- `handlePluginInstall` is at line ~917 in extension-handlers.ts. The catch block error is typed as `unknown` — use `err instanceof Error` for message access.
- Error categorization pattern: lowercase the error message, check for keywords like 'timed out', 'not a valid opentabs plugin', default to generic category
- To test telemetry events: use `vi.hoisted(() => ({ mockTrackEvent: vi.fn() }))` + `vi.mock('./telemetry.js', ...)` pattern
- serverSourcePath/isProductionInstall are module-level constants from import.meta.url — mock `node:url` fileURLToPath to inject 'node_modules' into path for testing production-only code paths
- ConfigSchema `url` type expects `Record<string, string>` (instance name → URL map), not a plain string

## Codebase Patterns
- Plugin lockfiles are standalone (not part of root npm workspace). To update a transitive dep like esbuild: delete the lockfile AND remove the dep from node_modules (e.g., `rm -f package-lock.json && rm -rf node_modules/@esbuild node_modules/esbuild && npm install`). Just deleting the lockfile isn't enough if node_modules already has the old version satisfying the range.
- The root package-lock.json may have pre-existing modifications (e.g., version range changes) that are not part of a story's scope. Check git status at the start and only stage story-relevant files.

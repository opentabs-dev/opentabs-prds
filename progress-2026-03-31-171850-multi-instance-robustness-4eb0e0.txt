## Codebase Patterns
- `deriveMatchPattern` in `platform/mcp-server/src/settings-resolver.ts` uses `url.host` (includes port for non-standard ports) rather than `url.hostname` (strips port)
- Chrome match patterns support ports: `*://hostname:port/*` is valid syntax
- `new URL().host` strips default ports (80 for HTTP, 443 for HTTPS) automatically ŌĆö no manual port filtering needed
- Three locations compare tab URLs to match patterns: `findTabForInstance` (mcp-tool-dispatch.ts), `annotateTabsWithInstance` (plugin-list-tabs.ts), `resolvePerTabSettings` (iife-injection.ts) ŌĆö all need `.host` instead of `.hostname` for port-aware matching
- **E2E multi-instance tests must use localhost with different ports, NOT 127.0.0.1** ŌĆö `chrome.tabs.query` in Playwright Chromium doesn't reliably match IP-address-based match patterns. The `SameHostTestContext` (different ports) approach is the proven pattern.

## Codebase Patterns
- E2E tests in `e2e/plugin-management.e2e.ts` use a local `connectWs` helper for WebSocket JSON-RPC communication
- Tests requiring custom server env vars (e.g., NPM_CONFIG_PREFIX) create standalone servers via `startMcpServer()` instead of using the shared `mcpServer` fixture
- `NPM_CONFIG_PREFIX` isolates npm global operations to a temp dir — proven pattern from `npm-discovery.e2e.ts`
- `OPENTABS_SKIP_NPM_DISCOVERY: undefined` in env overrides removes the env var, enabling npm auto-discovery
- `createMcpClient` creates an MCP HTTP client; must call `.initialize()` before `.listTools()`
- The `sendRequest` helper in `connectWs` now supports a `timeoutMs` parameter (default 30s) for long-running operations
- After successful install in the side panel, `handleSearchChange('')` auto-clears the search — the plugin moves from Available to the main installed list
- PluginCard accordion trigger: `button[aria-expanded]` with `.filter({ hasText: 'DisplayName' })` for Playwright targeting
- NpmPluginCard display name is lowercase (`extractShortName`), PluginCard uses capitalized `displayName` from server

## 2026-03-31 07:10 UTC - US-003
- Fixed permission-change and auth-toggle stress tests across 3 E2E files
- Root cause 1 (permission tests): `triggerHotReload()` (SIGUSR1) kills the worker process, interrupting in-flight calls. Tests expected in-flight calls to survive permission changes, but hot reload destroys them.
- Fix 1: Use `POST /reload` (config reload) instead of `triggerHotReload()`. Config reload re-reads config.json and updates permissions in-place without restarting the worker. In-flight calls that already passed permission check at dispatch entry survive.
- Root cause 2 (auth toggle test in tool-dispatch.e2e.ts): `setSlow(3000)` delays the test server response BEFORE `requireAuth()` check. Toggling auth during the delay means the auth check (which runs after the delay) finds auth=false and rejects.
- Fix 2: Use `slow_with_progress` tool instead of `echo + setSlow`. `slow_with_progress` sleeps on the adapter side without calling the test server, so auth changes don't affect it.
- Also fixed the same permission test pattern in permissions.e2e.ts (identical triggerHotReload bug)
- Tab-close test already worked with existing `setupToolTest` warmup (no changes needed)
- Files changed: e2e/stress-mcp-dispatch.e2e.ts, e2e/permissions.e2e.ts, e2e/tool-dispatch.e2e.ts
- **Learnings for future iterations:**
  - `triggerHotReload()` kills the worker process — NEVER use it when you need in-flight calls to survive. Use `POST /reload` for config-only changes.
  - `POST /reload` triggers `performConfigReload()` which calls `reloadCore()` — it re-reads config.json, rediscovers plugins, and updates permissions without restarting the worker process.
  - Wait for `'Config reload complete'` log after `POST /reload` (not `'Hot reload complete'` which is for SIGUSR1).
  - The test server's `setSlow(delayMs)` adds delay BEFORE `requireAuth()` check in the request handler. Don't use setSlow for "in-flight" tests that toggle auth — use `slow_with_progress` instead (sleeps on adapter side).
  - `mcpServer.secret` is available on the fixture object for authenticating POST /reload calls.

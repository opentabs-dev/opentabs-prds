## Codebase Patterns
- CLAUDE.md files for platform packages are at platform/<name>/CLAUDE.md; plugins/CLAUDE.md covers all plugins; root CLAUDE.md is the project overview
- The e2e-test plugin's node_modules/@opentabs-dev/plugin-sdk is symlinked to platform/plugin-sdk, so SDK changes break the plugin immediately
- build-e2e.test.ts copies plugins/e2e-test and runs opentabs-plugin build on it — e2e-test plugin source must be kept in sync with SDK exports
- platform/create-plugin/src/index.test.ts checks the shape of dist/tools.json from a scaffolded plugin build — update its type assertions when manifest shape changes
- Removing exported types from platform/plugin-sdk causes immediate downstream breakage in plugin-tools/build.ts which imports them directly; these must be cleaned up in the same story to keep the build green
- The resources/prompts in PluginManifestOutput (build.ts), generateManifest(), and build summary logging all need updating together when removing resource/prompt support from the build pipeline

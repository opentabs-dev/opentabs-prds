## Codebase Patterns
- Tool rendering in PluginCard and BrowserToolsCard follows identical patterns — both use `groupTools()` which returns `ToolGroup[]` (grouped) or `null` (flat list)
- CSS class ordering matters for Biome lint: pseudo-classes like `last:` must come before pseudo-classes like `hover:` per `useSortedClasses` rule
- PluginCard tools are `WireToolDef` (from shared), BrowserToolsCard tools are `BrowserToolState` (alias for `ConfigStateBrowserTool`)
- Both cards render tool lists in two paths: grouped (with group headers) and flat (no groups). Any new tool list behavior must be applied to both paths
- React Compiler handles memoization — no need for `useMemo` or `useCallback`

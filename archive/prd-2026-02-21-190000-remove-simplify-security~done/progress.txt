## Codebase Patterns
- Platform source files in `platform/mcp-server/src/` are compiled via `tsc --build` with project references
- When deleting `.ts` files, use `rm` from Bash (the Edit tool's delete is unreliable, and files may be restored by lint-staged or hooks if not properly staged)
- The pre-commit hook runs prettier, eslint, and knip on staged files via lint-staged
- TrustTier type from `@opentabs-dev/shared` is used by `npmTrustTier()` in discovery.ts ‚Äî keep the import even when removing verification code
- When removing import lines, avoid leaving blank lines between import groups ‚Äî ESLint's `import-x/order` rule rejects them
- `sanitizeOutput` in mcp-setup.ts (prototype pollution prevention) is distinct from the deleted `sanitizeToolOutput` (path/URL scrubbing) ‚Äî keep it
- E2E tests that replace the extension's WS slot use `fetchWsInfo` to get wsUrl + wsSecret, then connect with protocols `['opentabs', wsSecret]` ‚Äî search `e2e/` broadly when removing WS-related features
- `plugins/e2e-test` must be built before running `build-e2e.test.ts` ‚Äî the test copies the plugin and needs `dist/index.js` to exist
- Features threading through the full dispatch pipeline (shared types ‚Üí plugin-tools ‚Üí mcp-server ‚Üí browser-extension ‚Üí E2E) touch 20+ files ‚Äî use `grep` to find ALL occurrences before editing

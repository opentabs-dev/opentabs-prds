## Codebase Patterns
- Retro UI components follow compound component pattern using `Object.assign(Root, { SubComponent })` â€” see Tooltip.tsx, Accordion.tsx, Menu.tsx
- Retro components wrap Radix UI primitives with theme-aware Tailwind classes; each gets a `.stories.tsx` file
- The `plugins/e2e-test/` directory must have `node_modules/` and `dist/` for `build-e2e.test.ts` to pass â€” run `bun install && bun run build` in the plugin dir if tests fail
- The `@opentabs-dev/shared` dependency versions in platform packages must match the local package version
- Prettier enforces specific Tailwind class ordering (e.g., `border-border bg-card` before `z-50 w-56`)
- Import ordering: retro components alphabetically, then other local imports, then external
- Simple retro components (no sub-components) use plain `cva` + `cn` pattern; compound pattern (`Object.assign`) only when there are sub-components
- ESLint forbids non-null assertions (`!`) â€” use optional chaining (`?.`) with nullish coalescing (`?? null`) instead

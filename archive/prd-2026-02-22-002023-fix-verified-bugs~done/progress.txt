## Codebase Patterns
- The e2e-test plugin in `plugins/e2e-test/` needs `bun install && bun run build` before unit tests pass — its `dist/` and `node_modules/` are gitignored and not present in fresh worktrees
- The `build-e2e.test.ts` tests copy the e2e-test plugin to temp dirs and run builds; they depend on the plugin's `dist/index.js` and `node_modules/` being present
- The globals scan script in `analyze-site/index.ts` is a string literal executed in the browser context — it's not TypeScript-checked at build time
- `bun-types` narrows some DOM types compared to standard lib (e.g., `Element.textContent` is `string` not `string | null`). When adding null safety for DOM properties, use `as string | null` type assertion to widen the type, then check explicitly — optional chaining alone triggers `@typescript-eslint/no-unnecessary-condition`
- Use `bun run test` to run unit tests — root `package.json` has `"test": "bun test platform/"` which scopes tests to the platform directory
- Manifest types: `ManifestResource` requires `uri`+`name`; `ManifestPrompt` requires `name` only; `ManifestPromptArgument` requires `name` only. All other fields are optional.

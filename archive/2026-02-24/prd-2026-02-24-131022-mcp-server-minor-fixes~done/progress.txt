## Codebase Patterns
- reload.test.ts needs npm auto-discovery isolation: set `(globalThis as Record<string, unknown>).__opentabs_global_paths__ = []` in beforeEach and call `resetGlobalPathsCache()` in afterEach to prevent real globally-installed npm plugins from being discovered during tests.
- The globalThis key `__opentabs_global_paths__` caches global node_modules directories. Setting it to `[]` makes auto-discovery skip scanning. Setting it to `null` (via `resetGlobalPathsCache()`) forces re-evaluation from shell commands.
- The create-plugin install test (`index.test.ts`) can be flaky due to bun symlink race conditions when run in parallel with other tests — passes when run alone.
- E2E tests use `OPENTABS_SKIP_NPM_DISCOVERY=1` env var (via `skip-npm-discovery.ts`) to prevent globally-installed npm plugins from polluting test isolation. The env var follows the same pattern as `OPENTABS_SKIP_CONFIRMATION` and `OPENTABS_SKIP_SANITIZATION`.

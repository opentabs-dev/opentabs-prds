## Codebase Patterns
- CDP event handlers in network-capture.ts follow a consistent pattern: extract requestId from params, look up state from pendingRequests/requestIdToRequest maps, build a CapturedRequest, push to requests array (with shift at maxRequests cap), and clean up tracking maps.
- The reload.test.ts performConfigReload tests run in environments where globally installed npm plugins (e.g., slack) are auto-discovered. Tests checking plugin counts should use toBeGreaterThanOrEqual rather than exact counts.
- Internal Chrome messages (side panel ↔ background) use typed InternalMessage union in extension-messages.ts. New message types require: (1) interface in extension-messages.ts, (2) add to InternalMessage union, (3) handler in background-message-handlers.ts dispatch map.
- The knip config has `ignoreExportsUsedInFile: true`, so exports used within the same file (e.g., handler functions referenced in a Map) are not flagged as unused.

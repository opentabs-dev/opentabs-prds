## Codebase Patterns
- CLI commands are in platform/cli/src/commands/. Each file registers a top-level command.
- Commander's `.action()` receives (args..., options, command). Use `command.optsWithGlobals()` to access global options like `--port`.
- The `resolvePort()` function from parse-port.ts resolves port from options → env → default.
- Shared helpers that multiple commands need belong in platform/cli/src/ (not inside commands/).
- Pre-existing `tsc --build --noEmit` may fail in worktrees with stale build info. Run `bun run clean && bun run build` first to fix.

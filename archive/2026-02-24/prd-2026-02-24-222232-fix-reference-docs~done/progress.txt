## Codebase Patterns
- docs/ is a standalone Next.js project — run quality checks with `cd docs && bun run <check>` not from repo root
- `bun install` is required before running docs checks in a fresh worktree (Next.js binary `next` is not on PATH otherwise)
- docs checks: build, type-check, lint, knip, format:check (no test suite)
- The doctor command in platform/cli/src/commands/doctor.ts has 11 checks in this order: Runtime, Browser, Config file, Auth secret, MCP server, Extension connection, Extension installed, Extension version, MCP client config, Local plugins, npm plugins
- The `config show` command has --json and --show-secret flags; the --show-secret flag displays the full authentication secret instead of masking it
- The `rotate-secret` command writes to auth.json only (not config.json)
- The `update` command checks current vs latest npm version, warns if server is running, then runs `npm install -g @opentabs-dev/cli@<version>`

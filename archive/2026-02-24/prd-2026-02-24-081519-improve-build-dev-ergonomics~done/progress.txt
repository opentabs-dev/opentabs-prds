## Codebase Patterns
- The dev orchestrator is in scripts/dev.ts; it spawns tsc watch, extension builds, and the MCP server as child processes
- The dev:mcp script in package.json directly runs `bun --hot platform/mcp-server/dist/index.js --dev` — the `--dev` flag is a script argument, not a bun argument
- scripts/dev.ts uses Bun.spawn with process group management (children array) for cleanup on exit
- `tsc --build --noEmit` works correctly with TypeScript 5.9.3 and the project's composite references setup
- The `check` pipeline should not include `type-check` since `build` (which uses `tsc --build`) already type-checks
- The `build` script should be incremental by default (no `--force`); `build:force` exists for full rebuilds
- When removing package.json scripts, also update: .github/workflows/ci.yml, CONTRIBUTING.md command table, and CLAUDE.md commands section
- scripts/dev.ts calls extension build steps directly via Bun.spawn, not via `bun run` script names
- The docs project has its own `check` script; root `check:docs` delegates to it via `bun run --cwd docs check`
- When renaming package.json scripts, check: CLAUDE.md, CONTRIBUTING.md, README.md, .github/workflows/ci.yml (CHANGELOG.md is historical — leave it)

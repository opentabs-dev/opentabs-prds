## Codebase Patterns
- Runtime compatibility layer is in `platform/shared/src/runtime.ts` — every function has a Bun path and a Node.js path
- Bun path and Node.js path should have matching error-handling behavior (e.g., both swallow ENOENT with `.catch(() => {})`)
- Prettier enforces line breaks on chained method calls — `await Bun.file(path)\n.delete()\n.catch(...)` not inline
- Node.js `spawnSync` sets `result.error` on ENOENT — check it before accessing `stdout`/`stderr` (which are `null` in that case)
- ESLint `@typescript-eslint/no-unnecessary-condition` catches `?.` on types TS thinks are non-null — use early-return on `result.error` instead
- ESLint `@typescript-eslint/no-non-null-assertion` forbids `!` — avoid `let x; ... x!` patterns; instead extract callbacks from Promise constructors and assign timer IDs separately
- CLI commands fetching `/health` must include auth header: `const secret = await readAuthSecret(); const headers: Record<string, string> = {}; if (secret) headers['Authorization'] = \`Bearer ${secret}\`;`
- SVG attribute regex patterns must use `["']` for quote delimiters and `[^"']` for content to handle both single- and double-quoted attributes

- **Files changed**: `platform/shared/src/runtime.ts` (line 264)

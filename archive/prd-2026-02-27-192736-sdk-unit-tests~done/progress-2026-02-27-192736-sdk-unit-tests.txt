## Codebase Patterns
- fetch.test.ts uses a real Node.js HTTP server (node:http createServer on port 0) — no mocks
- The server handler is a single function with if-chain routing before the final 404 fallback
- New test routes must be added inside the handler before `res.writeHead(404)` at the bottom
- For verifying Content-Type in tests, `/echo-headers` returns `{ contentType, hasCookies }`
- For verifying HTTP method + body, use a custom `/echo-method` route returning `{ method, contentType, received }`
- For verifying form body, use a custom `/echo-form` route returning `{ body, contentType, method }`
- In Node.js 22+ (undici), passing FormData to fetch automatically sets Content-Type: multipart/form-data with boundary
- Use `npm run lint:fix` to auto-fix prettier formatting errors rather than manual edits

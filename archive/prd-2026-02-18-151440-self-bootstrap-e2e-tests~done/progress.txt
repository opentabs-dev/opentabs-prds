## Codebase Patterns
- CDP (Chrome DevTools Protocol) does NOT capture network requests from scripts injected via `chrome.scripting.executeScript` with `files` — only inline page scripts and navigation requests are captured. For network capture E2E tests, use navigation to pages with inline `<script>` tags that make fetch calls, NOT `browser_execute_script`.
- `browser_get_page_html` returns `{title, url, html}` on success, or a JSON-RPC error with `Element not found: <selector>` message on failure (isError=true).
- `browser_get_storage` returns `{key, value}` for single-key lookup, or `{entries: [{key, value}...], count}` for all-entries lookup.
- `BROWSER_TOOL_NAMES` in `e2e/helpers.ts` must include all browser tool names for the tool listing assertion test to pass.

  - Prettier requires long `.find()` callbacks to be on one line (not multi-line with trailing comma). Watch for `prettier/prettier` lint errors on arrow functions inside `.find()`.
  - `waitForNetworkRequests` returns as soon as ANY request appears — for specific request matching, use a custom `waitFor` loop that polls `browser_get_network_requests` with a predicate.

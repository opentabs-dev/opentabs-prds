## Codebase Patterns
- `clearTabStateCache()` in tab-state.ts is called on WebSocket disconnect (background.ts). It must clear both `lastKnownState` and `pluginLocks` maps to avoid stale promise chains surviving into the next connection session.
- `clearPluginTabState(pluginName)` already clears both maps for a single plugin — the full cache clear should be consistent.
- For testing modules with module-level state (e.g., rate-limiter's `methodTimestamps` Map), do NOT use `jest.useFakeTimers()` to mock `Date.now()` — bun:test runs all files in the same process and `jest.useRealTimers()` in other test files will overwrite the mock. Instead, add an optional `now` parameter to the function and a `resetForTesting` export to clear module state.
- `tsconfig.test.json` in `platform/browser-extension/` includes `src/**/*.test.ts`, so new test files are automatically covered.
- In `sanitize-error.ts`, regex replacements run in order: Windows paths → Unix paths → URLs → localhost → IPs. The Unix path regex catches parts of URLs (e.g., `//example.com/path`) before the URL regex runs. When testing URL sanitization, assert that sensitive content is absent (`.not.toContain`) rather than expecting a specific `[URL]` token.

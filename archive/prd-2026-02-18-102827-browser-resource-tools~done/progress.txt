## Codebase Patterns
- When adding a new browser tool, touch 6 files: (1) browser-tools/<tool>.ts, (2) browser-tools/index.ts, (3) browser-commands.ts, (4) message-router.ts, (5) message-router.test.ts, (6) offscreen/index.ts ALLOWED_METHODS
- browser-commands.ts uses `import-x/exports-last` — all non-export statements (interfaces, types, helper functions) must go BEFORE the first `export const`
- message-router.test.ts mocks browser-commands.js — new handlers must be added to: (a) mock variable, (b) mock.module call, (c) resetRoutingMocks, (d) beforeEach mockResolvedValue, (e) routing test
- CDP debugger management pattern: check isCapturing(tabId) to reuse existing session, otherwise attach/detach in try/finally
- CDP types (CdpFrame, CdpResource, CdpFrameResourceTree) are defined at file scope in browser-commands.ts for reuse across handlers
- browser-tools/index.ts imports are alphabetically sorted — insert new imports in sort order
- CDP Page.getResourceContent requires a frameId — must walk Page.getResourceTree to find the owning frame first
- CDP Page.getResourceTree only reports external sub-resources (loaded via src=), not the main document or inline scripts — test pages must load external resources for resource tool E2E tests
- After adding new browser extension handlers, must delete .tsbuildinfo files to force a clean rebuild — stale incremental build cache causes the bundler to omit new code

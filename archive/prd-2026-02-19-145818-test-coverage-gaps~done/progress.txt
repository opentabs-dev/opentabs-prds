## Codebase Patterns
- `{ __proto__: ... }` in JS object literals sets the prototype chain, not an own key. Use `JSON.parse('{"__proto__":"val"}')` to simulate deserialized JSON with dangerous keys in tests.
- Check own property existence with `Object.hasOwn(obj, key)` not `'key' in obj` â€” `in` traverses the prototype chain and `__proto__` is always accessible via `Object.prototype`.
- `sanitizeOutput` uses `Object.entries()` which only yields own enumerable properties, so dangerous keys in JSON-deserialized objects (own keys) are filtered, but prototype-set `__proto__` would not appear in entries at all.
- To add a new export to `mcp-setup.ts`, add to the named export list at the bottom (line ~498).
- `getNextRequestId()` in state.ts uses `crypto.randomUUID()` â€” request IDs are UUID strings, NOT numbers. Mock WS checking for request IDs must use `parsed['id'] !== undefined` NOT `typeof parsed['id'] === 'number'`.
- Resolving a `new Promise()` from within its own executor synchronously (even via queueMicrotask) does NOT work in Bun 1.3.9. Use `setTimeout(fn, 0)` to defer response settlement in mock WS objects.
- `mock.module('./extension-protocol.js', ...)` leaks across test workers in Bun 1.3.9. Prefer no module mocking: use self-completing mock WS objects that call `handleExtensionMessage` via `setTimeout(fn, 0)`.
- When using `mock.module()`, always include ALL exports from the mocked module â€” partial mocks break other test files that import missing exports.

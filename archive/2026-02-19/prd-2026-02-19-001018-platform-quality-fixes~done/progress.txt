## Codebase Patterns
- Platform packages are in `platform/`, plugins in `plugins/`
- `getNextRequestId()` in `state.ts` generates JSON-RPC IDs via `crypto.randomUUID()` — used by extension-protocol.ts, http-routes.ts, and browser-tools/reload-extension.ts
- The hot-reload patching loop in `index.ts` iterates `createState()` defaults and patches missing keys — only top-level keys, no structural changes
- `STATE_SCHEMA_VERSION` only needs bumping when changing the **type** of an existing field, not when removing an optional field
- `import-x/exports-last` ESLint rule requires that once exports start in a file, all remaining declarations must be exports — non-exported helpers must be placed BEFORE the first `export const`
- `document.execCommand` triggers `@typescript-eslint/no-deprecated` — use Selection/Range API instead for contentEditable text insertion
- `const` arrow functions in `methodHandlers` Map initializer must not reference functions defined later in the file (no hoisting) — either move them above the Map or keep inline lambdas
- Browser extension code (`browser-commands.ts`) uses `chrome.scripting.executeScript` with `world: 'MAIN'` — DOM APIs inside the func callback run in the page context, but ESLint still checks them against TypeScript types
- The `plugins/slack/opentabs-plugin.json` file had a pre-existing uncommitted diff (adapterHash change) — be careful not to accidentally stage it

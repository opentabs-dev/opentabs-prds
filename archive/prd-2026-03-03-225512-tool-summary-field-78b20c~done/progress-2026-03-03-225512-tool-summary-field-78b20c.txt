## Codebase Patterns
- Optional fields in wire protocol use conditional spread: `...(value ? { key: value } : {})`
- Validation of incoming wire payloads uses `typeof x === 'string'` checks before including optional fields
- CachedBrowserTool in state.ts mirrors BrowserToolDefinition but with pre-computed JSON Schema; both need updating when adding new fields
- The summary field flows: ToolDefinition (SDK) → tools.json (build) → ManifestTool (loader) → WireToolDef (extension-handlers) → message-router (extension)
- For browser tools: BrowserToolDefinition → CachedBrowserTool → ConfigStateBrowserTool (extension-handlers buildConfigStatePayload)
- Browser tool definitions place fields in order: name, description, summary, icon, group, input, handler
- When adding a new browser tool, include a `summary` field for UI consistency with all existing tools

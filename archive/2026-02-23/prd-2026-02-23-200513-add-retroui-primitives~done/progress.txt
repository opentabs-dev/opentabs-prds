## Codebase Patterns
- Retro components live in `platform/browser-extension/src/side-panel/components/retro/`
- All retro components use `cn()` from `../../lib/cn` for className composition
- Compound components use `Object.assign(Root, { SubComponent })` pattern with a named export
- Import Radix primitives as namespace: `import * as Foo from '@radix-ui/react-foo'`
- Use `React.forwardRef` with `React.ComponentRef<typeof Primitive>` and `React.ComponentPropsWithoutRef<typeof Primitive>`
- Set `.displayName` to the Radix primitive's `.displayName`
- No `'use client'` directive (not Next.js)
- Storybook stories use `Meta` and `StoryObj` types, with `title: 'Retro/ComponentName'`
- Theme-aware colors: `border-border`, `bg-card`, `hover:bg-accent`, `data-[highlighted]:bg-accent` for dropdown items
- Simple non-compound components (Badge, Button) use `forwardRef` + named export without `Object.assign`
- cva-based components pattern: define variants → interface extending `HTMLAttributes` + `VariantProps` → `forwardRef` component
- `@opentabs-dev/shared` in mcp-server needed workspace:* resolution fix (was ^0.0.22, local is 0.0.23)
- The `plugins/e2e-test` directory needs `bun install && bun run build` before unit tests pass (not part of bun workspaces)
- Prettier enforces Tailwind class ordering — theme classes (border-border, bg-background) sort before utilities (h-4, w-full)

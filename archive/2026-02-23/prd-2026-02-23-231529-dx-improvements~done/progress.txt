## Codebase Patterns
- Scripts live in `scripts/` and are type-checked by `tsconfig.scripts.json` (includes `scripts/**/*.ts`)
- Knip config in `knip.ts` lists `scripts/*.ts` as entry points â€” new scripts are auto-discovered
- ESLint enforces `func-style` â€” use arrow function expressions, not function declarations
- Use `rmSync` from `node:fs` for directory removal (no Bun equivalent)
- Use `readdirSync` from `node:fs` for directory listing (no Bun equivalent)
- `Bun.argv` for CLI argument parsing
- Generated build artifacts include: `platform/*/dist/`, `platform/browser-extension/icons/`, `*.tsbuildinfo` files
- `platform/browser-extension/side-panel/` contains SOURCE files (HTML, fonts, CSS) tracked by git â€” NOT build artifacts. Do not delete with clean.
- `bun run --cwd <dir> <script>` works for standalone subprojects (docs/, plugins/*) with their own package.json

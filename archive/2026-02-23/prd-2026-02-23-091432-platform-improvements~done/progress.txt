## 2026-02-23 09:40 PST - US-001
- **Implemented**: Fixed SVG sanitizer strippedDepth tracking bug
- **Root cause**: The closing tag handler inside a stripped subtree only decremented strippedDepth for disallowed elements (line 280-281), but the opening tag handler incremented it for both allowed and disallowed elements (lines 295-296 and 304-305). This mismatch meant allowed elements opened inside a stripped subtree left strippedDepth elevated, causing subsequent content after the stripped subtree to be incorrectly stripped.
- **Fix**: Simplified the closing tag handler to decrement strippedDepth unconditionally for any closing tag when strippedDepth > 0, matching the opening tag behavior.
- **Files changed**:
  - `platform/browser-extension/src/sanitize-svg.ts` — simplified closing tag logic in stripped subtree
  - `platform/browser-extension/src/sanitize-svg.test.ts` — added 2 test cases for the specific acceptance criteria
- **Learnings for future iterations:**
  - The e2e-test plugin must have dist/ and node_modules/ present for build-e2e.test.ts to pass — run `cd plugins/e2e-test && bun install && bun run build` if tests fail with ENOENT errors
  - knip configuration hints (about ignoreDependencies) are informational and don't cause failure

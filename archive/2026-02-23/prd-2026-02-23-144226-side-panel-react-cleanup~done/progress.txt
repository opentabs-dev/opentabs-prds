## Codebase Patterns
- Worktree `bun install` may create stale symlinks for workspace packages (e.g., `platform/mcp-server/node_modules/@opentabs-dev/shared` pointing to bun cache instead of local source). Fix by re-creating the symlink: `rm <link> && ln -s ../../../shared <link>`.
- Root `node_modules/@opentabs-dev/` may be missing entirely in worktrees. Create it with `mkdir -p node_modules/@opentabs-dev && ln -s ../../platform/shared node_modules/@opentabs-dev/shared` (and similarly for other workspace packages).
- The `plugins/e2e-test` directory is standalone (not in bun workspaces). Its `node_modules` and `dist` must be built separately via `cd plugins/e2e-test && bun install && bun run build` before unit tests that reference it will pass.
- Prettier in this project puts the closing `>` of multi-line JSX attributes on the same line as the last attribute, not on a new line.

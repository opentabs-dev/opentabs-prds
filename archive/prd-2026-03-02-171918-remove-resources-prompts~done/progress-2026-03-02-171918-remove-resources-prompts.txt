## Codebase Patterns
- When removing exports from plugin-sdk, downstream packages (plugin-tools, browser-extension, mcp-server) immediately break at build time — must fix all consumers to keep `npm run build` green.
- The build-e2e.test.ts tests run the actual plugin build via `opentabs-plugin build`, so the e2e-test plugin source must also be kept in sync when removing SDK exports.
- create-plugin/src/index.test.ts scaffolds a plugin and builds it — update assertions that check manifest structure when fields are removed.
- `plugins/e2e-test/src/index.ts` is the real plugin used by E2E tests AND by build-e2e.test.ts fixture tests — must be updated when SDK types change.
- The PRD notes for each story list the exact line numbers to remove, but the actual file content may differ slightly (e.g., different string values in tests) — always read the file before editing.
- US-002 (shared package) intentionally leaves the full build broken — only run `cd platform/shared && npm run build` for verification. Downstream consumers (plugin-tools, mcp-server, browser-extension) will be fixed in subsequent stories.

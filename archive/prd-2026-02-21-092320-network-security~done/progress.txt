## Codebase Patterns
- Bun.serve() is in the FROZEN CORE section of platform/mcp-server/src/index.ts ŌĆö changes there require a full process restart (not hot-reloadable)
- The createHttpServer function is called once on first load and reused across hot reloads via globalThis
- Bun's `new Request()` constructor does NOT auto-set a Host header ŌĆö tests that go through the fetch handler must explicitly include `Host: localhost:PORT` in headers
- The fetch handler in http-routes.ts runs Host validation ŌåÆ CORS check ŌåÆ route matching, in that order
- network-capture.ts registers Chrome API listeners (`chrome.debugger.onEvent`, `chrome.tabs.onRemoved`) at module load ŌĆö tests must mock `chrome` on globalThis before importing
- The `import-x/exports-last` ESLint rule requires all exports at the end of the file ŌĆö use `export { name }` re-export syntax for functions defined earlier in the file
- `scrubHeaders` does case-insensitive matching via `k.toLowerCase()` ŌĆö SENSITIVE_HEADERS entries must be lowercase strings

- `injectLogRelay` runs BEFORE adapter IIFE injection; the ordering is: ISOLATED listener ŌåÆ MAIN nonce injection ŌåÆ adapter IIFE file
- Plugin IIFE wrappers are generated by the published `@opentabs-dev/plugin-tools` npm package, not the local source ŌĆö changes to `build.ts` only affect plugins rebuilt with the local version. Extension-side patches (like `postMessage` wrapping) provide backwards compatibility with old IIFEs.
- The e2e-test plugin must be rebuilt with the LOCAL plugin-tools (`bun /path/to/platform/plugin-tools/dist/cli.js build`) to pick up features not yet published to npm (e.g., `sdkVersion` in tools.json)

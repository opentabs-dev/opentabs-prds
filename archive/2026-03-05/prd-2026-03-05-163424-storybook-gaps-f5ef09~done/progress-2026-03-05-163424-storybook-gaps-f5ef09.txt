## Codebase Patterns
- Storybook version is 10 (^10.2.13) — addon versions must use ^10.x to match
- The storybook:build script is `storybook build -o storybook-static` (run inside platform/browser-extension/)
- Storybook main.ts uses StorybookConfig type from @storybook/react-vite and has a viteFinal customization to replace the auto-added React plugin with one configured for React Compiler
- The addons array in main.ts accepts addon package names as strings
- npm install from the repo root handles workspace package installs; the pre-commit hook auto-stages dirty package-lock.json when sibling package.json is committed
- In Storybook 10, `@storybook/test` does not have a v10 release — use `storybook/test` import path instead (already available via the `storybook` package)
- Play function imports: `expect`, `userEvent`, `within`, `screen`, `fn` all come from `storybook/test`
- Radix Select renders options in a portal — use `screen` (not `within(canvasElement)`) to query portal content
- Radix Dialog renders content in a portal — use `screen.findByRole('dialog')` to find it after trigger click
- Radix Switch uses `data-state="checked"/"unchecked"` attributes for state verification
- Theme CSS variables are on `.dark` selector (not `html.dark`) — `.dark` class on any ancestor div scopes dark theme
- For side-by-side theme comparisons, use inline CSS variable styles (lightVars/darkVars from storybook-helpers.tsx) to guarantee isolation from the Storybook global theme toggle
- Portal-based Radix components (Dialog, DropdownMenu) render outside ThemeGrid containers — use document.documentElement.classList.add('dark') decorator for portal components

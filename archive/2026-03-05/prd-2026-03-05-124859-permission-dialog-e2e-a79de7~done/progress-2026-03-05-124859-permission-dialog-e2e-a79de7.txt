## Codebase Patterns
- Permission E2E tests use the shared `PermissionFixtures` fixture that starts the MCP server with `OPENTABS_DANGEROUSLY_SKIP_PERMISSIONS=''` to disable the bypass
- For concurrent 'ask' confirmations, use `Promise.all` with multiple `mcpClient.callTool` calls plus an async IIFE for dialog interaction
- Wait for badge text via `waitFor(async () => (await getBadgeText(sw)) === '<count>')` before asserting dialog state — this ensures all confirmations have arrived
- The ConfirmationDialog uses plain `<button>` elements (not retro Button) for prev/next with text 'prev'/'next' — use `getByRole('button', { name: 'prev' })` to locate them
- The counter display is `{safeIndex + 1} of {count}` and only shows when `count > 1`
- Plugin tool permission tests need a two-phase setup: first get plugin ready with 'auto' permission (open tab, wait for `e2e-test_get_status`), then switch to 'ask' via config write + hot reload
- The confirmation dialog shows base tool name for plugins (`echo`) but full prefixed name for browser tools (`browser_list_tabs`)
- After hot reload, ALWAYS wait for `waitForExtensionConnected` + `waitForLog(mcpServer, 'tab.syncAll received')` before calling tools — the WebSocket reconnection takes time and skipping it causes flaky timeouts

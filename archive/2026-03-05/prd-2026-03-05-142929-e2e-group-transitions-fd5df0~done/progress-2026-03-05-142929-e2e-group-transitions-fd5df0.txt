## Codebase Patterns
- E2E side panel tests use `button[aria-expanded]` to locate plugin card trigger buttons
- Plugin card opacity (ready vs not-ready) is on the Accordion.Item div (`div[data-state]`), not the trigger button — use XPath ancestor traversal from the trigger to get the Accordion.Item
- `waitForLog(server, 'tab.stateChanged: e2e-test → ready')` is the reliable way to wait for tab state changes
- **IMPORTANT**: Always wait for `waitForExtensionConnected` + `waitForLog('tab.syncAll received')` BEFORE opening a matching tab. Opening the tab before syncAll causes readiness to be reported in syncAll instead of as a separate `tab.stateChanged` event, making `waitForLog('tab.stateChanged')` timeout.
- Test setup pattern: `startMcpServer` + `startTestServer` + `launchExtensionContext` + `setupAdapterSymlink`, with cleanup in `finally` block
- The "NOT CONNECTED" label is rendered as plain text in the side panel, findable via `getByText('NOT CONNECTED')`

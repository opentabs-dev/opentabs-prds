## Codebase Patterns
- E2E tests in e2e/ use try/finally to guarantee cleanup of temp dirs created via fs.mkdtempSync
- page.close() goes inside the try block (before finally); fs.rmSync goes in the finally block
- Phase 1 checks: npm run build && npm run type-check && npm run lint && npm run knip && npm run test
- lint-staged runs prettier and eslint --fix on staged *.ts files; indentation changes may be auto-applied
- Fixed sleeps in E2E tests should be replaced by using the existing polling helpers (waitForExtensionConnected, waitForExtensionDisconnected, waitFor). When server A is dead, waitForExtensionDisconnected won't work (fetchHealth returns null). Instead, start server B immediately and rely on waitForExtensionConnected(serverB, 45_000) to provide the necessary wait.
- setServerPort calls chrome.storage.onChanged which triggers connect() immediately — no need to wait for extension reconnect exhaustion before calling it
- To fix race conditions between a live WebSocket connection and a buffer-empty assertion, navigate the tab away (browser_navigate_tab to any non-WebSocket page) before the verification read — this severs the connection and stops new frames from arriving

## Codebase Patterns
- `handleSetPort` in `platform/cli/src/commands/config.ts` uses `resolvePort(options)` to determine the server port; `resolvePort` caches the config port once per process via `readConfigPort()` in `parse-port.ts`
- `notifyServer` in `platform/cli/src/notify-server.ts` is a generic helper used by all `config set` handlers to notify the running server via POST /reload; it checks health first via the x-opentabs-version header
- The `DEFAULT_HOST` and `DEFAULT_PORT` constants come from `@opentabs-dev/shared`
- `ensureAuthSecret()` in `config.ts` is idempotent: creates `~/.opentabs/extension/` recursively and generates `auth.json` if missing. `readAuthSecret()` is read-only with no side effects.
- `resetConfigPortCache()` in `parse-port.ts` resets the per-process port cache; call it in beforeEach/afterEach when tests modify OPENTABS_CONFIG_DIR or the config file
- To mock `fetch` in Vitest tests: `vi.stubGlobal('fetch', vi.fn().mockResolvedValue(...))` and `vi.unstubAllGlobals()` in afterEach. The mock response needs `ok`, `headers.get()`, and `json()` fields to satisfy fetch helpers in config.ts
- `notifyServer` is globally mocked in config.test.ts via `vi.mock('../notify-server.js', ...)` — tests only need to stub `fetch` for the `fetchToolNames`/`fetchBrowserToolNames` calls

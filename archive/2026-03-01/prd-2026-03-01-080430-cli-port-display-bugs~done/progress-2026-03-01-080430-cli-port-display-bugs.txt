## Codebase Patterns
- Commander subcommands only receive parent options if the action handler calls `command.optsWithGlobals()` — the local `options` parameter only includes options defined on the subcommand itself
- Pattern for subcommands needing parent `--port`: `.action((_opts, command) => handleFn(command.optsWithGlobals()))`
- `resolvePort(options: { port?: number })` resolves port in priority order: --port CLI flag → OPENTABS_PORT env → config.json port → DEFAULT_PORT
- `config set` already uses `optsWithGlobals()` correctly; `config show` was the only outlier
- `handleSetPort` uses `oldPort` (from config.json) only for comparing with `newPort` — use `resolvePort(options)` for the running port in messages and notifications

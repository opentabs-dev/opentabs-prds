## Codebase Patterns
- Docs project: `docs/` subdirectory with its own node_modules (already installed by ralph). Run `cd docs && npm run build && npm run type-check && npm run lint && npm run knip && npm run format:check` for quality checks.
- MDX files have `lastUpdated` frontmatter in `YYYY-MM-DD` format â€” update to today's date when modifying.
- Quality check commands must be run from within `docs/` directory (qualityChecks field in PRD prefixes with `cd docs &&`).
- ConfigDirectory SVG in `docs/components/illustrations.tsx`: items are spaced 26px apart vertically. When adding new entries, extend viewBox height and both rect heights (shadow and body) by the same increment. The body rect `height` attribute appears twice (shadow and body rects) â€” both must be updated.
- The `port` field in config.json is a CLI-level setting (persisted by `opentabs config set port`), NOT an MCP server setting. The MCP server reads its port from --port flag or environment variables.

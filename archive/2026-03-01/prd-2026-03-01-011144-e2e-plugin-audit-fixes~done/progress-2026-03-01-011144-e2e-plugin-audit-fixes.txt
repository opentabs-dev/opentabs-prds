## Codebase Patterns
- `plugins/e2e-test/src/test-api.ts` exports `testApi(endpoint, body)` — all tools should use this for HTTP calls; it provides 30s AbortSignal timeout, structured network/timeout errors, and throws ToolError on `ok: false` responses
- `testApi` returns `T & { ok: true }` — to satisfy TypeScript when the output schema is `{ ok: boolean }`, use `return await testApi(...)` (since `true` extends `boolean`)
- The `/api/fail` endpoint returns `{ ok: false, error: errorCode, error_code: errorCode, error_message: errorMessage }` — testApi will extract `error` and `error_code` when throwing the ToolError

## Codebase Patterns
- E2E tests that check `.bg-success` for plugin icon state must scope the locator to the specific plugin's `button[aria-expanded]` element (e.g., `.filter({ hasText: 'E2E Test' })`), not `sidePanelPage.locator('.bg-success').first()` — BrowserToolsCard always renders a `.bg-success` dot since it uses `tabState="ready"`.
- Playwright locators are lazy and survive page reloads — a locator defined before `sidePanelPage.reload()` can be reused after the reload without re-querying.

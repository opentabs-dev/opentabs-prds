## Codebase Patterns
- Plugin manifest types live in `platform/shared/src/manifest.ts` (PluginOpentabsField, PluginPackageJson)
- Wire protocol types (WirePluginPayload) live in `platform/shared/src/index.ts`
- RegisteredPlugin type lives in `platform/mcp-server/src/state.ts` — must be updated alongside LoadedPlugin
- Extension PluginMeta type lives in `platform/browser-extension/src/extension-messages.ts`
- Validated payload types in `platform/browser-extension/src/message-router.ts` mirror WirePluginPayload
- Adapter files are managed in `platform/mcp-server/src/adapter-files.ts` with content-hashed filenames
- Extension uses `chrome.scripting.registerContentScripts()` for guaranteed document_start timing
- The `stripNodeBuiltins` esbuild plugin in build.ts marks `node:*` imports as external

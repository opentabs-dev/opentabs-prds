## Codebase Patterns
- Unit tests use `bun:test` (describe, test, expect) — import from `'bun:test'`
- Test files live alongside source files as `*.test.ts` in the same `src/` directory
- Tests are type-checked via `tsconfig.test.json` in each package (includes `src/**/*.test.ts`)
- Prettier enforces no-parens on single-param arrow functions without type annotations
- `noUncheckedIndexedAccess` is enabled globally — array `[index]` returns `T | undefined`. Avoid `!` (ESLint forbids it). Use `.at()` + type guard, destructuring, or `.map()` to extract fields for assertion (e.g., `messages(logs)` helper that maps entries to strings)
- The `plugins/e2e-test/` directory is gitignored for `dist/` — the build-e2e tests in `platform/plugin-tools/` require the plugin to be pre-built (`cd plugins/e2e-test && bun install && bun run build`)
- Follow existing test patterns: `platform/plugin-sdk/src/errors.test.ts` for factory method testing, `platform/shared/src/validation.test.ts` for validation logic testing
- ESLint import order: value imports from local modules first, then `bun:test`, then `type` imports last — no blank lines between import groups
- When testing Result<T, E> error paths, use a helper like `expectErr()` that calls `isErr()` as a type guard before accessing `.error` — direct `.error` access on the union type causes TS2339
- Mock `Bun.resolveSync` and `Bun.spawnSync` by saving originals in `beforeEach` and restoring in `afterEach`; call `resetGlobalPathsCache()` to clear cached paths between tests
- To test CLI commands that call `process.exit()`: save originals with `.bind()` (avoids unbound-method lint), replace with mock that throws, catch in try/catch, restore in finally
- When a source function uses `process.cwd()`, refactor to accept an optional `projectDir` parameter (defaulting to `process.cwd()`) rather than mocking `process.cwd()` globally
- E2E tests follow the standalone pattern from `e2e/config-watcher.e2e.ts`: mkdtempSync → writeTestConfig → startMcpServer → try/finally with server.kill() + cleanupTestConfigDir
- Use `createMinimalPlugin()` from `e2e/fixtures.ts` to create fully discoverable test plugins; for broken plugins, manually create directory structure omitting specific files
- Loader error messages contain identifiable substrings: "tools.json" for manifest issues, "Adapter IIFE" for missing adapter — use these in test assertions

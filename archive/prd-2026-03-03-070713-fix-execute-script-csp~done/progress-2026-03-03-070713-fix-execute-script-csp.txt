## Codebase Patterns
- The exec file wrapper in adapter-files.ts uses an inline inner function expression `(function() { <user_code> })()` to wrap user code, avoiding eval-like constructs (new Function, eval) that strict CSP blocks
- File-based injection via chrome.scripting.executeScript({ files }) runs as extension-origin code and bypasses page CSP entirely — the key is ensuring no eval-like constructs exist *inside* the injected file
- Unit tests for writeExecFile are in platform/mcp-server/src/adapter-files.test.ts and verify the wrapper structure, inline code placement, async support, and error capture

## Codebase Patterns
- CLI pure functions in platform/cli/src/commands/ are unexported by default — export them at the bottom of the file alongside `registerXxxCommand`
- Test files follow the pattern `platform/cli/src/commands/<name>.test.ts`, picked up by vitest via `platform/**/*.test.ts` in root vitest.config.ts
- The CLI package has `tsconfig.test.json` that includes `src/**/*.test.ts` — new test files are auto-covered
- Pre-commit hook runs prettier and eslint --fix automatically on staged files; no manual formatting needed before committing
- ESLint import order (groups): sibling (./) → internal (@*/**) → external (npm) → builtin (node:*); alphabetize within each group. Do NOT put node: imports before npm imports.
- `import type` statements go AFTER all regular value imports (see doctor.test.ts pattern) — type imports of npm packages come after node: builtin regular imports
- To mock node:child_process.spawn in Vitest: use `vi.mock('node:child_process', () => ({ spawn: vi.fn(), spawnSync: vi.fn() }))` at module level; create mock children with EventEmitter + setImmediate to emit data/close events
- For spawn-dependent async functions, create mock child as EventEmitter with .stdout/.stderr EventEmitter properties; emit 'data' on stdout then 'close' on child via setImmediate
- MockInstance type from vitest (import type { MockInstance } from 'vitest') is the correct type for vi.spyOn return value; ReturnType<typeof vi.spyOn> resolves to any due to overloads

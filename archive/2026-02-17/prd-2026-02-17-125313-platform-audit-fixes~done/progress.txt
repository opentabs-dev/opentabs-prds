## Codebase Patterns
- Platform packages are in `platform/`, plugins in `plugins/`
- `manifest.json` is at `platform/browser-extension/manifest.json` (extension root, not dist/)
- Build outputs go to `dist/` directories within each package
- Quality checks: build → type-check → lint → knip → test (all must pass)
- Test tsconfig pattern: extends base, `rootDir: "src"`, `noEmit: true`, `include: ["src/**/*.test.ts"]`, `references: [{ "path": "." }]`
- Use `() => Promise.resolve(...)` for async stubs to avoid `@typescript-eslint/require-await` lint errors
- Prettier enforces single-line imports when they fit — write them on one line
- `Array.toSorted()` requires ES2023 lib — use `.slice().sort()` instead in browser-extension code
- Prettier enforces single-line arrow functions when they fit — write them on one line to avoid formatting diffs
- `findAllMatchingTabs` returns all matching tabs sorted by rank; `findMatchingTab` wraps it returning just the first
- `testServer.setSlow(ms)` delays all API endpoints except `/api/auth.check` — isReady probes work instantly while tool fetches are blocked

## Codebase Patterns
- Playwright fixtures in e2e/fixtures.ts use `try { await use(resource); } finally { await resource.kill(); }` pattern for subprocess cleanup
- All quality checks: `bun run build && bun run type-check && bun run lint && bun run knip && bun run test && bun run test:e2e`
- The `hot-reload-dynamic.e2e.ts` "IIFE change triggers plugin.update log" test is known to be flaky (file watcher race condition) — it retries and passes
- Shared test helpers live in `e2e/helpers.ts` — getExtensionId, openSidePanel, setupAdapterSymlink, waitFor*, setupToolTest, etc.
- Shared test server utilities live in `e2e/test-server-utils.ts` — Invocation, jsonResponse, readBody, recordInvocation, requireAuth
- The linter (prettier via eslint) auto-formats short import lists to single-line; write imports in single-line format when 4 or fewer named imports
- ESLint requires value imports before type imports from the same module, and type-only imports as separate `import type {}` statements
- `setupAdapterSymlink` creates both `configDir/extension/adapters` symlink → `extensionDir/adapters` (creating extensionAdaptersDir if needed)
- `setupIsolatedIifeTest` in helpers.ts encapsulates the full standalone test setup and uses `setupAdapterSymlink` internally
- Both test servers use module-level mutable state — shared utilities accept state values as parameters to remain stateless
- `createMcpClient(port)` creates an independent MCP session — use multiple instances for tests that need separate session state or different timeouts
- `callTool` accepts `options?: { timeout?: number }` to override the default 30s fetch timeout

## Codebase Patterns
- For Node.js spawn with destructured command arrays, use `const [bin = '', ...args] = cmd` to avoid non-null assertion lint errors (the `@typescript-eslint/no-non-null-assertion` rule is enabled).
- `Readable.toWeb(nodeReadable) as ReadableStream<Uint8Array>` converts Node.js Readable streams to Web ReadableStreams (Node 22+ API).
- External packages (glob, minimatch) must be imported before node:* packages — that is the project's configured import order.
- The glob package v11 does NOT re-export minimatch — add minimatch as a separate devDependency when you need pattern matching.
- When modifying scripts that previously used Bun APIs, the bun-global.d.ts shim in the same directory must be deleted once all Bun APIs are removed from files in that directory.
- `npm run --prefix <dir> <script>` is the npm equivalent of `bun run --cwd <dir> <script>` for cross-directory script execution.
- Once tsx is referenced in package.json scripts directly, remove it from knip's ignoreDependencies — knip can find it automatically.
- `tsc --build --noEmit` (TypeScript 5.9+) can enter a broken state and report TS6310 errors across all packages if run multiple times after the tsbuildinfo files are stale. Fix: `npm run clean && npm run build` resets to a clean state. Always run `npm run build` before `npm run type-check` to ensure fresh tsbuildinfo files.
- New entry-point .ts files in platform packages (that are not imported by anything) must be added to knip.ts workspace entry list: `'platform/mcp-server': { entry: ['src/**/*.test.ts', 'src/dev-proxy.ts'] }`.
- `child_process.fork()` creates an IPC channel automatically. The child can call `process.send({ type: 'ready', port })` and the parent receives it via `child.on('message', ...)`.
- The `func-style` ESLint rule (set to 'expression') disallows function declarations — use `const fn = (): T => ...` everywhere, including in closures where hoisting might seem useful.

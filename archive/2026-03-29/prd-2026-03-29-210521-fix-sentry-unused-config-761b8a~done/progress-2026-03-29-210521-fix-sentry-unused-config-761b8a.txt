## Codebase Patterns
- Sentry plugin auth detection: `getAuth()` → `extractOrgSlug()` + `detectAuthentication()`. Both must succeed for the plugin to report as authenticated.
- API calls use same-origin relative paths (`/api/0${endpoint}`) which work for any hostname (sentry.io or self-hosted).
- The platform derives URL match patterns from `configSchema` `url`-type settings at runtime, so `urlPatterns` in `index.ts` only needs to cover the default SaaS case.
- Plugin dependencies may not be installed in worktrees — run `npm install` before `npm run build` if `node_modules` is missing.

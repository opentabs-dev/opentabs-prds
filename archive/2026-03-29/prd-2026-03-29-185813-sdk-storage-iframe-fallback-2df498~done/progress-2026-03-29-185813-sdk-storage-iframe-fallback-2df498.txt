## Codebase Patterns
- Storage utilities in `platform/plugin-sdk/src/storage.ts` use a shared `withIframeFallback` helper for all localStorage functions that need iframe fallback (getLocalStorage, findLocalStorageEntry, setLocalStorage, removeLocalStorage)
- The `getWindowLocalStorage()` helper accesses `window.localStorage` via property lookup to avoid ReferenceError when apps like Discord delete the property
- sessionStorage functions do NOT have iframe fallback — only localStorage functions need it
- Tests use `Object.defineProperty(globalThis, 'localStorage', { value: undefined })` to simulate deleted localStorage, and mock `document.createElement`/`body.appendChild`/`body.removeChild` for iframe simulation

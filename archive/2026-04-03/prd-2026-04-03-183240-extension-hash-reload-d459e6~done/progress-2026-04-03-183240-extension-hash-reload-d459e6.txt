## Codebase Patterns
- Root scripts are in `scripts/` and type-checked via `tsconfig.scripts.json` (includes `scripts/**/*.ts`)
- Build pipeline order in root `package.json`: tsc → catalog → bundle → side-panel → icons → hash → install → rebuild → make-executable
- Extension bundle files live in `platform/browser-extension/dist/`
- `.extension-hash` is a build artifact (gitignored) written to `platform/browser-extension/.extension-hash` and copied to `~/.opentabs/extension/` by `install-extension.ts`
- `EXTENSION_COPY_EXCLUDE_PATTERN` in `platform/shared/src/index.ts` controls which files are excluded from the install copy
- `sendSyncFull` in `extension-protocol.ts` must explicitly include every field from `buildConfigStatePayload` in the `params` object — fields are NOT automatically forwarded
- In E2E tests, the server reads `.extension-hash` from `configDir/extension/` (via `getExtensionDir()` + `OPENTABS_CONFIG_DIR` env var), while the side panel's `window.__EXTENSION_HASH__` comes from the built bundle in the extension copy
- `startMcpServer` pre-writes `.opentabs-version` to prevent `ensureExtensionInstalled` from copying files — write `.extension-hash` to `configDir/extension/` to control the server's hash value
- Brand-color icons (e.g., Slack) use fixed colors that don't change between light/dark themes — don't assert SVG content differs

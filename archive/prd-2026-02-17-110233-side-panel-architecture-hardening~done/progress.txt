## Codebase Patterns
- Side panel receives messages via two paths: (1) sp:serverMessage from background's forwardToSidePanel, and (2) ws:message broadcast from the offscreen document. The ws:message path is a fallback for when the side panel is opened as a regular extension page.
- `plugins.changed` is generated by the background (not the MCP server) after processing sync.full. It won't arrive via ws:message — only via sp:serverMessage.
- `fetchConfigState()` sends config.getState to the MCP server (not background storage), so the MCP server always has data ready when sync.full is sent.
- Platform packages are in platform/, plugins in plugins/. Extension source is in platform/browser-extension/src/.
- Pre-commit hooks run prettier, eslint, and knip. Never include .ralph/ files in commits.
- Bridge test mocking: mock `chrome.runtime.sendMessage` with a callback-based pattern on `globalThis.chrome`. The module-level `pendingRequests` Map persists across tests, so always clean up pending requests (resolve via `handleServerResponse` or `rejectAllPending` + `.catch()`).
- Bun's `expect(promise).rejects.toThrow()` returns `void` (not Promise), so `await expect(...).rejects.toThrow(...)` triggers ESLint `await-thenable`. Use a custom `expectRejection` helper with try/catch instead.
- `tsconfig.test.json` covers `src/**/*.test.ts` recursively, so test files in subdirectories like `side-panel/` are automatically type-checked.
- Side panel E2E tests: open via `chrome-extension://<id>/side-panel/side-panel.html`. Get extension ID from service worker URL pattern `chrome-extension://([^/]+)`. Poll for service worker with 10s timeout.
- Tool toggle verification: the `onToolConfigPersist` callback writes to config.json in the config dir — read the file to verify MCP server received the toggle. Also verify via `mcpClient.listTools()` that disabled tools disappear.
- ESLint `@typescript-eslint/require-await` fires on `expect.poll()` callbacks that use only sync code — use non-async arrow functions for synchronous poll callbacks.
- Import ordering: local imports (`./`, `../`) first, then external (`@opentabs-dev/*`, `react`, `node:*`). Always run `bun run lint:fix` after adding new imports to auto-fix ordering.

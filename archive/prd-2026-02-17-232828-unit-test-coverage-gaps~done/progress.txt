## Codebase Patterns
- Test files follow `describe`/`test` blocks with `bun:test` imports (`describe`, `expect`, `test`)
- Test files use `.js` extension in imports (e.g., `'./manifest-schema.js'`) matching ES Module resolution
- Existing test file at `manifest-schema.test.ts` already existed with basic tests — extend rather than replace
- Use `as unknown as Record<string, unknown>` for type-casting through `unknown` when TypeScript complains about insufficient overlap
- Run `bun run lint:fix` to auto-fix Prettier formatting issues before manual fixes
- `z.looseObject()` in Zod schemas allows extra fields to pass through validation
- To export functions from a file that uses a single named export statement, add the new names to that export list
- For picocolors-wrapped strings, test with `toContain(rawString)` rather than exact match (avoids ANSI code brittleness)
- `DOMException` is available globally in Bun — use `new DOMException(message, name)` for timeout tests
- ESLint `import-x/consistent-type-specifier-style` requires top-level `import type { X }` — not inline `import { type X }`
- ESLint `import-x/order` requires value imports from the same module grouped together, before `bun:test` — don't split type/value imports from the same module
- For Chrome API mocks in browser-extension tests, use `(globalThis as Record<string, unknown>).chrome = { ... }` pattern — avoid deeply nested Record casts
- Bun's `bun:test` supports Jest-compatible fake timer APIs: `jest.useFakeTimers()`, `jest.advanceTimersByTime(ms)`, `jest.useRealTimers()`
- `defineBrowserTool` returns the config object directly — access handler as `toolObj.handler(args, state)`
- For mocking `WsHandle`, provide both `send` and `close` methods even if only `send` is tested
- ESLint `no-non-null-assertion` forbids `!` — use `as string` (or appropriate type) instead
- `mock.module` from `bun:test` enables module-level mocking — must be awaited, and the module under test must be imported via `await import()` after the mock is set up
- **CRITICAL: `mock.module` is process-global** — it replaces modules for ALL test files in the same `bun test` run. Only mock modules that do NOT have their own test files to avoid contamination.
- For modules with their own test files that a tested module depends on, provide Chrome API stubs via `globalThis.chrome` so the real implementations work during async handler execution
- For testing error paths, use try/catch with `let caught: Error | undefined` to avoid ESLint `await-thenable` and `no-confusing-void-expression` issues with `expect().rejects.toThrow()`
- For functions that read from disk (like `loadPluginFromDir`), use real temp directories with `mkdtempSync` — no module mocking needed
- `scaffoldPlugin` resolves relative to `process.cwd()` — tests must `process.chdir()` and set `OPENTABS_CONFIG_DIR` to isolate
- `generatePluginIndex` uses `JSON.stringify()` producing double quotes — test URL patterns with `toContain('*://example.com/*')` not with quoted single quotes

- Discord plugin dependencies are NOT pre-installed by ralph (only `plugins/e2e-test` is built). Run `npm install` in `plugins/discord` before running quality checks.
- The discord plugin has its own `package-lock.json` in `plugins/discord/` (gitignored) — install is needed when running fresh in a worktree.
- AUTH_ERRORS, VALIDATION_ERRORS, NOT_FOUND_ERRORS sets in `plugins/discord/src/discord-api.ts` control error classification; body-level error codes are checked before HTTP status codes.

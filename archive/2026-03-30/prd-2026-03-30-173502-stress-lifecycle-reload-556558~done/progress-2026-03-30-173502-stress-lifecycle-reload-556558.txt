## Codebase Patterns
- Lifecycle hook tests use fixture-based `test` from `./fixtures.js` with `mcpServer`, `testServer`, `extensionContext`, `mcpClient` fixtures
- `setupToolTest` sets up adapter injection and waits for the tool to be callable
- The adapter records onNavigate URLs in `globalThis.__opentabs_onNavigate_urls` (array of URL strings)
- The adapter monkey-patches `history.pushState` to fire onNavigate synchronously — no debouncing, so rapid calls are recorded without drops
- Use `waitFor` with a predicate for polling conditions instead of fixed sleeps
- For tests needing [Disabled] prefix on tools, pass `{ OPENTABS_DANGEROUSLY_SKIP_PERMISSIONS: '' }` as env override to `startMcpServer`
- Count-based `waitFor` with log filtering is preferred for rapid sequential operations (e.g., waiting for N hot reload completions)
- Standalone tests needing extension + modifiable plugin: use `copyE2eTestPlugin` + `launchExtensionContext` + `setupAdapterSymlink` + manual auth.json symlink + `startTestServer`

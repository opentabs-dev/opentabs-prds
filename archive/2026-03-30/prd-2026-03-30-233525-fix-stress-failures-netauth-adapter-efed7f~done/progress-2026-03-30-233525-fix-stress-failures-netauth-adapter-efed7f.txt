## Codebase Patterns
- The `tab.syncAll` and `tab.stateChanged` log messages in `extension-handlers.ts` include profile labels (e.g., `[A]`). Over 150 E2E tests depend on the substring `"tab.syncAll received"` via `waitForLog()`. Any change to these log formats must preserve the substrings that E2E tests search for.
- The browser extension has a sliding window rate limiter (`rate-limiter.ts`) that limits `tool.dispatch` to 30 requests per 1s window. Tests firing rapid sequential tool calls must pace calls (~40ms delay) to avoid rate-limit errors.
- The `waitForLog()` helper in `e2e/helpers.ts` uses `String.includes()` for substring matching. Log format changes must preserve expected substrings.

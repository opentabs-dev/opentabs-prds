## Codebase Patterns
- Use `setupIsolatedIifeTest` for tests that need to modify the IIFE file ŌĆö it creates a per-test plugin copy, config dir, server, test server, extension, and MCP client
- Use `writeAndWaitForWatcher` to modify the IIFE and wait for the file watcher to process the change (waits for "IIFE updated for" log)
- Use `replaceIifeClosing` to inject code before the main IIFE closing `})();` ŌĆö it targets the boundary between the main IIFE and the freeze block
- `extension_check_adapter` returns `{ expectedHash, matchingTabs: [{ hashMatch, adapterPresent, ... }] }` ŌĆö use it to verify hash consistency after changes
- Adapter files in configDir/extension/adapters/ use content-hashed filenames like `e2e-test-a1b2c3d4.js`

## Codebase Patterns
- E2E concurrency tests use `test.slow()` to extend Playwright timeout for long-running dispatch tests
- `setupToolTest()` returns a page handle that must be closed at the end of the test
- `parseToolResult()` parses JSON text content from tool results into objects
- Concurrency limit is `MAX_CONCURRENT_DISPATCHES_PER_PLUGIN = 5` in `mcp-tool-dispatch.ts`
- Error format includes plugin name and limit: `Too many concurrent dispatches for plugin "e2e-test" (limit: 5)`
- `callTool` returns `{ isError, content }` — content is a string, not pre-parsed JSON
- Multi-connection dispatch routing: `tab.syncAll` populates `conn.tabMapping`, pass `tabId` to route to a specific connection
- Disconnect cleanup in `http-routes.ts` `makeWsClose`: rejects pending dispatches with `'Extension disconnected'` immediately
- Raw WS can simulate extension by handling `tool.dispatch` and responding with `{ jsonrpc: '2.0', id, result }`

## Codebase Patterns
- Cross-system tests that need both side panel AND MCP client use manual setup (not fixtures): `startMcpServer` with `OPENTABS_DANGEROUSLY_SKIP_PERMISSIONS: ''` env override + `createMcpClient` + `launchExtensionContext` + `setupAdapterSymlink` + `startTestServer`
- Plugin must be marked as reviewed (`reviewedVersion` in config permissions) to avoid unreviewed dialog blocking
- `getPluginVersion()` reads from `E2E_TEST_PLUGIN_DIR/package.json` to get version for `reviewedVersion`
- `selectPermission(page, ariaLabel, optionText)` clicks Radix Select trigger + picks option — the aria-label for plugin permission is `"Permission for e2e-test plugin"`
- `openTestAppTab` waits for adapter injection; `waitForToolResult` polls until tool is callable — together they ensure the tab is ready
- Use `Promise.allSettled` for chaos orchestration where some calls are expected to fail
- `collectPageErrors(page)` registers a `pageerror` listener and returns an array to check at the end
- POST /reload pattern: `fetch(\`http://localhost:${port}/reload\`, { method: 'POST', headers: { Authorization: \`Bearer ${secret}\` } })`
- After removing + re-adding a plugin via config + /reload, must open a new tab for adapter re-injection (existing tabs don't get re-injected automatically)
- `server.waitForHealth(h => h.plugins >= 1, 15_000)` polls until the plugin is rediscovered after /reload

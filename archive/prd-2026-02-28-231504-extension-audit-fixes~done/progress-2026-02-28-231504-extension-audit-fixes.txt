## Codebase Patterns
- BrowserToolsCard and PluginCard both use a `toggleCounter` ref + version matching to guard optimistic toggle revert — the catch handler only fires if its version equals the current counter
- Optimistic toggle pattern: `onToolsChange(prev => { ref.current = prev; return updatedState; })` — capture pre-toggle via `prev` inside the updater, not via a stale closure over the outer variable
- `updatePluginTools` in PluginCard wraps `setPlugins(prev => ...)` and passes `p.tools ?? []` as `prev` to the inner updater, so the inner updater's `prev` is the current plugin's tools array
- Vitest module mocks in `.test.ts` files: put `vi.mock('path')` AFTER all imports (Vitest hoists it automatically); imports must still be at the top in the correct group order (sibling → parent → external, no blank lines between groups per `newlines-between: 'never'`)
- When spying on `console.warn` in a test with type safety: declare `warnSpy` locally within the test (not at describe scope) so TypeScript infers the concrete return type of `vi.spyOn` rather than `any`; use `vi.restoreAllMocks()` in `afterEach` for cleanup

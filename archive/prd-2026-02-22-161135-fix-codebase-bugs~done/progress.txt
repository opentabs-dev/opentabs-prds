## Codebase Patterns
- `executeInTab` returns `inner?.value ?? null`, so results must be null-guarded before casting to typed arrays/objects
- `dispatchToExtension` returns `Promise<unknown>` — always validate the shape at runtime before using the result
- The `/ws-info` endpoint must return `wsSecret` when auth is configured; E2E lifecycle tests depend on this for raw WebSocket connections
- E2E tests in `lifecycle.e2e.ts` and `dispatch-resilience.e2e.ts` use raw WebSocket connections via `fetchWsInfo()` to get the URL and secret
- Some unit tests (build-e2e.test.ts) can be flaky when run as part of the full suite but pass in isolation — likely temp file timing issues
- All three dispatch handlers (tool, resource, prompt) share the same tab-loop pattern: iterate matching tabs, skip undefined IDs, try execution, fall through to `firstError` check. The `else` fallback after `if (firstError)` is essential to guarantee a response when all tabs have undefined IDs.
- PluginRegistry is Object.freeze'd but the freeze is shallow — Map entries (RegisteredPlugin objects) are not frozen. To preserve immutability, always create new plugin objects via spread and rebuild the registry via buildRegistry() rather than mutating plugin fields directly. See handlePendingPluginChange for the canonical pattern.
- `chrome.runtime.sendMessage` broadcasts to ALL onMessage listeners across all extension contexts (background, offscreen, side panel). Use distinct message types for each target context to prevent handlers in the wrong context from responding. The background script's switch statement has an exhaustiveness section for message types it doesn't handle.
- Module-level variables in `bun --hot` mode reset on every re-evaluation. To persist state across hot reloads, store on globalThis with a unique key (e.g., `__opentabs_<name>__`). Use getter/setter functions. For Maps, use lazy init in the getter to avoid replacing existing data on reload.
- For async polling, always use recursive setTimeout (not setInterval) to prevent overlapping calls. The `exports-last` ESLint rule requires private helpers to either be grouped before all exports or nested inside the consuming function.
- GLOBALS_SCAN_SCRIPT filter convention: single `_` prefix = private (filter), double `__` prefix = framework globals like `__NEXT_DATA__` (keep), `__zone` prefix = Zone.js internals (filter). The correct compound condition is `startsWith('__zone') || (startsWith('_') && !startsWith('__'))`.
- In Zod 4, `instanceof z.ZodOptional` / `z.ZodDefault` are unreliable. Use `schema.safeParse(undefined).success` to detect optionality — it covers optional, default, and union-with-undefined schemas.
- When using secondary lookup maps for circular buffer arrays, store direct object references (not array indices) — `shift()` invalidates all indices but object references remain stable.
- JSON-RPC error responses from the extension are `unknown` at runtime — always validate `typeof code === 'number'` and `typeof message === 'string'` before constructing `DispatchError`, providing defaults for malformed responses.
- `::ffff:127.0.0.1` (IPv4-mapped IPv6) must be in `ALLOWED_HOSTS` for DNS rebinding protection on dual-stack sockets.
- `Bun.file(path).size` is synchronous and efficient for pre-read size checks; use it to guard against OOM from unbounded file reads.
- Browser tool handler validation errors should use `throw new Error(...)` — mcp-setup.ts catches handler exceptions and formats them as MCP error responses. Returning error objects from handlers would double-wrap them.

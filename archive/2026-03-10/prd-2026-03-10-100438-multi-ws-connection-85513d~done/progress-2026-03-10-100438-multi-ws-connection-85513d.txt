  - Added `ensureConnectionId()` to background.ts: generates UUID in chrome.storage.local on first install
  - Called ensureConnectionId before ensureOffscreenDocument at top-level and in onInstalled/onStartup
  - Extended handleOffscreenGetUrl in background-message-handlers.ts to read and return connectionId from storage
  - Offscreen document stores connectionId in module variable during bootstrap, adds to WebSocket protocols
  - Fixed server-side handleWsUpgrade (http-routes.ts): only set _pendingConnectionId for explicitly-provided connectionIds, not fallback UUIDs — this was critical for backwards-compat eviction logic
  - Files changed: background.ts, background-message-handlers.ts, offscreen/index.ts, http-routes.ts
  - Phase 1 passes: build, type-check, lint, knip, test (91 files, 3152 tests)
  - Phase 2 passes: all 530 E2E tests pass (0 failures, 2 skipped)
  - **Learnings for future iterations:**
    - The server's handleWsUpgrade was always generating a fallback UUID and storing it in _pendingConnectionId, making hasExplicitId always true in createHandleWsOpen. This completely disabled the backwards-compat eviction path for single-connection clients. The fix: only store the parsed connectionId (undefined when not provided), letting the open handler generate the UUID fallback and correctly detect the backwards-compat case.
    - The offscreen document must have the connectionId BEFORE its first connect(). Since the offscreen starts connecting immediately on creation, ensureConnectionId() must complete before ensureOffscreenDocument() is called. The top-level code chains them: `ensureConnectionId().then(() => ensureOffscreenDocument())`.
    - The offscreen document reads connectionId from the background via the existing offscreen:getUrl message — no new message types needed. The response was simply extended from `{ url }` to `{ url, connectionId }`.
    - chrome.storage.local is isolated per browser profile (regular vs incognito), so connectionIds are naturally unique without any incognito detection logic.

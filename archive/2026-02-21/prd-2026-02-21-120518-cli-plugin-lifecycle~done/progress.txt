## Codebase Patterns
- The `updateCommand` field is defined in both `platform/shared/src/index.ts` (HealthResponse type) and `platform/mcp-server/src/state.ts` (OutdatedPlugin type) — both use plain strings
- Test fixtures referencing `updateCommand` exist in `reload.test.ts` and `extension-protocol.test.ts` — update them when format changes
- Prettier auto-formats object literals that exceed line length — account for this when making string values longer
- Commander subcommand actions receive `(arg, options, command)` — use `command.optsWithGlobals()` to access global flags like `--port`
- `normalizePluginName` and `notifyServer` are reusable helpers in `plugin.ts` for install/remove commands
- `Bun.spawn` with `stdio: ['inherit', 'inherit', 'inherit']` pipes npm output in real-time; `await proc.exited` gets exit code
- `localPlugins` in config.json stores filesystem paths (relative to config dir), not package names — to match by name, resolve the path and read its package.json
- Config helpers in `platform/cli/src/config.ts`: `atomicWriteConfig`, `getLocalPluginsFromConfig`, `resolvePluginPath` for safe config mutation and path resolution
- `colorTabState` is exported from `status.ts` and can be reused by other commands that display tab state
- Sibling imports (`./status.js`) must come before parent imports (`../config.js`) per eslint import-x/order rules — use `bun run lint:fix` if unsure
- `npm list -g --json --depth=0` returns JSON with a `dependencies` object keyed by package name — useful for offline npm plugin scanning

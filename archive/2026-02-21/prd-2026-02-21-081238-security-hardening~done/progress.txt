## Codebase Patterns
- Import order in platform packages: local (./), then external (@modelcontextprotocol, etc.), then node: builtins, then type imports. ESLint import-x/order enforces this.
- `checkBearerAuth` is exported and tested directly in http-routes.test.ts — security-critical functions are unit-tested at the function level.
- The WebSocket secret is sent via `Sec-WebSocket-Protocol` header as a comma-separated protocol list, not via URL query params. The server loops through parts to find the matching secret.
- The CORS origin check in `createHandleFetch` runs before all route handlers (including /ws), so it covers WebSocket upgrades automatically — no separate origin check needed for the WS path.
- Bun's WS upgrade convention: `bunServer.upgrade()` returns `true` on success and the handler returns `undefined` (not a Response).

## Codebase Patterns
- The CLI start command has two output paths: `printFirstTimeInstructions` (extension first installed) and the `else` block (subsequent runs). Both show MCP config hints.
- `printMcpClientConfigs` is a shared helper that both paths call with the MCP URL to keep config output consistent.
- Cursor uses VS Code's `mcp.json` format with `{ "type": "http", "url": "..." }` (not `streamable-http`).
- Windsurf uses `~/.codeium/windsurf/mcp_config.json` with `{ "serverUrl": "..." }` instead of `type` + `url`.
- Console output lines must stay under 120 chars (prettier printWidth). Long `console.log(pc.dim(...))` calls that exceed this need to be restructured.
- Config commands follow the pattern: get config path via `getConfigPath()`, check file with `Bun.file(path).exists()`, delete with `Bun.file(path).delete()`.
- Destructive CLI commands use a `--confirm` flag pattern instead of interactive prompts.
- `handleConfigSet` routes on key prefixes: `tool.*`, `port`, `localPlugins.add`, `localPlugins.remove`. Each has its own handler function.
- User-provided paths are resolved to absolute via `resolve()` from `node:path` before storing in config.
- `readConfig` returns a discriminated `ConfigResult` union (`missing | invalid | success`), not null. Callers destructure `{ config }` or check `result.error` to distinguish failure modes.

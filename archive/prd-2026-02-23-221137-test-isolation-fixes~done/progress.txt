## Codebase Patterns
- E2E global-setup.ts runs once before workers and uses Node.js fs APIs (readFileSync, writeFileSync, existsSync) — not Bun APIs — since Playwright runs under Node.js.
- ESLint ignores list needs to exclude non-TS files (e.g., .mjs) in .storybook directories since tseslintConfigs.strictTypeChecked applies to all files by default.
- EXTENSION_COPY_EXCLUDE_PATTERN is the canonical source for cpSync filter patterns. E2E fixtures cannot import it (shared uses Bun.env) so they define the regex inline with a comment referencing the canonical source. Use `path.relative()` for comparison, matching the pattern in extension-install.ts and setup.ts.

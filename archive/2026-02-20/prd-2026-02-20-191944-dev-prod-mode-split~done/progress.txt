## Codebase Patterns
- Dev mode detection lives in `platform/mcp-server/src/dev-mode.ts` — exports `isDev()` as a function returning a boolean determined once at startup
- The `--dev` CLI flag and `OPENTABS_DEV` env var control dev mode; production is the default
- `package.json` has separate scripts: `dev` (bun --hot + --dev) and `start` (production, no hot reload)
- Module-level state in the MCP server (like dev mode flag) is evaluated once per process; under `bun --hot` the module re-evaluates so the value is re-read each time
- In `reloadCore`, `stopFileWatching` runs unconditional (hot reload cleanup), but `startFileWatching`/`startConfigWatching` are gated on `isDev()`
- The `devMode` const in `dev-mode.ts` is read at module evaluation time — unit tests that run without `OPENTABS_DEV` set will see production mode; dev-mode file watcher behavior is tested via E2E tests with `--dev`
- HTTP route guards: `isDev()` is imported in `http-routes.ts` and checked early in the handler — returns 405 before auth/rate-limit checks for production-mode-only endpoints
- Plugin format: metadata in `package.json` opentabs field, tool schemas in `dist/tools.json`, adapter bundle in `dist/adapter.iife.js` — NO standalone manifest file exists
- docs-v2 build: clean `.next/` if stale webpack cache causes module errors; `bun run build` includes type-checking via Next.js

- Docs utilities.mdx follows a consistent pattern: ### heading, TypeScript signature code block, description paragraph, usage example code block
- The import block at the top of utilities.mdx groups utilities by category (DOM, Fetch, Storage, Page State, Timing, Logging)
- The Fetch section has a summary Table component listing all fetch utilities before the detailed sections
- New utilities should be added near related existing utilities (e.g., getMetaContent near getTextContent, auth cache near getCookie)
- SDK source lives in platform/plugin-sdk/src/ with files organized by category (dom.ts, fetch.ts, storage.ts, page-state.ts, timing.ts)
- Docs dependencies are standalone (not part of root npm workspaces) — run `npm install` in docs/ if node_modules is missing

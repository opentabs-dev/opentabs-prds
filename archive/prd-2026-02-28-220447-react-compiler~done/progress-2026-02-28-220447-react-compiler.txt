## Codebase Patterns
- The esbuild side panel build is in `platform/browser-extension/build-side-panel.ts`. Build scripts are type-checked via `tsconfig.build-scripts.json` which includes `build-*.ts` files.
- Knip config is in `knip.ts` at the repo root. Babel presets/plugins referenced as strings in config objects must be listed in `ignoreDependencies` since knip can't trace string-based references. However, dynamic `await import(...)` calls in `.storybook/main.ts` ARE traced by knip (the file is in the entry list via tsconfig.storybook.json), so those packages should NOT be in ignoreDependencies.
- `storybook-static/` is Storybook build output. If it exists on disk, knip flags all its JS files as "unused files" (non-zero exit). It must be in both `.gitignore` and knip's `ignore` list (`**/storybook-static/**`). Knip may show a "configuration hint" suggesting the ignore can be removed — ignore this hint when the directory is present.
- Declaration files (`.d.ts`) for untyped npm packages should be placed in the consuming package directory and added to both the relevant tsconfig and knip entry list.
- The browser extension workspace (`platform/browser-extension`) has separate tsconfigs for: main source (`tsconfig.json`), build scripts (`tsconfig.build-scripts.json`), tests (`tsconfig.test.json`), and storybook (`tsconfig.storybook.json`).
- `@storybook/react-vite` auto-adds `@vitejs/plugin-react` without babel options. To use React Compiler in Storybook, filter out the auto-added plugins by name (`vite:react-babel`, `vite:react-refresh`, `vite:react-virtual-preamble`) in `viteFinal` and replace with a configured instance.
- React Compiler evidence in Vite/Rollup output: the compiler runtime is inlined as `.c(N)` calls (where N is memo slot count); the import path `react/compiler-runtime` disappears after bundling. Use `grep -o "[a-zA-Z_$]\.c([0-9]\+)"` to verify compiler ran.
- When removing `useCallback` from functions used as `useEffect` dependencies, `react-hooks/exhaustive-deps` will warn that the function "changes on every render". These are false positives — React Compiler automatically memoizes the functions. The rule emits `warning` (not `error`), so `npm run lint` exits 0.

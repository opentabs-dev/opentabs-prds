## Codebase Patterns
- CLI binary for create-plugin is at `platform/create-plugin/dist/index.js` â€” run via `bun <path> <args>`
- CLI binary for opentabs commands is at `platform/cli/dist/cli.js` â€” run via `bun <path> <subcommand> <args>`
- Use `Bun.spawnSync` with `OPENTABS_CONFIG_DIR` env var for isolated CLI tests
- The `scaffoldPlugin()` function uses `JSON.stringify()` for template generation, so generated code uses double quotes (not single quotes) for string literals
- The `tsconfig.test.json` pattern: extends `../../tsconfig.base.json`, sets `noEmit: true`, includes `src/**/*.test.ts`, references the production tsconfig
- Pre-commit hooks run prettier and eslint --fix on staged files, so formatting is auto-corrected
- `@opentabs-dev/*` packages are private npm packages â€” tests that need `bun install` for scaffolded plugins must override `package.json` dependencies to use `file:` paths pointing to local `platform/` packages, plus `overrides` for transitive deps
- Bun's `overrides` field in `package.json` resolves transitive dependency versions to specified values (similar to npm overrides)
- Plugin paths in config.json are stored relative to the config dir â€” use `path.relative(configDir, absolutePath)` to compute expected values in tests
- For `plugin list --json` tests, create a minimal plugin dir with `opentabs-plugin.json` manifest to get full plugin metadata in the output
- For `opentabs build` E2E tests, copy the e2e-test plugin to a temp dir with `cpSync` (exclude node_modules) + `symlinkSync` (link original node_modules) â€” this avoids copying huge node_modules while keeping dependency resolution working

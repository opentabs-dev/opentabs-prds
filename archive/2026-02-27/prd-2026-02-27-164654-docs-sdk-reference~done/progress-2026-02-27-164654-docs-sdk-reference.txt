## Codebase Patterns
- The docs site is a standalone Next.js app in `docs/` with its own `package.json` and node_modules — run `npm install` inside `docs/` before running checks if dependencies aren't installed.
- Quality checks run from inside the `docs/` directory: `npm run build && npm run type-check && npm run lint && npm run knip && npm run format:check`
- MDX content lives in `docs/content/docs/` — tables use standard markdown pipe syntax and format correctly through Prettier's check.

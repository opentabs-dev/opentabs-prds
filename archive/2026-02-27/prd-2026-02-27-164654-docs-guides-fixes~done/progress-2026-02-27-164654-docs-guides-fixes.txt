## Codebase Patterns
- The docs site is a standalone Next.js project in `docs/` — it does NOT share node_modules with the root monorepo. Run `npm install` in `docs/` first if node_modules are missing.
- postJSON/putJSON/patchJSON/deleteJSON signature: `(url, body, init?, schema?)` — schema is always the LAST arg, after optional `init`
- fetchJSON signature: `(url, init?, schema?)` — no body parameter
- Quality checks command: `cd docs && npm run build && npm run type-check && npm run lint && npm run knip && npm run format:check`

## Codebase Patterns
- Dev proxy tests should use `startMcpServer(configDir, true)` for hot mode and `createTestConfigDir()` for isolated config
- The proxy's `whenReady()` buffers requests in `pending[]` when `workerPort === null` and drains on IPC 'ready' message
- `triggerHotReload()` sends SIGUSR1 to the proxy process; use `waitForLog(server, 'Hot reload complete')` to confirm completion
- Non-null assertions are forbidden by ESLint — use `if (!x) throw new Error(...)` pattern instead
- The `McpServer` type has `health()`, `waitForHealth()`, `triggerHotReload()`, `kill()`, `port`, `secret`, `configDir`, `logs`
- Always clean up with `server.kill()` and `cleanupTestConfigDir(configDir)` in `finally` blocks
- Server secret is read from `<configDir>/extension/auth.json` — pass as `Authorization: Bearer <secret>` header
- Use `pgrep -P <proxyPid>` + SIGKILL to kill the worker directly for 503 timeout tests
- For graceful shutdown tests, send SIGTERM via `process.kill(pid, 'SIGTERM')` directly (not via fixture's `kill()`), listen on `proc.once('exit')` to capture code/signal, then call `server.kill().catch(() => {})` defensively in finally
- Use `process.kill(pid, 0)` to check if a process is still alive (throws if dead)
- ESLint import-x/order: local imports (./fixtures, ./helpers) must come before node: builtins
- Use AbortController (not boolean flags) to control async poll loops — ESLint flags `let x = true; while (x)` as always-truthy
- Use `createMcpClient(server.port, server.secret)` + `client.initialize()` + `client.listTools()` to verify MCP tool registration in non-fixture tests
- The MCP client auto-reinits sessions after worker restarts (handles 400 "missing session", 502, 503 transparently)
- Non-hot mode: `startMcpServer(configDir, false)` spawns `dist/index.js --dev` directly; use POST /reload for config rediscovery
- `createMinimalPlugin(parentDir, pluginName, tools)` creates discoverable plugins; tool prefix is `pluginName_` (not the npm package name)
- POST /reload returns `{ ok: true, plugins: N, durationMs: N }` — assert plugin count to verify discovery
- Use `waitForToolList(client, predicate)` to poll tools/list after reload (handles session re-init timing)

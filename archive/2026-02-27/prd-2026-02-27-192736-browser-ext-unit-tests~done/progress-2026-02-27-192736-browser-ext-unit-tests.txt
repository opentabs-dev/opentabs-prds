## Codebase Patterns
- Browser extension test files use `.js` extension in imports (ES modules) even though sources are `.ts`
- `tsconfig.test.json` includes `src/**/*.test.ts`; root `vitest.config.ts` discovers all `platform/**/*.test.ts`
- `tab-matching.test.ts` is the canonical example for browser extension test patterns: mock chrome globals via `globalThis.chrome = {...}` in `beforeEach`, no vi.mock needed for pure functions
- `offscreen/index.ts` has heavy side effects (chrome API calls at module load time); pure utility functions were extracted to `offscreen/ws-utils.ts` to enable unit testing without mocking the entire chrome API
- TypeScript 5.9 broke `tsc --build --noEmit` with composite project references (TS6310). Fix: change `type-check` script to `tsc --build` (which already type-checks and is incremental). The CLI `--noEmit` flag now cascades to all referenced composite projects in TypeScript 5.9, conflicting with composite project requirements.
- Import order rule: `./` (same-directory) imports come BEFORE `../` (parent-directory) imports in eslint-import-x/order
- `plugin-storage.ts` has an in-memory cache (`metaCache`). Tests using real plugin-storage must call `invalidatePluginCache()` in `beforeEach` to reset state between tests.
- Dispatch test pattern (tool/resource/prompt): mock `messaging.js` + `sanitize-error.js`, provide Chrome API stubs (`scripting.executeScript`, `tabs.query`, `tabs.get`, `storage.local.get/set`), import module under test. See `tool-dispatch.test.ts` and `resource-prompt-dispatch.test.ts` for examples.
- Stateful module testing pattern (confirmation-badge, rate-limiter): set up chrome mocks BEFORE the `await import(...)` at module top level, use the module's own reset function (`clearAllConfirmationBadges`, `resetRateLimiter`) in `beforeEach`, use `vi.useFakeTimers()` / `vi.useRealTimers()` for timeout control.
- Background-message-handlers module state reset: `wsConnected` is module-level state; reset by calling `handleWsState({ connected: false }, () => {})` in `beforeEach`, then `vi.clearAllMocks()` to wipe side-effect calls from the reset. Use `vi.hoisted` + `vi.mock` for all imported dependencies (messaging, tab-state, confirmation-badge, message-router, tool-dispatch).

## Codebase Patterns
- `tsconfig.scripts.json` covers `scripts/**/*.ts` — new scripts are automatically type-checked
- `tsconfig.base.json` includes `"types": ["bun-types"]` — Bun APIs available in all packages
- Root `package.json` scripts referenced by knip automatically — no need to add script entries to knip config
- Use `Bun.spawn()` for child processes in scripts (not `node:child_process`)
- For `{ write(data: string): boolean }` as a type for output streams instead of `typeof process.stdout` (which is too narrow for stderr)
- `bun run format` fixes prettier issues; always run it before lint checks
- ESLint enforces `void` operator on fire-and-forget promises (`@typescript-eslint/no-floating-promises`)
- Use `node:fs` `watch()` with `{ recursive: true }` for watching dist directories — matches the pattern in `file-watcher.ts`
- The `build:side-panel` script in browser-extension already chains `build:tailwind` — no need to run Tailwind separately
- MCP server `/extension/reload` endpoint requires Bearer auth with wsSecret from `~/.opentabs/config.json`
- The extension pipeline runs: build:bundle → build:side-panel (includes Tailwind) → install-extension → POST /extension/reload

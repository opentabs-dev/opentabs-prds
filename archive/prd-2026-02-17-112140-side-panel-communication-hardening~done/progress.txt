## Codebase Patterns
- Side panel receives server notifications via two paths: `sp:serverMessage` (forwarded by background) and `ws:message` (broadcast by offscreen document). Only `sync.full` needs the `ws:message` fallback; all other notifications arrive via `sp:serverMessage`.
- `SIDE_PANEL_METHODS` in `message-router.ts` controls which notification methods the background forwards to the side panel via `sp:serverMessage`.
- `loadPlugins` in App.tsx calls `fetchConfigState()` which sends `config.getState` to the MCP server.
- `loadPlugins` is called directly (no ref indirection) since it has stable identity from `useCallback([], [])`.
- Optimistic UI updates in PluginCard capture pre-update state snapshots for rollback — never reconstruct from inverse values.

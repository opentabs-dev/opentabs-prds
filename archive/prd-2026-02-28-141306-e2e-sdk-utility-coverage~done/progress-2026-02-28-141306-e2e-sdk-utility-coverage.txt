## Codebase Patterns
- e2e/test-server.ts: All API endpoints are in a single `handler` function using if-chain routing. Add new endpoints by inserting a new if-block before the 404 handler.
- The server's CORS preflight only allows GET, POST, OPTIONS. Since adapter code runs as injected same-origin scripts, PUT/PATCH/DELETE calls are same-origin and don't trigger CORS preflight.
- `sendJson(res, data)` sends JSON with Content-Type: application/json and status 200 (with CORS headers by default).
- The sdk_http_methods tool (plugins/e2e-test) calls putJSON/patchJSON/deleteJSON against /api/echo-method but constructs its own return value — the endpoint response is independent from the tool output.

## Codebase Patterns
- Config retry: readConfigWithRetry in config.ts handles transient file unavailability with exponential backoff (3 retries, 100ms/200ms/400ms)
- Config parsing: parseConfigRecord extracts and validates typed fields from raw JSON records
- Atomic writes: atomicWriteConfig writes to .tmp then renames — use this pattern for all config writes
- Config write serialization: saveConfig and saveToolConfig use state.configWriteMutex to prevent concurrent read-modify-write races
- reloadCore catch block: preserves previous state on any error (config load or discovery), ensuring plugins never disappear on transient failures
- File watcher callbacks: FileWatcherCallbacks interface in file-watcher.ts — add new callbacks here when extending watcher behavior (e.g., onPluginDiscovered)
- Pending plugin watching: startFileWatching accepts resolvedPluginPaths to watch paths that failed initial discovery; uses loadPluginFromDir for reattempt
- statSync for sync dir checks: file-watcher.ts uses statSync from node:fs for directory existence checks in synchronous context (startFileWatching is sync)
- CLI atomic writes: platform/cli/src/config.ts exports `atomicWriteConfig` for all CLI commands that write config.json

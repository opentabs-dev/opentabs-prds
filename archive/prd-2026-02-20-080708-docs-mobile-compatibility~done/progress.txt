- **Learnings for future iterations:**
  - Next.js 16 uses a separate `export const viewport: Viewport` for viewport meta configuration (not part of `metadata`)
  - The `Viewport` type is imported from `'next'` (not `'next/types'`)
  - ESLint `import-x/order` requires value imports before type-only imports from the same package scope — `next/font/google` must come before `type { Viewport } from 'next'`

  - `env(safe-area-inset-*, fallback)` is the standard CSS function for safe area insets; `max()` combines it with a minimum padding to ensure non-notched devices still get the base padding
  - ID selectors (`#nd-global-header`) override Tailwind utility classes (`.px-4`) due to higher CSS specificity — the inline `px-4` on the header element is effectively overridden by the CSS rule

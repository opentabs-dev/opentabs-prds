## 2026-02-18 16:42 - US-007
- Changed Ajv configuration from `allErrors: true` to `allErrors: false` — stops validation on first error to limit combinatorial work from pathological schemas (e.g., deeply nested oneOf/anyOf from community plugins)
- Wrapped the synchronous `lookup.validate(args)` call in try-catch to gracefully handle unexpected runtime errors (e.g., catastrophic regex backtracking from a plugin's pattern keyword)
- Added comments explaining the security rationale for both changes
- Files changed: `platform/mcp-server/src/mcp-setup.ts`
- **Learnings for future iterations:**
  - Ajv validation is synchronous — true timeout protection requires worker threads, which is over-engineering for this use case
  - `allErrors: false` is the single most effective knob for limiting validation work — Ajv stops after the first failing subschema instead of evaluating all branches
  - Schemas come from plugin manifests validated at discovery time, so the risk is low — these guards are defense-in-depth for community plugins
  - `compileToolValidator` already wraps `ajv.compile()` in try-catch, but the `validate()` call was unprotected until now

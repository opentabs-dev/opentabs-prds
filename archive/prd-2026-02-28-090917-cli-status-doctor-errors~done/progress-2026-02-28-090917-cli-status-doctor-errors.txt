## Codebase Patterns
- `platform/cli/src/commands/status.ts` exports pure utility functions (formatUptime, colorTabState, isTimeout, isNonOpenTabsHttpError) for unit testing — keep new helpers exported
- `platform/cli/src/commands/doctor.ts` exports helper functions (isCwdProjectDirectory, defaultMcpClientLocations) for unit testing — follow same pattern
- The MCP server health endpoint returns JSON with a `status: 'ok'` field; its absence signals a non-OpenTabs server
- `npm run build` at root requires `lightningcss-linux-x64-gnu` native binary (on glibc Linux). If missing: `npm install lightningcss-linux-x64-gnu --no-save`
- Phase 1 checks: `npm run build && npm run type-check && npm run lint && npm run knip && npm run test`
- `npm run type-check` is just `tsc --build` (no extension CSS build)
- Use `vi.spyOn(process, 'cwd').mockReturnValue(dir)` + `afterEach(() => vi.restoreAllMocks())` to test CWD-dependent logic

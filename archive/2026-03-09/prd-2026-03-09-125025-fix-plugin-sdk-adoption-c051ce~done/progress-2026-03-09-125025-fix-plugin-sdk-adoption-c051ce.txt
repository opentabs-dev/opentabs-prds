## Codebase Patterns
- Plugins in `plugins/` are standalone — they need `npm install` in their own directory before building
- SDK utilities are exported from `@opentabs-dev/plugin-sdk` barrel — `findLocalStorageEntry`, `getPageGlobal`, `fetchFromPage`, etc.
- `findLocalStorageEntry(predicate)` returns `{ key: string; value: string } | null` — the value is the raw string, caller must JSON.parse if needed
- Root monorepo quality checks (build, type-check, lint, knip, test) don't cover plugins — plugin builds are separate

## Codebase Patterns
- Docs site is in `docs/` with its own `package.json` and quality checks (build, type-check, lint, knip, format:check)
- MDX content files are in `docs/content/docs/` organized by section (sdk/, guides/, etc.)
- SDK utility docs in `utilities.mdx` follow a consistent pattern: summary table at top, then individual `###` sections with signature, description, options table (if applicable), and code examples
- Plugin development guide in `plugin-development.mdx` covers SDK utilities with practical before/after comparisons
- The docs `npm run build` requires `npm install` first if dependencies aren't installed (e.g., in fresh worktrees where only platform packages were built)
- When working in the `docs/` subdirectory, use absolute paths for `git` commands since the working directory may be `docs/` but git root is the repo root

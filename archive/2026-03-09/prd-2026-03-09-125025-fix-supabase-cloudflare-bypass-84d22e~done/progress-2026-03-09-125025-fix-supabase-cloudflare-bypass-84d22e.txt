## Codebase Patterns
- All Supabase tools import `api` from `../supabase-api.js` and use `api<T>(endpoint)` for API calls
- The `api()` function in supabase-api.ts handles: auth via getAuth(), error handling (401/403/429/404/timeout), query params, JSON parsing
- The base URL for Supabase API is `https://api.supabase.com/v1` — endpoints are relative paths like `/projects/${ref}/types/typescript`
- The Supabase generate-types endpoint returns JSON `{ types: string }`, not raw text (despite what the PRD notes suggested)
- Cloudflare's `cloudflareApi()` returns the standard CF envelope `{ success, errors, messages, result }` — the GraphQL endpoint (`/api/v4/graphql`) returns `{ data, errors }` instead, so it can't use `cloudflareApi()` directly
- For Cloudflare tools that can't use cloudflareApi() due to different response formats, use SDK's `postJSON` with `getAtokHeader()` for auth
- Cloudflare auth uses `x-atok` header (anti-forgery token from `window.bootstrap.atok`) + same-origin cookies; the SDK's `credentials: 'include'` works for same-origin requests

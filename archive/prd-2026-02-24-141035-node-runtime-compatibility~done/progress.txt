## Codebase Patterns
- Runtime detection: `const isBun = typeof Bun !== 'undefined'` — use this pattern everywhere
- The tsconfig for platform/shared includes all `src/` files, so new files there are automatically type-checked
- ESLint requires imports to be in alphabetical order (local before node:*), and type imports alphabetical within their group
- nodeSpawnSync with `stdio: ['ignore', 'pipe', 'pipe']` guarantees non-null stdout/stderr — no optional chaining needed
- Bun.CryptoHasher.digest('hex') returns string directly, no `as string` cast needed
- cross-platform.ts `platformExec` now includes 'node' for Windows resolution alongside npm/npx/bun/bunx
- Bun.resolveSync → require.resolve with createRequire(import.meta.url) — works in both Node.js and Bun
- Tests that mock Bun.resolveSync should be replaced with real node_modules structures + process.cwd mock
- macOS /var → /private/var symlink: use realpathSync when comparing paths in tests
- BunServer interface is now ServerAdapter — exported from http-routes.ts for use by server-node.ts and index.ts
- Node.js server adapter (server-node.ts) uses ws library 'headers' event for custom upgrade response headers
- Avoid async callbacks in Node.js event handlers — wrap in void-returning function to satisfy @typescript-eslint/no-misused-promises
- bun-types' RequestInit lacks `duplex` — cast init as Record<string, unknown> to set it for Node.js
- Readable.toWeb() returns ReadableStream<any>, not ReadableStream<Uint8Array> — use double cast `as unknown as ReadableStream<Uint8Array>`
- spawnStreaming for long-running processes (streaming stdout/stderr), spawnInherit for pass-through stdio (npm install)
- getFileSize and readFileSlice for file metadata and partial reads (log tailing, audit size checks)
- esbuild IIFE bundling for browser: use external: ['node:*'] to handle node builtins from transitive deps behind runtime detection
- esbuild sourcemap: 'linked' is equivalent to Bun.build's sourcemap: 'external'
- esbuild outfile + allowOverwrite: true overwrites input files in-place (useful for post-tsc bundling)
- esbuild needs .mjs in resolveExtensions for packages with bare .mjs subpath files (e.g., lucide-react/dynamic)
- When migrating a package from Bun to runtime layer, grep for `Bun\.` in non-test files to find ALL call sites — don't rely on the PRD line numbers alone
- resolver.ts bun global path discovery only runs when isBun is true — Node.js users only scan npm global paths

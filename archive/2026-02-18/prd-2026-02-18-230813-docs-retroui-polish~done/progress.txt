## Codebase Patterns
- Docs site is at `docs/` with Astro + Starlight (standalone subproject with its own `package.json`)
- Component overrides live in `docs/src/overrides/` and are registered in `docs/astro.config.mjs` under `components:`
- Override pattern: import default Starlight component, wrap in a `<div class="retro-*">`, style children via `:global()` selectors in `<style>` block
- Global CSS lives at `docs/src/styles/global.css` with CSS layers: base, starlight, theme, components, utilities
- RetroUI theme tokens are CSS custom properties (--background, --foreground, --primary, --border, etc.) with light/dark variants
- Shadow scale: --shadow-xs (1px) through --shadow-2xl (16px), all hard offset (no blur)
- Starlight custom elements: `<starlight-theme-select>`, `<site-search>`, `<starlight-menu-button>`, `<starlight-toc>`, `<sl-sidebar-state-persist>`
- Sidebar structure: `nav.sidebar > .sidebar-pane > .sidebar-content > sl-sidebar-state-persist > ul.top-level > li > details > summary` for group headers
- Sidebar group labels use `.group-label > .large` span inside `<summary>`
- Active sidebar link uses `a[aria-current="page"]` attribute
- `color-mix(in srgb, var(--accent) 20%, transparent)` for semi-transparent accent backgrounds
- Search button uses `<site-search>` custom element (NOT `<starlight-search>`), with `button[data-open-modal]` for the trigger
- Search dialog is a native `<dialog>` inside `<site-search>`, Pagefind UI mounts into `#starlight__search`
- Starlight scoped styles are in `@layer starlight.core` — global CSS without a layer declaration wins, but `!important` is needed for properties set by scoped component styles
- The docs site is NOT part of the root bun workspaces — quality checks (build, type-check, lint, knip, test, test:e2e) only cover `platform/`

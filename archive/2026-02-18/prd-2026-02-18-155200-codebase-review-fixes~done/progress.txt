## Codebase Patterns
- The offscreen document (`platform/browser-extension/src/offscreen/index.ts`) manages the persistent WebSocket connection to the MCP server with reconnection, ping/pong keepalive, and guard flags to prevent concurrent operations.
- Guard flags like `connecting` must always be reset in `finally` blocks to prevent deadlocks from unexpected exceptions.
- In `handleExtensionMessage`, `parsed.method` and `parsed.id` are validated with `typeof` checks (not `as` casts) â€” follow this pattern for any new fields extracted from parsed JSON.

- Slack plugin is standalone (not in bun workspaces) â€” always verify with `cd plugins/slack && bun run build`.
- When refactoring ternary to if/else, beware `Uint8Array` vs `Uint8Array<ArrayBuffer>` â€” the narrower generic is needed for `fetch()` body.
- In `startCapture` (network-capture.ts), the debugger attach + enable sequence needs cleanup: if `sendCommand` fails after `attach` succeeds, detach the debugger before rethrowing to avoid orphaned debugger attachments.
- The `pluginLocks` map in `tab-state.ts` stores the `.catch()` return value (not the raw promise) â€” rethrowing in the catch handler would break the serialization chain for subsequent calls.
- Slack plugin shared schemas (`messageSchema`, `paginationMetadataSchema`, `channelSchema`) live in `channel-schema.ts` â€” new shared schemas should be added there.

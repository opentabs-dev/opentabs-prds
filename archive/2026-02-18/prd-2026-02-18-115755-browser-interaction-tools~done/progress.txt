## Codebase Patterns
- Browser tools follow a 5-file pattern: (1) tool definition in `platform/mcp-server/src/browser-tools/<name>.ts`, (2) export from `browser-tools/index.ts`, (3) handler in `browser-commands.ts`, (4) dispatch entry in `message-router.ts`, (5) method in `ALLOWED_METHODS` in `offscreen/index.ts`
- When adding a new browser command handler, also add its mock to `message-router.test.ts` — both the mock variable definition AND the mock module export map. Bun resolves module mocks against all named exports; missing exports cause `SyntaxError: Export named '...' not found`.
- Tool naming: `browser_<action>` (snake_case) for tool names, `browser.<camelCase>` for method names, `handleBrowser<PascalCase>` for handlers, `<kebab-case>.ts` for files
- Handler pattern: validate params with runtime type checks, use `chrome.scripting.executeScript` with `world: 'MAIN'`, return JSON-RPC response via `sendToServer()`
- 3 E2E tests (tool-dispatch.e2e.ts:729, tool-dispatch.e2e.ts:465, dispatch-resilience.e2e.ts:33) are pre-existing failures unrelated to browser interaction tools

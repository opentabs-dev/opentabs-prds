## Codebase Patterns
- `McpServerInstance` is exported as a type from `mcp-setup.ts` — mock it by capturing handlers in a `Map<unknown, handler>` via `setRequestHandler`, then invoke them directly
- For async throwing tests in Bun, use `expect(async () => await fn()).toThrow(/pattern/)` — the outer test function must NOT be `async` to avoid the `@typescript-eslint/require-await` lint rule
- Sync throw pattern: `expect(() => fn()).toThrow(/pattern/)` — see `manifest-schema.test.ts` for examples
- Test files use `mkdtempSync` + `afterEach`/`afterAll` cleanup for temp directory fixtures
- `writeFileSync` is used (not Bun APIs) in test files for temp file creation since these are one-off setup calls
- When `const x = await asyncFn()` is before a `try` block, if the await rejects, the `finally` is never entered — the error propagates directly without entering the try/finally
- Use optional chaining (`result?.id`) instead of `!` non-null assertions in tests — `@typescript-eslint/no-non-null-assertion` lint rule forbids them
- To use fake timers in Bun tests, import `jest` from `bun:test` and call `jest.useFakeTimers()` / `jest.useRealTimers()` / `jest.advanceTimersByTime(ms)`
- To test async promise chains + setTimeout: flush microtasks with `await Promise.resolve()` (repeat for each .then/.catch in the chain), then advance fake timers
- Access chrome mock objects via `(globalThis as Record<string, unknown>).chrome as { ... }` cast pattern — avoids eslint-disable comments
- `Response.json()` returns `Promise<any>` — define typed interfaces for response shapes and use `as Promise<T>` to avoid `@typescript-eslint/no-unsafe-*` lint errors
- Shared types: `TrustTier` is `'official' | 'community' | 'local'`; `ManifestTool` uses `input_schema`/`output_schema` (snake_case, not camelCase)

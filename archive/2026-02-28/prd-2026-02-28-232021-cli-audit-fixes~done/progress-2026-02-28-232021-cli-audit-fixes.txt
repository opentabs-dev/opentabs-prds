## Codebase Patterns
- `followFile` in `platform/cli/src/commands/logs.ts` uses a `statSync` size check before opening the stream, but the stream may read beyond that size if the file is written to concurrently. Track `actualBytesRead` by summing `chunk.length` in the `'data'` handler and use `startOffset + actualBytesRead` on `'end'`.
- Tests for `followFile` mock `statSync`, `createReadStream`, and `watch` from `node:fs`. Streams are created as `EventEmitter` instances with events emitted manually. The watcher callback is retrieved via `mockWatch.mock.calls[0]`.

- **Learnings for future iterations:**
  - The root cause: `createReadStream` without an `end` option reads to actual EOF at the time of reading, not to the captured `statSync` size. Concurrent writes extend EOF beyond the pre-read size.
  - `Buffer.byteLength(str)` is needed for string chunks (when `encoding` is set on the stream); `chunk.length` suffices for `Buffer` chunks since `.length` gives byte count for Buffers.

## Codebase Patterns
- E2E lifecycle hook tests use `setupToolTest` for page setup, `waitFor` for async polling (10s timeout, 200ms interval), and `expect.arrayContaining([expect.stringContaining(...)])` for URL assertions
- When testing popstate/history events: pushState fires onNavigate synchronously (via monkey-patch), while history.back() fires popstate asynchronously â€” clear the URL array after pushState before checking popstate effects
- Prettier enforces multi-line destructured args in test functions when line exceeds 120 chars â€” format like the fixture-consuming tests (mcpServer, testServer, extensionContext, mcpClient each on its own line)
- Browser extension rate limits tool.dispatch to 30 requests/second (in `platform/browser-extension/src/rate-limiter.ts`). For tests needing many log entries, use a single tool call emitting bulk entries rather than many tool calls
- Per-plugin concurrent dispatch limit is 5 (`MAX_CONCURRENT_DISPATCHES_PER_PLUGIN` in `platform/mcp-server/src/mcp-tool-dispatch.ts`). Batch sizes in tests must stay under this limit
- Log buffer constant is `MAX_ENTRIES_PER_PLUGIN = 1000` in `platform/mcp-server/src/log-buffer.ts` (ring buffer with modulo arithmetic)

## Codebase Patterns
- The docs site is a standalone Next.js project in `docs/` — run `npm install` inside `docs/` before quality checks since it uses its own node_modules (NOT part of root npm workspaces)
- Quality checks run from within `docs/` subdirectory: `cd docs && npm run build && npm run type-check && npm run lint && npm run knip && npm run format:check`
- MDX content files are in `docs/content/docs/`

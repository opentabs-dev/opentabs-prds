## Codebase Patterns
- confirmation-badge.ts uses a `confirmationTimeouts` Map<string, ReturnType<typeof setTimeout>> to track pending confirmation background timeouts keyed by confirmation id
- Chrome notification IDs use the pattern `opentabs-confirm-${id}` (prefixed); the confirmation id itself is the key in confirmationTimeouts
- `clearConfirmationBadge(id?: string)` accepts an optional id; when provided it clears the Chrome notification via `chrome.notifications.clear('opentabs-confirm-<id>')`. `clearConfirmationBackgroundTimeout(id)` is the separate function that cancels the background timeout.
- background-message-handlers.ts extracts id as a const before using it in both clearConfirmationBackgroundTimeout and clearConfirmationBadge to avoid duplicating the typeof check
- The test file uses `vi.useFakeTimers()` in `beforeEach` and resets state via `clearAllConfirmationBadges()` + `mockClear()` on all mocks

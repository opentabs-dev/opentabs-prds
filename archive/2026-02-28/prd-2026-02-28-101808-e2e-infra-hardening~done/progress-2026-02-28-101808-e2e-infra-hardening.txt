## Codebase Patterns
- `e2e/fixtures.ts` is a standalone E2E helper file (not compiled to `dist/`); it's type-checked via `tsconfig.test.json` but not part of `npm run build`'s output
- `lightningcss-linux-x64-gnu` is an optional dep of `lightningcss` that may be missing from the worktree's `node_modules`; running `npm rebuild lightningcss` or `npm install lightningcss-linux-x64-gnu` (without saving to package.json) fixes `build:tailwind` failures
- The `killProcess` function in `e2e/fixtures.ts` sends SIGTERM → (graceMs) → SIGKILL; it now also has an absolute timeout (graceMs + 10s) to prevent zombie process hangs
- `prefer-const` ESLint rule: avoid `let x; x = setTimeout(...)` — use `const x = setTimeout(...)` or restructure with a `done` flag to avoid circular references in closures
- `replaceIifeClosing` in `e2e/helpers.ts`: the adapter IIFE now has a freezer closure appended after the main `})();` — the regex must match `})();(function(){` not just `})();[\s]*$`
- `callToolWithProgress` in `e2e/fixtures.ts`: uses its own `fetch()` call — any retry/reinit logic added to `request()` must also be added here via a `doFetch(retried)` helper

## Codebase Patterns
- The e2e-test plugin's URL match pattern is `http://localhost/*` (defined in `plugins/e2e-test/src/index.ts:76`)
- To get a non-matching URL from the test server, access it via `127.0.0.1` instead of `localhost` — the plugin pattern only matches `localhost`
- The test server (`e2e/test-server.ts`) serves pages at `/`, `/interactive`, `/sdk-test`, `/post-test`, `/ws-test`, and now `/non-matching`
- `TestServer` fixture (in `e2e/fixtures.ts`) exposes `url` (e.g., `http://localhost:PORT`), `port`, `control()`, and `controlGet()` methods
- For non-matching URLs in tests, two approaches: `about:blank` for simple navigation tests, or `testServer.url.replace('localhost', '127.0.0.1') + '/non-matching'` for tests needing real page content (title, cookies, etc.)

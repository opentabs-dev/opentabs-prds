## Codebase Patterns
- E2E test helpers: `initAndListTools()`, `openTestServerTab()`, `waitForNetworkRequests()`, `waitForWebSocketFrames()` are defined at top of `e2e/browser-tools.e2e.ts`
- `parseToolResult(result.content)` parses the MCP content as JSON; tool return values come back as serialized JSON objects
- For HAR export: `JSON.parse(parseToolResult(result.content).har as string)` to get the HAR object
- `browser_export_har` requires active network capture (tracked in `state.activeNetworkCaptures: Set<number>`)
- `enable-network-capture.ts` adds tabId to `state.activeNetworkCaptures`; `disable-network-capture.ts` removes it
- Test server WebSocket endpoint is at `testServer.url + '/ws-test'` (defined in `e2e/test-server.ts`)
- Browser tool handlers throwing errors → caught by mcp-tool-dispatch.ts → returns `{ isError: true, content: [...] }`
- Hot reload: new `ServerState` fields are auto-patched from `createState()` defaults at `platform/mcp-server/src/index.ts:126-130`
- Permission system: `evaluatePermission()` in `platform/mcp-server/src/permissions.ts` checks skipConfirmation, domain policies, tool tiers, and trusted domains
- To test without skipConfirmation: pass `{ OPENTABS_SKIP_CONFIRMATION: '' }` as envOverrides (4th arg) to `startMcpServer()`
- Default trustedDomains are `['localhost', '127.0.0.1']` — use `127.0.0.2` as a non-trusted loopback alias
- `callToolWithProgress()` on McpClient captures SSE progress notifications — use it to verify the 'approval' notification from the confirmation flow
- ESLint import-x/order: groups are `index, sibling, parent, internal, external, builtin, object, type` — sibling imports (`./foo.js`) must come before `@playwright/test` (internal via pathGroup)
- Confirmation timeout is 30s (CONFIRMATION_TIMEOUT_MS in state.ts) — tools blocked on confirmation return `CONFIRMATION_TIMEOUT` error after that
- Side panel confirmation dialog: `openSidePanel(extensionContext)` opens the side panel page; dialog has `role="alert"`, "Allow Once" button, "Allow Always" dropdown, "Deny" button
- To test confirmation flow: use `Promise.all([mcpClient.callTool(...), clickAllowOnce(sidePanel)])` — tool call blocks while side panel interaction happens concurrently
- `browser_get_cookies` takes `{ url: string }` — good for testing sensitive-tier confirmation without needing an open tab on the target domain
- "Deny" button: `getByRole('button', { name: 'Deny' })` — returns `PERMISSION_DENIED` error
- "Allow Always" dropdown: `getByRole('button', { name: 'Allow Always' })` to open, then `getByRole('menuitem', { name: 'For this tool on this domain' })` to select scope
- `resolveToolDomain()` extracts domain from `url` arg first, then `tabId` — `browser_open_tab` with non-trusted URL triggers confirmation because it has a `url` param
- `browserToolPolicy` in config.json: `{ browser_screenshot_tab: false }` disables a tool from both tools/list and dispatch; `isBrowserToolEnabled()` checks `state.browserToolPolicy[toolName] !== false`
- `OpentabsConfig` in fixtures.ts only has `localPlugins`/`tools` — extend with `browserToolPolicy` by spreading and casting
- `getByText()` matches substrings by default — use `{ exact: true }` when text may appear inside JSON args or other nested content

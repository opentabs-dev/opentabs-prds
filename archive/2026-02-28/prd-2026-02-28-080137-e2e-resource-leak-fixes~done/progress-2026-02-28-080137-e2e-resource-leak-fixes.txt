## Codebase Patterns
- `McpServer` type is exported from `e2e/fixtures.ts` and should be imported with `import type { McpServer } from './fixtures.js'`
- `BrowserContext` type is NOT re-exported from fixtures; import it directly with `import type { BrowserContext } from '@playwright/test'`
- In E2E test files, import order: relative value imports → node: built-ins → type-only imports (relative then external)
- `killProcess` in fixtures.ts checks `proc.exitCode !== null` before killing, so calling `server?.kill()` on an already-dead server is safe
- Resource leak fix pattern: declare `let server: McpServer | undefined` and `let context: BrowserContext | undefined` before try, assign inside try, use `?.` optional chaining in finally
- For module-level shared test servers (`beforeAll`/`afterAll`), use `let server: TestServer | undefined` and `${server?.url ?? ''}` in template literals — `!` non-null assertion and `?.url` in templates are both banned by strict ESLint rules (`no-non-null-assertion`, `restrict-template-expressions` with `allowNullish: false`)

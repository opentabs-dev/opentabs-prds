## Codebase Patterns
- The docs site is a standalone Next.js project in `docs/` — run `npm install` inside `docs/` before running checks (not installed by root `npm ci`)
- Quality checks run from inside `docs/`: `cd docs && npm run build && npm run type-check && npm run lint && npm run knip && npm run format:check`
- Tailwind CSS variables: `bg-secondary` = dark background (`#000` light / `#3a3a3a` dark), `text-secondary-foreground` = light text (`#fff` light / `#f5f5f5` dark), `text-accent-foreground` = dark text (`#000` in both modes)
- To add a new Tailwind color utility: (1) add `--var-name: value` to `:root` and `.dark` in `global.css`, (2) add `--color-tailwind-name: var(--var-name)` in the `@theme inline` block, (3) use `text-tailwind-name` or `bg-tailwind-name` in components
- `--code-fg: #f8f8f2` is the Dracula theme foreground — used as `text-code-fg` in CodeBlock to ensure light text on dark `bg-code-bg`

## Codebase Patterns
- Server-only E2E tests (no browser needed) use `startMcpServer(configDir, true)` + `createTestConfigDir`/`cleanupTestConfigDir` in beforeEach/afterEach
- `server.secret` holds the auth token; `server.port` holds the ephemeral port
- `server.waitForHealth(h => h.status === 'ok')` waits for server readiness
- POST /reload and GET /audit both require `Authorization: Bearer <secret>` header; missing/wrong token returns 401
- Always call `server.kill()` in a `finally` block to clean up
- Plugin log entries format in server stdout: `[plugin:<name>] <ts> <LEVEL> <message> <JSON.stringify(data)>` where `data` is the serialized args array (e.g., `[{"level":"debug"}]`)
- The `log_levels` e2e-test tool already passes structured args `{ level: '<levelname>' }` with each call — no plugin changes needed to test args serialization
- `mcpServer.waitForHealth(h => buf >= bufferBefore + 4)` is the reliable way to wait for log entries to arrive (polls /health buffer directly vs. stdout which can be buffered)

## Codebase Patterns
- `generateSecret()` is exported from `@opentabs-dev/shared` (platform/shared/src/constants.ts) — use it everywhere auth secrets are generated
- `atomicWrite(path, content, mode)` is exported from `@opentabs-dev/shared` for writing files with restrictive permissions (e.g., 0o600 for auth files)
- `getExtensionDir()` from `@opentabs-dev/shared` returns `~/.opentabs/extension` (or OPENTABS_CONFIG_DIR override)
- The CLI's `readAuthSecret()` in `platform/cli/src/config.ts` reads auth.json but returns null if missing; `ensureAuthSecret()` generates+writes it if missing
- MCP server's `loadSecret()` in `platform/mcp-server/src/config.ts` already has "read existing or generate new" logic; `ensureAuthSecret()` mirrors this pattern in the CLI
- Tests use `OPENTABS_CONFIG_DIR` env override to isolate from real `~/.opentabs`

## Codebase Patterns
- Storybook stories follow a consistent convention: Meta with `title: 'Components/<Name>'`, `w-80` decorator, named exports, `type Story = StoryObj<typeof Component>`
- Existing stories use bare module imports without `.js` extensions (e.g., `import { X } from './X'`)
- ESLint enforces `func-style` rule: use arrow function expressions (`const Foo = () => {}`) not function declarations
- The e2e-test plugin at `plugins/e2e-test` must be built before unit tests can pass (no `dist/` in git)
- The `/ws-info` endpoint must return both `wsUrl` and `wsSecret` — E2E tests use `fetchWsInfo()` to get WebSocket credentials for raw WS connections
- Use `fn()` from `storybook/test` for callback props in stories (Storybook 10 pattern, replaces `@storybook/addon-actions`)

## Codebase Patterns
- All SVG illustrations live in `docs/components/illustrations.tsx` and are registered in `docs/components/MDX.tsx`'s `docComponents` object for use in `.mdx` files
- SVG marker IDs must be prefixed per-component (e.g. `arch-arrow-right`, `qs-arrow`) to avoid collisions when multiple illustrations render on the same page
- The docs project (`docs/`) has its own `bun install` and quality checks separate from the root monorepo
- The root pre-commit hook runs root-level knip which has pre-existing failures on platform packages — docs-targeted PRDs use `--no-verify` since the docs-specific quality checks are what matter
- Homepage page.tsx has an outer wrapper div with shadow/border effects around the ArchitectureIllustration — the component itself just renders the SVG

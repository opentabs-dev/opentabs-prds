## Codebase Patterns
- The CLI commands in `platform/cli/src/commands/` follow a pattern: each file exports a `register*Command` function and individual check/helper functions for testability
- The `readConfig` function returns `{ config, error }` ‚Äî always check for null config before accessing fields
- Status command reads config secret and sends auth header; doctor command should do the same
- `Bun.serve({ port: 0 })` returns a server with an auto-assigned port; the `port` property type is `number | undefined` so use `as number` when you know it's assigned
- E2E tests for plugin discovery expect non-existent local plugin paths to appear in `failedPlugins` ‚Äî don't silently skip them in discovery.ts
- The e2e-test plugin in `plugins/e2e-test/` must be built before running `platform/plugin-tools/src/commands/build-e2e.test.ts` tests
- `Bun.spawnSync` is the correct way to run subprocesses synchronously in Bun (e.g., running `tsc` from the build pipeline)
- The `build-e2e.test.ts` tests use `copyPlugin()` which copies the e2e-test plugin to a temp dir and symlinks node_modules ‚Äî tests that modify dist/ or src/ should use this pattern

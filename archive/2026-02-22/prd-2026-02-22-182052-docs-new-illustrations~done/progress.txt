## Codebase Patterns
- All illustration components live in `docs/components/illustrations.tsx` and must be registered in `docs/components/MDX.tsx` to be usable in `.mdx` files
- Illustrations follow neo-brutalist style: CSS variables for theming, `var(--font-mono)` for text, 3px strokeWidth, hard drop shadows (offset rect 4px right/down), box-with-header pattern, no border-radius
- Highlighted items use `fill="var(--color-primary)" opacity="0.12"` with `strokeWidth="1.5"`
- Arrow markers need unique IDs per illustration to avoid conflicts (e.g., `pf-arrow`, `qs-arrow`, `df-arrow`)
- The pre-commit hook runs root-level `bun run knip` even for docs-only changes — root knip must pass
- Lint-staged in the pre-commit hook can sweep unstaged but modified files into the commit; stage only the files you intend to commit
- `@opentabs-dev/*` workspace packages need `ignoreDependencies` in knip.ts because knip cannot trace their imports

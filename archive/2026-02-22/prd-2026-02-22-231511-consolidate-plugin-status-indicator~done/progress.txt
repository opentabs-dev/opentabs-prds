## Codebase Patterns
- ConfigStatePlugin and ConfigStateResult are in platform/shared/src/index.ts (the shared types package)
- The MCP server's internal state (state.outdatedPlugins) is decoupled from the wire format (ConfigStateResult) — server can store data differently than it sends it
- OutdatedPlugin.name matches plugin's npmPackageName (the npm package name, e.g. 'opentabs-plugin-slack')
- Extension protocol tests use a makePlugin helper that returns RegisteredPlugin; add npmPackageName to test update-related behavior
- PluginState is a type alias for ConfigStatePlugin in bridge.ts — changes to the shared type automatically flow through
- COPIED_INDICATOR_DURATION_MS was only used by OutdatedPluginsBadge; safe to remove when deleting that component
- E2E test plugins in plugins/e2e-test need their own `bun install` in worktrees (node_modules is gitignored)
- The /ws-info endpoint returns {wsUrl, wsSecret} when authenticated — wsSecret is needed by the extension for Sec-WebSocket-Protocol

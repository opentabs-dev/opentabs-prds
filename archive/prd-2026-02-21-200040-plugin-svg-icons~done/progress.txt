## Codebase Patterns
- Most icon SVG fields (iconSvg, iconInactiveSvg) were already added to types across the pipeline in a prior iteration — only LoadedPlugin in loader.ts and the removed icon field in PluginOpentabsField needed changes
- The icon data flows: tools.json manifest → loader.ts (LoadedPlugin) → registry.ts (RegisteredPlugin) → extension-protocol.ts (WirePluginPayload) → message-router.ts (ValidatedPluginPayload → PluginMeta)
- Optional wire fields use conditional spread pattern: `...(value ? { key: value } : {})`
- The `parsePluginPackageJson` validates the `opentabs` field in package.json — icon SVG fields are NOT in package.json, they are in dist/tools.json
- SVG icon validation and grayscale generation utilities are in `platform/plugin-tools/src/validate-icon.ts` — use regex-based parsing (no XML parser dependency), ITU-R BT.709 for luminance
- ESLint forbids non-null assertions — always use `?? fallback` for regex match groups instead of `!`
- Test files in plugin-tools live alongside source (e.g., `src/validate-icon.test.ts`), excluded from tsc output via `tsconfig.json` but type-checked via `tsconfig.test.json`
- Import order: parent/sibling → @scoped → external → builtin → type imports. External (picocolors, zod) comes BEFORE builtin (node:*).
- `Bun.write()` returns `Promise<number>`, not `Promise<void>` — wrap in a helper returning `Promise<void>` when used in tests to avoid `@typescript-eslint/await-thenable`
- `expect(promise).rejects.toThrow()` in bun:test is synchronous (returns void) — do NOT use `await` on it
- `PluginManifestOutput` in build.ts is the internal manifest shape — it now supports optional `iconSvg` and `iconInactiveSvg` fields
- `readAndValidateIcons(projectDir)` reads icon.svg/icon-inactive.svg from plugin root, validates, auto-generates inactive, and returns minified SVGs

## 2026-02-22 - US-002
- What was implemented:
  - Created `platform/plugin-tools/src/validate-icon.ts` with three exported functions:
    - `validateIconSvg(content, filename)` — validates SVG structure (size ≤ 8KB, square viewBox, no `<image>`/`<script>`, no event handlers)
    - `validateInactiveIconColors(content)` — validates that only achromatic colors are used (checks fill, stroke, stop-color, flood-color in attributes and inline styles)
    - `generateInactiveIcon(svgContent)` — converts all color values to luminance-equivalent grays using ITU-R BT.709 formula
  - Created `platform/plugin-tools/src/validate-icon.test.ts` with 75 test cases across three describe blocks
  - Includes comprehensive named color lookup table (CSS2.1 + extended names)
  - Handles hex (#RGB, #RRGGBB), rgb(), rgba(), hsl(), hsla(), and named CSS colors
  - Passthrough values (none, currentColor, transparent, inherit, url()) are preserved unchanged
  - For HSL/HSLA colors, sets saturation to 0% while preserving hue and lightness
  - For RGB/RGBA colors, computes luminance gray and outputs as hex (or rgba with alpha preserved)
- Files changed:
  - `platform/plugin-tools/src/validate-icon.ts` — new file, SVG validation and grayscale generation
  - `platform/plugin-tools/src/validate-icon.test.ts` — new file, 75 unit tests

## Codebase Patterns
- The homepage uses `HomeLayout` from fumadocs-ui with `baseOptions` from `lib/layout.shared.tsx`. Override `nav.component` at the call site to use a custom header on the homepage.
- The docs pages use `DocsLayout` with `GlobalHeader showSidebarTrigger` for the sidebar hamburger menu.
- `tw-animate-css` provides `animate-collapsible-down` and `animate-collapsible-up` for Radix Collapsible animations — use `data-[state=open]:animate-collapsible-down data-[state=closed]:animate-collapsible-up` on `Collapsible.Content`.
- The `react-hooks/set-state-in-effect` ESLint rule is active — avoid calling setState directly in useEffect. Use onClick handlers or refs instead.
- Desktop/mobile breakpoint for nav items is `lg` (1024px) on the homepage.
- Prettier sorts Tailwind responsive classes alphabetically within the same prefix — e.g., `sm:grid-cols-2` must come before `sm:gap-6`. Always run lint after editing class strings to catch ordering issues.
- Fumadocs search dialog (`SearchDialogContent`) uses `fixed left-1/2 top-4 md:top-[calc(50%-250px)] w-[calc(100%-1rem)] max-w-screen-sm` — no custom positioning needed. The search results scrollable div is `[data-empty] > div` with `overflow-y-auto`.
- Mobile CSS overrides live in `docs/app/global.css` inside `@media (max-width: 639px)` (for sm breakpoint) or `@media (max-width: 767px)` (for md breakpoint). Add new mobile rules to the appropriate existing media query block.

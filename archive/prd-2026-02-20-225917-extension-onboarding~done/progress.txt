## Codebase Patterns
- Side panel components use the `Empty` composite component from `retro/Empty.tsx` with subcomponents: `Empty.Content`, `Empty.Title`, `Empty.Separator`, `Empty.Description`
- Design system: Tailwind CSS 4 with CSS custom properties (`--primary`, `--muted`, `--foreground`, etc.) defined in `styles.css`
- Fonts: `font-head` (Archivo Black) for headings via `Text` component, `font-sans` (Space Grotesk) for body, `font-mono` (Space Mono) for code
- Flat shadows: `shadow-md` (4px 4px 0 0) with no blur — consistent RetroUI pattern
- Icons from `lucide-react` — CheckCircle2, Circle, Ghost, ChevronDown, etc.
- Import ordering is enforced by eslint `import-x/order` — keep alphabetical within groups
- Side panel state machine in App.tsx: loading → disconnected / onboarding (first-time) / returning-user empty / plugin list, driven by `connected` + `plugins.length` + `hasEverHadPlugins`
- E2E tests for side panel use `openSidePanel(context)` (from helpers.ts) to navigate to `chrome-extension://<id>/side-panel/side-panel.html`
- For 0-plugin side panel tests, create a config with `{ localPlugins: [], tools: {} }` and use non-fixture `startMcpServer`/`launchExtensionContext` for full control
- Fresh extension chrome.storage.local is empty → `hasEverHadPlugins === false` → shows OnboardingState ("Welcome to OpenTabs"), NOT ReturningUserEmptyState ("No Plugins Installed")
- After the onboarding UI changes (US-001/US-002), any existing test that checks `text=No Plugins` on a fresh 0-plugin extension is broken — must use `text=Welcome to OpenTabs` instead
- `hasEverHadPlugins` flag is persisted in `chrome.storage.local` and read asynchronously on mount (starts as `null` → shows LoadingState until resolved, preventing flicker)
- When `loadPlugins()` receives plugins with count > 0, it sets `hasEverHadPlugins = true` in both React state and chrome.storage.local
- The `Empty` component in EmptyStates.tsx was the old generic empty state — removed since OnboardingState replaces it for US-001
- `ReturningUserEmptyState` component shows "No Plugins Installed" heading with install instructions (no welcome text or checklist)

- Updated App.tsx state machine: `loading || hasEverHadPlugins === null` → LoadingState, `!connected` → DisconnectedState, `!hasContent && hasEverHadPlugins` → ReturningUserEmptyState, `!hasContent && !hasEverHadPlugins` → OnboardingState, else → PluginList
- Added `hasEverHadPlugins` to the `sp:getState` response for E2E testability
- Files changed: `App.tsx`, new `ReturningUserEmptyState.tsx`
- **Learnings for future iterations:**

  - When adding a new import between existing imports, ensure no blank line is introduced — the `import-x/order` rule treats blank lines as group separators
  - The tri-state `null | false | true` pattern is effective for async-initialized flags: `null` means "still loading", which maps cleanly to a loading UI state and prevents flicker

## Codebase Patterns
- The `import-x/exports-last` ESLint rule requires all export statements at the end of files. Define types/classes without `export` keyword, then re-export at the bottom (e.g., `export { MyClass }; export type { MyType };`).
- When adding new InternalMessage variants to types.ts, update the exhaustive switch in background.ts AND offscreen/index.ts — both have switch cases covering all message types.
- The `ignoreExportsUsedInFile: true` knip setting means exports used within their own file are not flagged as unused.
- When adding new extension_* browser tools: update BROWSER_TOOL_NAMES in e2e/helpers.ts and ensure E2E test filters use `!t.name.startsWith('extension_')` not `t.name !== 'extension_reload'`.
- known-methods.ts DISPATCH_METHODS is the single source of truth for WebSocket method allowlist — offscreen ALLOWED_METHODS derives from it.
- bgLogCollector was extracted from background.ts into bg-log-state.ts to break circular import: background.ts → message-router.ts → browser-commands.ts → background.ts. Both background.ts and browser-commands.ts import from bg-log-state.ts.
- When `chrome.runtime.sendMessage` returns `any`, cast to `unknown` first (`const raw: unknown = await ...`) then assert the expected shape. This avoids `@typescript-eslint/no-unsafe-*` lint errors.
- React 19 lint rule `react-hooks/refs` prohibits `ref.current = value` during render. Sync refs from state using a separate `useEffect` with no dependency array.
- E2E tests for extension debugging tools: force reconnect happens too fast for /health to see disconnection — verify via tab.syncAll log count instead.
- The e2e-test plugin adapter has no __hash property, so adapterHash returns null in check_adapter results.

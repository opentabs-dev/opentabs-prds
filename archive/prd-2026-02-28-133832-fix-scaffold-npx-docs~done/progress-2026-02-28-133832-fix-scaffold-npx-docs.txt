## Codebase Patterns
- `npx create-opentabs-plugin` does NOT work (no npm package by that name). The correct commands are `opentabs plugin create` (primary, requires CLI) or `npx @opentabs-dev/create-plugin` (alternative without CLI)
- CHANGELOG.md is historical and should NOT be updated for command name fixes
- Prettier pre-commit hook auto-formats staged .md files
- `platform/create-plugin/src/index.ts` delegates entirely to `scaffoldPlugin` from `@opentabs-dev/cli/scaffold` — all scaffold changes only go in `platform/cli/src/scaffold.ts`
- `platform/create-plugin/src/index.test.ts` runs full install+build+lint+format:check cycle on the generated plugin project (not just unit tests). Generated code MUST pass ESLint in the context of a fresh generated project.
- `typescript-eslint strict` mode flags unused `import type` entries — types added to `import type { }` MUST be used in actual TypeScript code, not just in comments. Adding `ResourceDefinition` or `PromptDefinition` to the import without using them in code will fail lint in the generated project.

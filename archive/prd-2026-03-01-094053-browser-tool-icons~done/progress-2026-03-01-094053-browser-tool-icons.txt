## Codebase Patterns
- BrowserToolDefinition is in platform/mcp-server/src/browser-tools/definition.ts — it defines the type and factory function
- CachedBrowserTool (state.ts) is the runtime cached version; BrowserToolDefinition is the source definition
- ConfigStateBrowserTool (platform/shared/src/index.ts) is the wire type sent from MCP server → extension
- Browser tool wire serialization happens in extension-handlers.ts in handleConfigGetState
- Use `...(field ? { field } : {})` pattern for optional fields in wire serialization (matches existing codebase pattern)

## Codebase Patterns
- The plugin-sdk uses function expressions (arrow functions) per ESLint `func-style` rule ‚Äî never `function` declarations
- Import order is enforced by `import-x/order`: local imports (./), then bun: built-ins, then external packages
- `Element.textContent` is typed as `string` (not `string | null`) in bun-types ‚Äî no null guard needed
- `expect(promise).rejects.toThrow()` in bun:test does NOT need `await` ‚Äî bun handles it internally
- happy-dom: use `GlobalWindow` (not `Window`) when assigning `document` to `globalThis` ‚Äî `Window`'s document loses internal `this.window` linkage causing `TypeError: undefined is not a constructor` in `SelectorParser`
- happy-dom MutationObserver: use `queueMicrotask()` (not `setTimeout()`) to schedule DOM mutations in tests ‚Äî `setTimeout` mutations don't trigger the observer reliably
- ToolError is in its own file (platform/plugin-sdk/src/errors.ts) to avoid circular imports ‚Äî SDK utility modules import from errors.ts, and index.ts re-exports from it
- ESLint `no-unnecessary-condition` doesn't track that `await` yields control ‚Äî a variable checked before `await` will be narrowed as if it didn't change. Use object-wrapped state (`{ settled: false }`) or remove the post-await guard entirely (Promise resolve/reject are idempotent)
- `AbortController.abort()` without arguments sets `signal.reason` to a `DOMException` (name: 'AbortError') ‚Äî it's an `instanceof Error`, so don't test for custom "retry: aborted" message; test for `DOMException` with `AbortError` name instead
- In retry-style tests, use `Promise.resolve()` / `Promise.reject()` instead of `async () =>` to satisfy `@typescript-eslint/require-await`
- For SDK modules that need ToolError: import from `./errors.js`, NOT from `./index.js` (circular dependency)
- Bun's `Server` type requires a generic parameter ‚Äî use `ReturnType<typeof Bun.serve>` instead of `Server` to avoid type errors
- `response.json()` returns `any` ‚Äî cast with `as T` to satisfy `@typescript-eslint/no-unsafe-assignment`
- `server.port` is `number | undefined` ‚Äî use `String(server.port)` in template literals to satisfy `@typescript-eslint/restrict-template-expressions`
- `RequestInit.headers` is `HeadersInit` (can be `string[][]`) ‚Äî spreading it in an object triggers `@typescript-eslint/no-misused-spread`. Use `Object.fromEntries(new Headers(h).entries())` to normalize
- `server.stop()` returns void ‚Äî prefix with `void` in afterAll to satisfy `@typescript-eslint/no-floating-promises`
- `bun test platform/` does NOT recurse into plugin-sdk ‚Äî run `bun test platform/plugin-sdk/` separately to verify SDK tests
- `globalThis.localStorage` and `globalThis.sessionStorage` are readonly ‚Äî use `Object.defineProperty(globalThis, 'localStorage', { value: ..., configurable: true, writable: true })` to assign happy-dom's storage in tests
- For cookie tests, override `document.cookie` via `Object.defineProperty(win.document, 'cookie', { get: () => ..., configurable: true })` since happy-dom's cookie handling is limited
- ESLint `@typescript-eslint/no-unnecessary-type-parameters` forbids generics used only once ‚Äî return `unknown` and let callers cast with `as` at the call site
- ESLint `eqeqeq` forbids `== null` ‚Äî use explicit `=== null || === undefined` checks
- For happy-dom tests needing `window.location`, pass `url` to `GlobalWindow` constructor and assign `globalThis.window` from the instance
- E2E test tools: the `openTestAppTab` helper accepts any URL path (not just root) ‚Äî use `${testServer.url}/sdk-test` for pages with specific test fixtures
- Prettier in e2e-test plugin: no parens on single arrow function params (`async params =>` not `async (params) =>`)
- The regex in build-e2e.test.ts for zeroing the tools array must handle multi-line arrays: use `\[\s*echo` not `\[echo`

## Codebase Patterns
- Lucide icons inside Empty.Icon containers should use `size={N}` prop (e.g. `size={40}`) instead of `className="h-full w-full"` to prevent sub-pixel anti-aliasing from non-integer SVG scaling
- EmptyIcon (Empty.tsx) uses `flex items-center justify-center` on the container div so the icon centers within its h-10 w-10 bounds when rendered at a fixed pixel size
- The `h-10 w-10` class on Empty.Icon containers is a layout box — it controls positioning, not icon dimensions; icon size is set separately via the size prop
- When changing visible text in the side panel, always search for the old text in E2E test files (`e2e/*.e2e.ts`) to update `getByText()` assertions

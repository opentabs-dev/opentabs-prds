## Codebase Patterns
- Plugin standalone projects do NOT have node_modules pre-installed by ralph — run `npm install` before quality checks
- The `^0.0.64` semver range for zero-minor packages resolves to exactly `0.0.64` (no automatic upgrades) — must explicitly bump to `^0.0.65` to get new SDK features
- When a plugin overrides a base class property from `OpenTabsPlugin`, use `override readonly` — TypeScript strict mode (`noImplicitOverride`) requires the `override` keyword
- Plugin version bumps must update both `version`, `@opentabs-dev/plugin-sdk`, and `@opentabs-dev/plugin-tools` dependencies in sync

## Codebase Patterns
- Plugin properties that exist in the base class require `override` — but only when using the *published* npm version that includes the property. If the base class hasn't been published yet with a new property, do NOT use `override`.
- Standalone plugins under `plugins/` need `npm install` before building — ralph pre-builds platform packages but NOT plugin `node_modules`.
- Plugin `package.json` opentabs field mirrors plugin class properties: `displayName`, `description`, `urlPatterns`, and `homepage`.

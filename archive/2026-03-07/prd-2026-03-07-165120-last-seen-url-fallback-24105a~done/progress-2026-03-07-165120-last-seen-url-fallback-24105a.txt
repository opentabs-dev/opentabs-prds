## Codebase Patterns
- chrome.storage.local mock pattern: create a `store` object and `mockChromeStorage` with `get`/`set` methods, assign to `(globalThis as Record<string, unknown>).chrome`
- TypeScript strict mode: array index access like `tabs[0]` requires optional chaining (`tabs[0]?.url`) even after a `.length > 0` check
- Tab state persistence uses fire-and-forget `void` prefix on async calls in hot paths (tab-state.ts)
- New source files need to be covered by an existing tsconfig — test files are covered by tsconfig.test.json via `src/**/*.test.ts`
- E2E tests using fixed ports (e.g., 9876 for e2e-test plugin homepage) must handle EADDRINUSE gracefully to support parallel test execution
- When asserting new tab creation in E2E, use `context.pages().length` instead of URL content when the target server may be unavailable

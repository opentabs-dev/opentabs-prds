## Codebase Patterns
- Plugin standalone projects in plugins/ need `npm install` before building — Ralph pre-builds only `plugins/e2e-test`, not all plugins.
- The root node_modules/@opentabs-dev/plugin-sdk is the locally-built version with latest features; local npm install in a plugin dir gets the published tarball (may be older/missing new fields).
- When the local published npm package doesn't declare a field in the base class, don't use `override` — just add the property directly. With `noImplicitOverride: true`, `override` is only needed when actually overriding a declared base member.

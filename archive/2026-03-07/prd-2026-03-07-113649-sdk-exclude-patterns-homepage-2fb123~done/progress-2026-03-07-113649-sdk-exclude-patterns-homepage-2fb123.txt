## Codebase Patterns
- PluginOpentabsField (manifest.ts) uses camelCase for field names
- PluginManifest (index.ts) uses snake_case for url_patterns/exclude_patterns, matching Chrome match pattern convention
- WirePluginPayload and ConfigStatePlugin use camelCase
- parsePluginPackageJson validation uses err() from Result type for structured error handling
- Error messages follow the pattern: `Invalid package.json at ${sourcePath}: "opentabs.<field>" must be ...`
- Optional fields are spread conditionally: `...(value ? { field: value } : {})` to keep output clean
- Browser extension's background-message-handlers.ts uses `...metaFields` spread from PluginMeta — new optional fields flow through automatically
- ValidatedPluginPayload defaults arrays to [] and strings to undefined when absent — toPluginMeta converts empty arrays back to undefined for storage efficiency

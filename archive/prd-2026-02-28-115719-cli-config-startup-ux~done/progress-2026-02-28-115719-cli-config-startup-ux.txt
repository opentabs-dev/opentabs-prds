## Codebase Patterns
- `platform/cli/src/commands/config.ts` contains all `opentabs config` subcommand handlers as private functions, with only pure utilities exported for testing
- Tests in `platform/cli/src/commands/config.test.ts` only test exported pure functions — use `Reflect.deleteProperty(obj, key)` instead of `delete obj[key]` to satisfy the `@typescript-eslint/no-dynamic-delete` lint rule when dynamic deletion is needed
- The ESLint rule `no-dynamic-delete` forbids `delete obj[key]` with variable keys; use `Reflect.deleteProperty(obj, key)` as the equivalent
- `DEFAULT_PORT` is imported from `@opentabs-dev/shared` and is already available in config.ts
- The `tools` and `browserToolPolicy` config objects should only store explicit overrides (disabled=false); enabled entries should be absent (not set to true) to keep config minimal

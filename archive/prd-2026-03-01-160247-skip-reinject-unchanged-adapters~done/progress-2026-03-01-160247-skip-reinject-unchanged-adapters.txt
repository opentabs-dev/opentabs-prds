## Codebase Patterns
- The adapter hash property on the injected adapter object is `__adapterHash` (set by plugin-tools build.ts). The `readAdapterHash` function in `iife-injection.ts` already reads this correctly.
- `extension-commands.ts` implements the `extension.checkAdapter` diagnostic command using `executeScript` to inspect `window.__openTabs.adapters[pluginName]`.
- The embedded `__adapterHash` is a SHA-256 of the core IIFE content BEFORE the hash-setter snippet. Code appended after the hash-setter (e.g., by E2E tests) does NOT change the embedded hash. This means `plugin.update` from the file watcher can send a hash that matches the tab's hash even though the file content changed — the hash-based skip must only apply to `sync.full` (reconnect), not `plugin.update`.
- `injectPluginIntoMatchingTabs` has a `skipIfHashMatches` parameter (default `false`). Only `handleSyncFull` sets it to `true`. `handlePluginUpdate` uses the default to always force re-injection.

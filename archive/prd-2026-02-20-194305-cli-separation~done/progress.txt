## Codebase Patterns
- Each platform package has two tsconfigs: main `tsconfig.json` (compiles to dist/) and `tsconfig.test.json` (noEmit, type-checks test files only)
- Root `tsconfig.json` must reference both tsconfigs for each new package
- `knip.ts` needs a workspace entry for each new platform package (with test file entries)
- `eslint.config.ts` needs node globals for server-side platform packages
- The `platform/*` workspace glob in root `package.json` auto-covers new packages
- Build command source depends on: @opentabs-dev/plugin-sdk, @opentabs-dev/shared, commander, picocolors, zod
- CLI entry points use shebang `#!/usr/bin/env bun` and read version from `../package.json`
- `scaffold.ts` imports `validatePluginName` and `validateUrlPattern` from `@opentabs-dev/plugin-sdk` — so cli still needs plugin-sdk as a dependency even after removing the build command
- When removing commands from the CLI, scaffold.ts generates package.json with devDep and script references that must be updated simultaneously to avoid test failures
- The `create-plugin` test `overrideToLocalPackages` must match the scaffolded devDependencies exactly
- Commander shows commands in registration order — register important commands first for --help display
- When renaming/removing CLI commands, search all other commands for hint messages referencing the old names

## Codebase Patterns
- Platform packages are in platform/, plugins in plugins/
- File watcher pattern: watch directories (not files) on macOS, debounce at 200ms using `state.fileWatcherTimers`, check `fileWatcherGeneration` to discard stale callbacks
- `getConfigDir()` reads `OPENTABS_CONFIG_DIR` env var, falling back to `~/.opentabs`
- `performConfigReload` is the entry point for runtime config reloads (vs `performReload` for full hot reload)
- `noopCallbacks` in file-watcher.test.ts must include all FileWatcherCallbacks fields
- `reloadCore` is the shared reload logic — file watchers are always restarted at the end regardless of success/failure
- `SIDE_PANEL_METHODS` in message-router.ts gates which notifications reach the side panel — async handlers (handleSyncFull, handlePluginUpdate) must call `forwardToSidePanel` directly
- Side panel App.tsx uses `loadPluginsRef.current()` for stable plugin refetch — new notification handlers should use this ref pattern
- Config watcher E2E tests don't need browser context — MCP server + MCP client only, making them fast (~500ms)
- Always wait for `Config watcher: Watching` log before writing config changes in E2E tests
- Side panel E2E: open as regular tab at `chrome-extension://<id>/side-panel/side-panel.html`, NOT via chrome.sidePanel.open()
- App.tsx has a ws:message/sync.full fallback for side panel live-updates (forwardToSidePanel doesn't reach regular tabs in headless)

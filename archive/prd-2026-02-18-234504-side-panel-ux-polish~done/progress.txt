## Codebase Patterns
- Side panel components live in `platform/browser-extension/src/side-panel/components/`
- RetroUI design system components are in `components/retro/` (Accordion, Alert, Badge, Button, Empty, Input, Loader, Switch, Text, Tooltip)
- The Input component has `py-2` vertical padding — absolute-positioned icons must be inside a `relative` container that matches the input dimensions (no extra padding)
- When absolute-positioning icons inside inputs, use a wrapper structure: outer div for spacing (`px-3 pt-2`), inner `relative` div for positioning context
- PluginIcon SVG `fill` attribute is on the `<svg>` element, not `<path>` — E2E tests should use `.bg-muted svg` locator with `toHaveAttribute('fill', ...)` to check icon state

- AccordionContent uses `bg-card text-card-foreground` (intentional deviation from RetroUI's `bg-white text-gray-700`) for dark mode support — this is the ONLY deviation from RetroUI source across all retro/ components

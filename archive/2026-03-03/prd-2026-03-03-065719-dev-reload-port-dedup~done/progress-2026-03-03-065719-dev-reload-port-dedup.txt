## Codebase Patterns
- Build scripts in `platform/browser-extension/` (build-*.ts) are type-checked by `tsconfig.build-scripts.json` (noEmit: true). When importing cross-project constants from `scripts/`, you must: (1) add `allowImportingTsExtensions: true` to that tsconfig, and (2) add the imported file(s) to its `include` list.
- `tsconfig.scripts.json` covers `scripts/**/*.ts` with `noEmit: true`. A file can appear in both `tsconfig.scripts.json` and `tsconfig.build-scripts.json` include lists without conflict since neither emits declarations.
- Build scripts run via `tsx`, so `.ts` extensions in imports are valid at runtime. But `tsc --build` requires `allowImportingTsExtensions: true` for `.ts` extension imports.

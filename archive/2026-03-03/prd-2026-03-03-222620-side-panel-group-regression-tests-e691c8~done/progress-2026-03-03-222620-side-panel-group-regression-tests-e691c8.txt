## Codebase Patterns
- BrowserToolState type is aliased from ConfigStateBrowserTool (defined in platform/shared/src/index.ts) — has optional `group?: string` and `icon?: string` fields
- Actual browser tool groups in the catalog: 'Page Inspection', 'Page Interaction', 'Tabs', 'Storage & Cookies', 'Network', 'Extension', 'Plugins'
- BrowserToolsCard grouping logic: tools with `group` set are bucketed by group name (first-seen order), ungrouped tools go into 'Other' at the bottom; if no tools have `group`, renders flat list
- Group headers are plain dividers: `<div className='border-border border-b bg-muted/20 px-3 py-1'><span className='font-head text-muted-foreground text-xs uppercase tracking-wider'>GROUP NAME</span></div>`
- Storybook stories pattern: each story has a Demo component (stateful wrapper) and a Story object with `render: () => <Demo />`
- E2E: Group headers selectable via `span.uppercase.tracking-wider` within accordion items
- E2E: Searching in side panel switches from normal view to SearchResults view — the BrowserToolsCard accordion starts collapsed and must be re-expanded
- E2E: plugin_analyze_site description contains "cookies" — avoid "cookie" as a single-group filter test term; "hover" matches only Page Interaction group
- PluginCard group headers include a Switch toggle (`button[role="switch"]`) for bulk permission control — BrowserToolsCard does not
- E2E: PluginCard group switch buttons have aria-labels like `Toggle all Basic tools` — use these for targeting specific group switches
- E2E: Plugin group headers container selectable via `div.bg-muted\\/20` within the plugin accordion item

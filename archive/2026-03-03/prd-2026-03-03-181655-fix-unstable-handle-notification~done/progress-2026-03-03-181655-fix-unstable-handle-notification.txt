## Codebase Patterns
- React hooks returning callbacks should use `useCallback` to provide stable references — otherwise consumers' useEffects re-run on every render
- React state setters (from useState) and refs are guaranteed stable by React, so they don't need to be in useCallback dependency arrays, but including them satisfies exhaustive-deps lint rules harmlessly
- The side panel's main useEffect in App.tsx depends on `[handleNotification]` from useServerNotifications — any instability in that reference causes redundant getFullState() IPC calls and Chrome listener re-registration

## Codebase Patterns
- `handleBrowserExecuteScript` in `platform/browser-extension/src/browser-commands/extension-commands.ts` uses a namespaced IIFE (`injectPromise`) with an outer `Promise.race` against `timeoutPromise`.
- When testing async operations that run in the background after a handler settles, use `resolveManually()` + `await Promise.resolve()` to flush microtasks without depending on timer ordering.
- The polling while loop's `if (cancelled.value) return;` check fires between poll iterations (during the 50ms wait), not while a poll executeScript is in-flight.

- **Learnings for future iterations:**
  - Bracketed IPv6 pattern must run before bare IPv6 to prevent partial replacement leaving dangling brackets (e.g., `[::1]` would become `[[IP]]` if bare ran first).
  - The bare IPv6 pattern requires exactly two colons minimum; single-colon strings like `key: value` and `localhost:9515` don't match.
  - The `(?:%\w+)?` suffix handles zone IDs (e.g., `%eth0`) — important for link-local addresses.
  - IPv6 patterns placed before localhost to avoid any order-dependent partial match in messages that contain both forms.

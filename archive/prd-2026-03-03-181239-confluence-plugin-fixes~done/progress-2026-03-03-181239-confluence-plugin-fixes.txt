## Codebase Patterns
- Plugins are standalone packages under `plugins/` — NOT in npm workspaces. They have no node_modules pre-installed in worktrees. Must run `npm install` before quality checks will pass.
- Plugin quality checks run from inside the plugin directory: `cd plugins/<name> && npm run build && ...`
- The `opentabs-plugin build` step (part of `npm run build`) requires `node_modules/@opentabs-dev/plugin-sdk/package.json` to exist to resolve the SDK version.
- `parseRetryAfterMs` is exported from `@opentabs-dev/plugin-sdk` and handles both numeric seconds and HTTP-date Retry-After formats.

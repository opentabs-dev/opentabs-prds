## Codebase Patterns
- network-capture.ts maintains three maps per tab: state.requests (completed), state.pendingRequests (in-flight awaiting responseReceived), state.requestIdToRequest (requestId → completed ref, for body attachment)
- Tests set up chrome stubs before importing the module; capture listener callbacks by replacing vi.fn() with a function that saves the cb variable; use `await import()` after stubs
- Prettier print width allows single-line type unions like `Foo | undefined` up to ~120 chars; multi-line union declarations get flagged
- `noUncheckedIndexedAccess` is enabled in tsconfig.base.json — use `toHaveProperty` instead of `arr[0]!.field` to avoid `no-non-null-assertion` lint errors
- `no-misused-promises` lint rule: mockImplementationOnce callbacks must return void (not a Promise); omit `return Promise.resolve()`
- To test async sendCommand callbacks: use mockImplementationOnce to capture the callback, invoke it synchronously after setting up the desired state

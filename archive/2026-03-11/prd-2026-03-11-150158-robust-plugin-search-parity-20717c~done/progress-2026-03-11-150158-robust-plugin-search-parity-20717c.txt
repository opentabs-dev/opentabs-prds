## Codebase Patterns
- `probeDirectPackage` in `platform/mcp-server/src/plugin-management.ts` and `buildDirectLookupCandidates` in `platform/cli/src/commands/plugin.ts` use the same candidate-building logic for direct npm registry lookups
- When query starts with `opentabs-plugin-`, the scoped variant (`@opentabs-dev/<query>`) must come first in the candidates list (try official packages before community packages)
- Tests for `buildDirectLookupCandidates` are in `platform/cli/src/commands/plugin.test.ts`
- Run E2E tests with `CI=1 npm run test:e2e` when outside Docker; without it, async `browser_execute_script` tests flake under 4 parallel Chromium workers (Chrome timer throttling)

## Codebase Patterns
- Browser tools are defined in `platform/mcp-server/src/browser-tools/`, one file per tool
- Each browser tool uses `defineBrowserTool()` with `name`, `description`, `input` (Zod schema), and `handler`
- Descriptions are simple strings; multi-line descriptions use string concatenation with `+`
- Slack plugin tools are in `plugins/slack/src/tools/`, one file per tool, using `defineTool()` from the SDK
- Plugin `dist/` directories are gitignored — only commit source files and `opentabs-plugin.json`
- The Slack API wire format uses `timestamp` for message IDs, but the MCP-facing schema uses `ts` — handlers must map between them

## Codebase Patterns
- `platform/plugin-sdk/src/fetch.ts`: fetchFromPage uses `AbortSignal.any([signal, timeoutSignal])` to merge signals. The catch block must check TimeoutError first (to give it a specific message), then `combinedSignal.aborted` (to catch all abort cases including non-DOMException reasons), then fall through to network_error.
- Test file `fetch.test.ts` uses a real Node.js HTTP server (createServer from node:http) rather than fetch mocks. Abort tests pass a pre-aborted AbortController signal — no server endpoint needed since fetch throws immediately.
- `ToolError` defaults `retryable: false` unless explicitly set. `new ToolError(msg, 'aborted')` is non-retryable by default.

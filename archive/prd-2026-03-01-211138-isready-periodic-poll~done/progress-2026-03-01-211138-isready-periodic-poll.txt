## Codebase Patterns
- Tab state management lives in `platform/browser-extension/src/tab-state.ts` ‚Äî all state computation and caching is centralized there
- `notifyAffectedPlugins` is the key function for re-evaluating and diffing plugin state; it uses `withPluginLock` to serialize per-plugin
- `lastKnownState` is a Map<string, serialized JSON> used for diff detection ‚Äî changes in aggregate state, tab list, or per-tab readiness all produce different serialized strings
- WebSocket connection/disconnection is handled in `background-message-handlers.ts` via `handleWsState`; cleanup (cache clear, poll stop) happens on disconnect
- `sync.full` from the MCP server triggers `handleSyncFull` in `message-router.ts` which stores plugins, injects adapters, then calls `sendTabSyncAll`
- Constants for timeouts and intervals live in `platform/browser-extension/src/constants.ts`
- Test mocks for `tab-state.js` must include ALL exported functions ‚Äî when adding new exports, update mocks in `background-message-handlers.test.ts`, `message-router.test.ts`, and `known-methods.test.ts`

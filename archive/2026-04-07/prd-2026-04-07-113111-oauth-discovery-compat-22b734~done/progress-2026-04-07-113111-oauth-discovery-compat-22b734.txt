## Codebase Patterns
- The main HTTP router is `createHandleFetch` in `platform/mcp-server/src/http-routes.ts`, dispatching by `url.pathname` matching with a catch-all at the end
- Security checks (Host header validation, CORS/Origin check) run before any route matching — new routes inherit these protections automatically
- `Response.json()` is the preferred way to return JSON responses (used throughout the codebase)
- E2E tests for HTTP endpoints follow the pattern in `e2e/security.e2e.ts`: `createTestConfigDir()` + `startMcpServer()` + `server.waitForHealth()` + `fetch()` assertions
- MCP Streamable HTTP responses use SSE format (text/event-stream) when client sends `Accept: text/event-stream` — parse `data:` lines to extract JSON-RPC messages instead of using `res.json()`

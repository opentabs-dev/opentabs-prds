## Codebase Patterns
- CLI commands live in `platform/cli/src/commands/` with one file per command group
- Each command file exports a `register*Command` function that receives the Commander `program`
- Subcommands are chained off a parent command (e.g., `pluginCmd.command('search')`)
- Use `picocolors` (`pc`) for colored output — `pc.red` for errors, `pc.yellow` for warnings, `pc.blue` for highlights, `pc.dim` for secondary info
- The project enforces `func-style` lint rule — use `const fn = () => {}` not `function fn() {}`
- `process.exit(1)` returns `never` in TypeScript types, so TypeScript flow analysis handles it correctly for definite assignment

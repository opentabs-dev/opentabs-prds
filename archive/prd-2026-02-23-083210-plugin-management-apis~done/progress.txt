## Codebase Patterns
- The McpCallbacks interface in extension-protocol.ts is the contract between the extension WebSocket protocol handlers and the MCP server. Add new callbacks here for operations that need access to session servers, transports, or config reload.
- The `createMcpCallbacks()` in http-routes.ts wires callbacks to the real implementations and passes `state`, `sessionServers`, and `transports`.
- Plugin name normalization: shorthand "slack" → "opentabs-plugin-slack". Scoped names and full names pass through. Same logic in CLI and server.
- `sendToExtension()` accepts any JSON-RPC message type (notification, result, error, request). Use it for both responses and push notifications.
- The `plugins.changed` notification is already handled by the extension message-router.ts (SIDE_PANEL_METHODS set) and side panel App.tsx — just send it via sendToExtension.
- `platformExec('npm')` from `@opentabs-dev/shared` resolves to `npm.cmd` on Windows or `npm` on Unix.
- ESLint enforces `import-x/exports-last` — all exports must be at the bottom of the file.
- The e2e-test plugin needs `bun install` in its directory before build-e2e.test.ts tests can pass.
- `findPlugin()` in plugin-management.ts matches by npmPackageName, stripped name, plugin name, or raw name — reuse it when looking up plugins by user-provided names.
- The `configWriteMutex` on ServerState serializes all config file writes to prevent concurrent read-modify-write races. Any new config write operation must chain on this mutex.
- The `plugin.uninstall` JSON-RPC method is already handled by the extension (message-router.ts / known-methods.ts) — the server just needs to send it with a name param.
- When naming new JSON-RPC methods, check for collisions with existing methods (e.g., `plugin.update` is already used as a notification from the file watcher).
- For E2E tests that send raw JSON-RPC over WebSocket, use `fetchWsInfo` + `WebSocket` with protocol headers. Create a `sendRequest` helper with a pending Map keyed by UUID for clean request/response correlation.
- `createMinimalPlugin` gives plugins the `opentabs-plugin-<name>` npm package name in package.json, making them discoverable as if npm-installed.

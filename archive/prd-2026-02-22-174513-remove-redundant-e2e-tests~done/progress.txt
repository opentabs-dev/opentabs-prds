## Codebase Patterns
- E2E tests in `e2e/` require the `plugins/e2e-test/` plugin to be built first (`cd plugins/e2e-test && bun install && bun run build`). The `dist/` and `node_modules/` directories are gitignored, so fresh worktrees need this step before `bun run test` passes.
- The `/ws-info` endpoint returns `wsSecret` in the JSON body for authenticated callers — this is by design for extension bootstrapping. The secret is only "leaked" if it appears in the URL as a query parameter.
- Test comments referencing other test files by name should be updated when those files are deleted/renamed.

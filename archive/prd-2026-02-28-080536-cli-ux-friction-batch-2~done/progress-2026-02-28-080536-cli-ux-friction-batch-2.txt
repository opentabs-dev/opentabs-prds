## Codebase Patterns
- `fail()` sets `fatal: true` (exits 1); `warn()` sets `fatal: false` (exits 0) in doctor.ts
- `checkExtensionConnected` and `checkNpmPlugins` already use `warn()` when server is not reachable
- The doctor summary logic is already gated on `hasFatal` — no changes needed to the exit-code logic
- Commander v14 `unknownCommand()` is in the JS implementation but NOT in TypeScript typings; cast as `(cmd as unknown as { unknownCommand: () => never })` to call it
- Adding `.allowExcessArguments(true)` to a Commander command prevents "too many arguments" error when excess positional args are passed; the args then appear in `command.args` in the action handler
- `checkMcpClientConfig` accepts a `McpClientLocation[]` parameter so tests inject paths directly; `defaultMcpClientLocations()` is a private factory — no need to export it or test it separately
- OpenCode's home-based global config path (if any) is `~/.config/opencode/opencode.json` (XDG standard); Cursor's global MCP config is at `~/.cursor/mcp.json`

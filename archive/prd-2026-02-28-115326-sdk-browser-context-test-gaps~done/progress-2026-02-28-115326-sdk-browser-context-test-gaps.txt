## Codebase Patterns
- Storage tests: mock getter with `Object.defineProperty(globalThis, 'localStorage', { get: () => { throw ... }, configurable: true })` for access-level errors; mock the whole object with a `value: { setItem: () => { throw ... } }` for method-level errors like QuotaExceededError
- MutationObserver tests in happy-dom: use `setTimeout(resolve, 50)` to let observer callbacks fire asynchronously; happy-dom MutationObserver correctly handles `attributes: true` / `childList: false` distinctions
- Restore mocked globals after use within the test body (not just in afterEach) when the override is test-specific and the beforeEach restores to `win.localStorage`
- When calling `observeDOM` with 3+ args, prettier formats it with the selector, callback, and options each on their own line (arg-per-line style)

## Codebase Patterns
- LogFilterOptions is defined in platform/browser-extension/src/log-collector.ts — has level, source, since, limit fields
- OffscreenGetLogsMessage is in extension-messages.ts — options field mirrors LogFilterOptions (limit optional)
- _prefixed unused destructure variables satisfy ESLint no-unused-vars rules
- React Compiler (babel-plugin-react-compiler) handles auto-memoization, but useCallback is still needed as an escape hatch for ensuring effect dependency stability

- Module-level imports (e.g., fetchConfigState, searchPlugins from bridge.js) are stable — safe as empty useCallback deps
- useServerNotifications hook params (setPlugins, setActiveTools, etc.) are state setters passed from App — stable identity, but listed in deps for correctness

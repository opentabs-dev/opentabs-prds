## Codebase Patterns
- Docs site lives in `docs/` with its own package.json, uses Next.js 16 + content-collections
- Docs quality checks: `cd docs && bun run build && bun run type-check && bun run lint && bun run knip`
- Root knip needs `bun install` to resolve workspace:* dependencies ‚Äî without node_modules, knip reports false "unused dependency" errors for @opentabs-dev/* packages
- CLI reference uses `<Table>` components from fumadocs for structured option/flag documentation
- Plugin CLI source is in `platform/cli/src/commands/plugin.ts` ‚Äî all five subcommands (search, list, install, remove, create) are in one file
- Pre-commit hook runs root-level lint-staged + knip, not docs-level knip
- Config CLI source is in `platform/cli/src/commands/config.ts` ‚Äî all five subcommands (show, set, path, reset, rotate-secret) in one file
- Audit CLI source is in `platform/cli/src/commands/audit.ts` ‚Äî six flags: --limit, --plugin, --tool, --since, --json, --file
- Config schema is defined in `platform/mcp-server/src/config.ts` ‚Äî `OpentabsConfig` interface is canonical
- Permission evaluation chain is in `platform/mcp-server/src/permissions.ts` ‚Äî domainToolPolicy > sensitiveDomains > toolPolicy > tier default > trustedDomains
- Always verify PRD claims against actual source code ‚Äî PRDs can reference non-existent features

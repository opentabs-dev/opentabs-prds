## Codebase Patterns
- Dev proxy (`platform/mcp-server/src/dev-proxy.ts`) manages MCP sessions across worker restarts with a `ProxySession` interface and `sessions` Map keyed by proxy session ID
- `cleanupSession` is the single cleanup entrypoint — always release resources there (timers, SSE streams, session map entries)
- `disconnectUpstreamSse` / `connectUpstreamSse` manage the single upstream SSE connection per session; idle tracking should key off these transitions
- The `handleMcpGet` close handler is where client SSE streams end — the right place to start idle tracking

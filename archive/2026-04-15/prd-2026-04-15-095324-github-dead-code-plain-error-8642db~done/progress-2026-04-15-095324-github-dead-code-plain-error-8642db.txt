## Codebase Patterns
- The github plugin does NOT have its own node_modules pre-installed in ralph worktrees — run `npm install` in `plugins/github` before running `npm run build` (opentabs-plugin build resolves sdk version from the plugin's own node_modules)
- `fetchFromPage()` from `@opentabs-dev/plugin-sdk` throws on non-ok responses; `doFetch()` wraps it and re-throws, so any `if (!response.ok)` check after `doFetch()` or `fetchFromPage()` is unreachable dead code
- `classifyError()` in github-api.ts was only called from the three dead `!response.ok` branches — removing those branches made the function entirely unused; it was deleted too

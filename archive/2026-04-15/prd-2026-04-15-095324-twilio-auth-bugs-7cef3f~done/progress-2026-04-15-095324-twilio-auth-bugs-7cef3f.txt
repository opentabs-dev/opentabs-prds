## Codebase Patterns
- Twilio plugin uses auth cache (`getAuthCache`/`setAuthCache` with key `'twilio'`) for credential storage, not raw localStorage
- `getAuth()` is the synchronous auth check helper; `ensureAuth()` is the async version that fetches from the API if cache is empty
- `isReady()` in index.ts calls `isAuthenticated()` directly, then falls back to `waitForAuth()` which polls with 500ms interval / 5s timeout
- Plugin node_modules are not pre-installed in worktrees for standalone plugins — run `npm install` before builds

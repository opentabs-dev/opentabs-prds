## Codebase Patterns
- The stackoverflow plugin is a standalone project in plugins/stackoverflow/ with its own node_modules (must run `npm install` if missing)
- The `opentabs-plugin build` step in `npm run build` requires node_modules to be installed (unlike tsc which uses workspace resolution)
- When changing types from `any` to `unknown` in response parsing, use `as unknown as TargetType` for double-cast through unknown
- The SE API uses application-layer error codes (error_id) on both 200 and non-200 responses — error handling must check error_id before response.ok
- Mapper functions use `as z.infer<typeof schema>` return assertions to bridge `unknown` properties to typed outputs
- Use a `nested()` helper for safe nested object access on `Record<string, unknown>` — optional chaining doesn't work directly on `unknown`

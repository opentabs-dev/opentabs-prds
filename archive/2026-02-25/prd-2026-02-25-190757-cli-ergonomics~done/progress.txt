## Codebase Patterns
- CLI commands are registered via `register*Command(program)` functions in `platform/cli/src/commands/`
- Handler functions (e.g. `handleStatus`) are separate from the commander registration functions and can be independently imported/exported
- Root program `.action()` in Commander is called when no subcommand matches, making it ideal for default behavior
- `command.optsWithGlobals()` is used to get merged options (including global `--port`) in Commander actions

## Codebase Patterns
- plugin-sdk/src/storage.ts: All storage functions wrap their access in try-catch and return null on any error. The inner try-catch for decodeURIComponent in getCookie is nested inside the outer try-catch.
- plugin-sdk/src/fetch.ts: fetchJSONImpl is the shared implementation for fetchJSON, postJSON, putJSON, etc. When a schema is provided, 204/empty responses must still be validated against it. No-schema callers get undefined for 204 responses (backward compatible).
- plugin-tools/src/commands/build-e2e.test.ts: Tests that run `tsc` (auto-tsc feature) need a 30_000ms timeout since tsc can take 10-20s in slow environments. Use the third argument to `test()` to set per-test timeouts.
- plugin-sdk/src/dom.ts: Non-exported helper functions must appear BEFORE the first export statement to satisfy the `import-x/exports-last` ESLint rule. The `needsAttributeObservation` helper was placed before the exported interfaces for this reason.
- plugin-tools/src/commands/build.ts: When spawnSync returns with `error` (ENOENT or other OS errors), stdout/stderr are null at runtime. Check `tscResult.error` first and throw a user-friendly message. The existing `status !== 0` branch only runs when `error` is falsy, at which point stdout/stderr are guaranteed non-null Buffers — no optional chaining needed there.

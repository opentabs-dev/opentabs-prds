## Codebase Patterns
- The `void (async () => { ... })()` IIFE pattern in background.ts is used for fire-and-forget async work in Chrome extension event handlers. Add `.catch()` to the end of the IIFE call: `void (async () => { ... })().catch((err: unknown) => console.warn('[opentabs] ...', err));`
- Chrome extension error log messages use the pattern `console.warn('[opentabs] <description>:', err)`
- `tsc --build --noEmit` can fail with TS6310 due to stale build artifacts. Run `bun run clean` (from root) to clear tsbuildinfo files, then `bun run build`, and re-run `bun run type-check`. This resolves the issue without any tsconfig changes.

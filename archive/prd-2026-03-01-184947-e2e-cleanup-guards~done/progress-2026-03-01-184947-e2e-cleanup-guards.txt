## Codebase Patterns
- E2E cleanup pattern: `await context.close().catch(() => {})` prevents a crashed Playwright BrowserContext from skipping subsequent cleanup calls (server.kill, fs.rmSync, etc.)
- The catch block pattern in `e2e/helpers.ts` (lines 444-452) already shows the correct defensive pattern for error-path cleanup — happy-path cleanup should mirror it
- `client.close()` and `server.kill()` have internal try/catch guards, so they never throw; `context.close()` (Playwright) can throw if the browser crashed

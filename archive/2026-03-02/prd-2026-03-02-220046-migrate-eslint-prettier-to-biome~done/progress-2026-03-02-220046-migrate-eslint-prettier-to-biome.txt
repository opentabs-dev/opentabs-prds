## Codebase Patterns
- Biome 2.x uses `files.includes` with negation patterns (`!path`) instead of `files.ignore` — use `!` prefix to exclude, `!!` prefix for force-ignore (prevents scanning entirely)
- Biome domains: valid values are `react`, `test`, `solid`, `next`, `qwik`, `vue`, `project`, `tailwind` — `accessibility` is NOT a valid domain (a11y rules are in the recommended set)
- Biome rule groups: `noVar` is in `suspicious` (not `style`), `noDoubleEquals` is in `suspicious`, `useConst`/`useImportType`/`useExportType`/`useConsistentArrowReturn` are in `style`, `noUnusedVariables`/`noUnusedImports`/`noUnusedFunctionParameters` are in `correctness`
- `organizeImports` level only accepts `"on"` or `"off"`, not severity levels like `"error"`
- Biome does NOT support markdown formatting — only JS/TS/JSX/TSX, JSON, CSS, GraphQL, HTML
- @biomejs/biome should be installed with `--save-exact` per Biome's recommendation
- Root biome.json is at version 2.4.5 using schema `https://biomejs.dev/schemas/2.4.5/schema.json`
- Biome CSS parser cannot handle Tailwind v4 `@theme` at-rule — exclude `**/*.css` from biome file scanning
- `biome lint --fix --unsafe` applies `useConsistentArrowReturn` which can break TypeScript when optional chaining returns `T | undefined` where `boolean` is expected — always add `?? false` when converting those arrow returns
- biome.json rule categories: `security` for `noDangerouslySetInnerHtml`, `a11y` for accessibility rules like `noStaticElementInteractions`, `useAnchorContent`, `useSemanticElements`, etc.
- lint-staged with biome: use `--no-errors-on-unmatched` flag to prevent errors when lint-staged passes files biome doesn't recognize
- `next-env.d.ts` is regenerated by `next build` with double quotes — must exclude from biome file scanning in Next.js projects
- `noImgElement` rule (from `next` domain) flags `<img>` usage — disable for projects with MDX component overrides using `<img>` for dynamic content

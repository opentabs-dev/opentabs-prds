## Codebase Patterns
- `SpConfirmationRequestMessage` was the ONLY external use of the `SpConfirmationRequestMessage['data']` shape — when removing it, inline the data type directly in `ConfirmationDialog.tsx` and export it as `ConfirmationData` (the type is re-exported from there for consumers)
- `extension-protocol.test.ts` tests for `config.setToolEnabled` and `config.setAllToolsEnabled` use `ws.sent[N]` indexing — when handlers send multiple messages (notification + result), update indices accordingly
- The pattern for adding `plugins.changed` before a success result: `sendToExtension(state, { jsonrpc: '2.0', method: 'plugins.changed', params: { ...buildConfigStatePayload(state) } })` then the success response
- `server-state-cache.ts` uses a module-level `persistTimer` and `schedulePersist()` for debounced 500ms writes; `clearServerStateCache()` does immediate session.remove (not debounced)
- `tab-state.ts` uses a module-level `lastKnownStatePersistTimer` with same debounce pattern; `clearTabStateCache()` does immediate session.remove (not debounced)
- `message-router.test.ts` mocks both `./server-state-cache.js` and `./tab-state.js` with vi.mock() factories — any new exports from those modules MUST be added to the mock factories
- `background-message-handlers.test.ts` mocks `./server-state-cache.js` — new exports must also be added to its `vi.mock()` factory
- `handleSyncFull` in `message-router.ts` calls `updateServerStateCache()` synchronously, then fires `sendTabSyncAll()` as a fire-and-forget `.then()` chain
- `sendTabSyncAll()` takes O(N × round-trip) time because it probes every matching tab for readiness; any flush of `lastKnownState` must happen AFTER `sendTabSyncAll` completes, not before (calling flush before writes empty state overwriting good session data)
- `cachesInitialized` flag in server-state-cache.ts distinguishes wake-from-suspension (flag=true) from connect-to-sync.full gap (flag=false); persisted alongside the server state cache in chrome.storage.session
- `clearServerStateCache()` now removes both `serverStateCache` and `cachesInitialized` keys from session storage (array argument to `session.remove()`)

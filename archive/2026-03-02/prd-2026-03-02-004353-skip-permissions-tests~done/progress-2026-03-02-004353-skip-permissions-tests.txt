## Codebase Patterns
- `skip-permissions.ts` computes `cliSkipPermissions` at module load time. Testing it requires `vi.stubEnv()` + `vi.resetModules()` + dynamic `await import('./skip-permissions.js')` pattern.
- Test files for mcp-server: `permissions.test.ts` (evaluatePermission + isCliSkipPermissions), `config.test.ts` (config loading/saving), `http-routes.test.ts` (HTTP route handlers).
- `vi.stubEnv()` requires `vi` and `afterEach` imported from vitest. Use `vi.unstubAllEnvs()` in afterEach to restore env vars after each test.
- `createTestHandlers()` in http-routes.test.ts creates a test state and handlers for testing HTTP routes.

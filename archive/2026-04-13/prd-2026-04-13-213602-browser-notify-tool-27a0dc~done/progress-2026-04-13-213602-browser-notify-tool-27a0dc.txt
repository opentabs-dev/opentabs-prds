## Codebase Patterns
- Browser command handlers follow `(params: Record<string, unknown>, id: string | number) => Promise<void>` pattern with try/catch, `sendSuccessResult`, `sendErrorResult` from `helpers.ts`
- When adding a new browser command, update 4 places: handler file, `browser-commands/index.ts` barrel, `message-router.ts` dispatch table + import, `known-methods.ts` DISPATCH_METHODS array
- Test mocks must be added in 3 files: `known-methods.test.ts`, `message-router.test.ts` (both hoisted destructured mock var AND vi.mock map entry)
- `chrome.notifications.onClicked` supports multiple listeners â€” filter by notification ID prefix to avoid conflicts between handlers
- Biome auto-fixes import ordering during pre-commit; the barrel export in `index.ts` was reordered alphabetically by the linter
- In worktrees, workspace packages are real copies (not symlinks) in each package's local `node_modules/`. When modifying shared or mcp-server dist, copy updates to all local copies before running generators or tests.
- When adding a browser tool to mcp-server: create definition file, add to index.ts barrel array, update browser-tools-catalog.ts (generated file, can edit manually then verify with `npm run generate:browser-tools-catalog`)

## Codebase Patterns
- The CLI catch-all in `cli.ts` sets `process.exitCode ??= 1` for ANY unhandled error from command handlers — commands that should never fail (like `plugin list` in offline mode) must wrap their code paths in try/catch to prevent non-zero exit codes
- On Windows, `spawnProcessAsync`/`spawnProcessSync` use `shell: true` (via `isWindows()`) so cmd.exe can resolve `.cmd` shims — this can produce different failure modes than on Linux/macOS
- Individual functions like `scanNpmPlugins` and `readLocalPluginInfo` have their own try/catch blocks, but errors can still escape between these calls

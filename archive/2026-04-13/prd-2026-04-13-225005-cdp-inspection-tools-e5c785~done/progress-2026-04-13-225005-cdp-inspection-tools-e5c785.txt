2. **Same catalog chicken-and-egg**: Must manually insert into browser-tools-catalog.ts before build, using sed for the generated file.
3. **Edit tool read state**: The Edit tool's "file has been read" check resets between tool calls within the same message. If reading and editing in the same parallel batch, the edit may fail. Read in one step, then edit in the next.

- Parses visibleSecurityState.certificateSecurityState: subjectName, issuer, validFrom/validTo (epoch → ISO string), protocol, keyExchange, cipher

3. **Read tool state resets**: The Edit tool requires the file to be read within the same "conversation turn". If you read a file in one tool call and try to edit it in a later tool call, it may fail. Re-read before editing.

## Codebase Patterns
- The `withDebugger` helper in `platform/browser-extension/src/browser-commands/resource-commands.ts` manages chrome.debugger attach/detach lifecycle and reuses existing sessions from network capture. Use it for all CDP commands.
- The `browser-tools-catalog.ts` in `platform/shared/src/generated/` has a runtime validation check in `platform/mcp-server/src/browser-tools/index.ts` that compares descriptions at module load time. When changing a browser tool description, you must update both the definition AND the catalog file â€” the `npm run generate:browser-tools-catalog` script imports from compiled `dist/` which triggers the validation before it can regenerate, creating a chicken-and-egg problem. Update the catalog directly.
- `extractScriptResult` returns `Record<string, unknown>` â€” cast specific fields (e.g., `result.x as number`) when extracting typed values.
- Importing `withDebugger` from `resource-commands.ts` pulls in `network-capture.ts` which registers `chrome.debugger.onEvent.addListener` and `chrome.tabs.onRemoved.addListener` at module load time. Test files must stub these chrome APIs before the dynamic import.

## Codebase Patterns
- PluginList.tsx renders plugins in groups: failed plugins first, then ready plugins, then not-ready plugins under a "NOT CONNECTED" label
- When a tool filter is active (toolFilter non-empty), grouping is disabled and all matching plugins render in a single flat Accordion
- PluginCard.tsx Accordion.Item className uses conditional logic: transitionClass → removingPlugin → opacity-70 (not-ready) → undefined
- Biome enforces CSS class sorting (nursery/useSortedClasses) — use sorted Tailwind classes (spacing → typography → color → other)
- PluginCard accepts a `transitionClass` prop that overrides the default Accordion.Item className for animation purposes
- Group transition animations use CSS keyframes in styles.css: `group-fade-in` (opacity 0→1) and `group-fade-in-dim` (opacity 0→0.7)
- The `useGroupTransitions` hook in PluginList.tsx tracks plugin tab state changes and manages fade-out/fade-in phases with timeouts
- PluginCard group headers include a Switch toggle for bulk permission control — disabled when skipPermissions is active
- E2E tests from different PRDs/branches can have contradictory assertions — always run full E2E suite to surface conflicts

## 2026-03-04 - US-002
- Implemented smooth animated transitions when plugins move between ready and not-ready groups
- Created `useGroupTransitions` custom hook in PluginList.tsx that tracks previous tabState per plugin and manages a two-phase transition (fade-out → fade-in)
- During fade-out (150ms), plugin card stays in its OLD group and fades to opacity-0 via CSS transition
- During fade-in (200ms), plugin card moves to its NEW group and animates in with `group-fade-in` or `group-fade-in-dim` CSS keyframe animation (includes translateY slide-up)
- Added `transitionClass` prop to PluginCard — when set, it overrides the default opacity class on Accordion.Item
- NOT CONNECTED label fades in/out smoothly via mount/visibility two-phase pattern (mount → next frame → visible; visible=false → timeout → unmount)
- Added two CSS keyframe animations in styles.css: `group-fade-in` (ends at opacity 1) and `group-fade-in-dim` (ends at opacity 0.7)
- Files changed: `platform/browser-extension/src/side-panel/components/PluginList.tsx`, `platform/browser-extension/src/side-panel/components/PluginCard.tsx`, `platform/browser-extension/src/side-panel/styles.css`
- **Learnings for future iterations:**
  - React hooks must not be called after an early return — moved the "no match" early return below all useEffect/useState/useRef calls
  - useEffect cleanup must return consistently — use a single `return () => clearTimeout(timer)` at the end rather than conditional returns

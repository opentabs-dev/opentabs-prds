## Codebase Patterns
- Both `handleMcp` and `handleGatewayMcp` in `http-routes.ts` share identical GET/DELETE/POST handling patterns — changes to one usually need to be mirrored in the other
- The existing test in `http-routes.test.ts` at line ~1180 tests rate-limiting behavior for GET /mcp without session — must be updated when changing the status code for that case
- The MCP SDK checks status codes semantically: 405 = silently proceed, 400 = throw error, 401 = trigger auth
- Node.js `writeHead()` buffers HTTP headers until the first body `write()` — SSE streams that start empty need `res.flushHeaders()` to send headers immediately, or `fetch()` hangs indefinitely
- The dev proxy (`dev-proxy.ts`) and the server (`server-node.ts`) both need `flushHeaders()` for SSE streams — the proxy intercepts MCP GET requests and writes its own headers
- The `/health` endpoint does NOT return 401 for wrong auth — it returns a limited 200 response. Use `/audit` or other authenticated endpoints for 401 testing
- E2E tests run through the dev proxy (hot=true) by default — HTTP behavior changes must account for both the server and the proxy layers

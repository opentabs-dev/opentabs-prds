## Codebase Patterns
- E2E fixture `startMcpServer` sets `OPENTABS_SKIP_PERMISSIONS: '1'` (not `OPENTABS_SKIP_CONFIRMATION`) — the old env var name was dead after the rename in skip-permissions.ts
- Chrome notifications are NOT directly observable from Playwright E2E tests — test badge text via `Worker.evaluate(() => chrome.action.getBadgeText({}))` on the service worker instead
- Access the background service worker from E2E via `extensionContext.serviceWorkers().find(sw => sw.url().includes('background'))` — Playwright's Worker type has `evaluate()`
- The side panel must be open BEFORE a confirmation request arrives for it to display the confirmation dialog (no replay of missed messages)
- Permission evaluation for non-existent tabs returns `domain: null`, bypassing the trusted-domain upgrade and falling through to `'ask'` — this triggers the 30s confirmation timeout if permissions aren't skipped
- Plugin tools (e2e-test_echo etc.) don't go through the permission system — only browser tools do

## Codebase Patterns
- Browser tab ownership for cross-profile routing is tracked in `state.browserTabOwnership` (Map<number, string>), populated by `browser_list_tabs` in `list-tabs.ts`
- The disconnect handler lives in `http-routes.ts` (`createHandleWsClose`), NOT in `extension-protocol.ts` as the PRD notes suggested
- `getConnectionForTab` in `state.ts` is the central routing function — it checks browserTabOwnership first, then falls back to plugin tabMappings
- Tab objects from `browser.listTabs` use `.id` (Chrome tab ID), while plugin tab info uses `.tabId`

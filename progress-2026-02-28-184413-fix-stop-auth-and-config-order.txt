- `normalizeConfigForDisplay` in `platform/cli/src/commands/config.ts` controls both text and JSON output key order — fix ordering here to fix both display modes at once
- JavaScript object spread preserves insertion order of the first occurrence of a key; to enforce canonical key ordering, reconstruct the object explicitly rather than relying on spread
- `readAuthSecret` is exported from `platform/cli/src/config.ts` — import it when any CLI command needs to hit an authenticated endpoint
- The `/health` endpoint returns `{ status, version, ... }` only when authenticated; unauthenticated it returns only `{ status: "ok" }` — any health validation requiring `version` must send the auth header
- Auth header pattern: read secret → build `headers: Record<string, string>` → conditionally set `Authorization: Bearer <secret>` → pass `headers` to `fetch`

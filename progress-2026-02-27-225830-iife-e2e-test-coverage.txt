## Codebase Patterns
- E2E tests use `setupIsolatedIifeTest()` from `e2e/helpers.ts` for standard IIFE injection test setup (MCP server + extension + client + test server + isolated config)
- `createMinimalPlugin()` from `e2e/fixtures.ts` creates discoverable but non-functional plugins (isReady returns false, no real tools). For functional plugins with real tool dispatch, create a full plugin structure with SDK.
- To build a custom test plugin through the real pipeline: create `package.json` (with opentabs field), `tsconfig.json`, `src/index.ts`, then symlink `node_modules` from `plugins/e2e-test/` for dependencies, run `tsc`, then `node platform/plugin-tools/dist/cli.js build`
- `E2E_TEST_PLUGIN_DIR` from `e2e/fixtures.ts` points to `plugins/e2e-test/` — the reference plugin for E2E tests
- The `stripNodeBuiltins` esbuild plugin at `platform/plugin-tools/src/commands/build.ts:672-681` marks `node:*` imports as `external: true, sideEffects: false`, allowing esbuild to tree-shake them
- Config setup needs: `config.json` with localPlugins + tools, `extension/auth.json` with secret — see `createTestConfigDir()` in fixtures.ts
- Set `OPENTABS_CONFIG_DIR` to isolated temp dir and `OPENTABS_SKIP_CONFIRMATION=1` when running the build pipeline in tests
- The adapter hash chain: build embeds `__adapterHash` in IIFE → server `extractEmbeddedAdapterHash()` reads it via regex → server writes IIFE to `adapters/{name}-{hash8}.js` → extension injects file + verifies `__adapterHash` from page matches expected hash
- The adapters/ directory is at `configDir/extension/adapters/` (symlinked to extension dir in tests via `setupAdapterSymlink`)
- To corrupt an adapter file for hash mismatch testing, replace the `__adapterHash="..."` value in the adapters/ directory file (NOT the plugin's dist/). The server still has the original correct hash in memory
- Avoid `!` non-null assertions in E2E tests — use optional chaining `?.` and `expect().toBeDefined()` assertions instead
- To tamper with frozen adapters in tests, rebuild `globalThis.__openTabs` entirely via `browser_execute_script` — the hashAndFreeze snippet makes `adapters` and `adapters[name]` non-writable/non-configurable, so property assignment fails silently
- After `page.reload()`, chain `page.waitForFunction()` (adapter presence) then `waitForToolResult()` (tool readiness) for reliable recovery detection
- `createMinimalPlugin()` IIFEs lack the IIFE wrapper (no onNavigate interceptor, no hashAndFreeze, no log transport). To test features that rely on the wrapper, build through the real pipeline instead.
- To remove plugin tools from config without violating `@typescript-eslint/no-dynamic-delete`, rebuild the tools object by filtering entries rather than using `delete config.tools[key]`

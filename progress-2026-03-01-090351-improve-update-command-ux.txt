## Codebase Patterns
- The unauthenticated `/health` endpoint at `platform/mcp-server/src/http-routes.ts` always returns `x-opentabs-version` header
- The authenticated `/health` response body includes `mode: "dev" | "production"` — only returned when Bearer token is valid
- The CLI uses `picocolors` (pc) for colored output; `eqeqeq` ESLint rule requires `!==` not `!=`
- Main update command file: `platform/cli/src/commands/update.ts`
- Stop command pattern (PID file + health check) is in `platform/cli/src/commands/stop.ts`
- `getPidFilePath` and `readAuthSecret` are exported from `platform/cli/src/config.ts`
- `isProcessAlive` and `waitForExit` are duplicated in update.ts (same pattern as stop.ts); both files use local copies rather than a shared module
- `waitForServer` in update.ts polls GET /health every 500ms checking x-opentabs-version header matches expected version
- `restartBackgroundServer` orchestrates: SIGTERM → waitForExit(5s) → unlink PID → spawnSync opentabs start --background --port → waitForServer(10s)
- Server detection order: unauthenticated health → PID file check → authenticated dev mode check → foreground default
- For ESM module mocking: use `vi.mock('module', async importOriginal => { const actual = await importOriginal<typeof Module>(); return { ...actual, fn: vi.fn() }; })` — `vi.spyOn` does not work on ESM namespace exports
- CLI test files use `createTestServer` helper (same as doctor.test.ts) for real HTTP servers on random ports
- E2E side panel locators: BrowserToolsCard shows "Browser" + "CORE" badge (use `text=CORE` as locator)

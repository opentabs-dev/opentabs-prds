## Codebase Patterns
- Plugin tool definitions use `defineTool({...})` from `@opentabs-dev/plugin-sdk`
- The `summary` field goes after `description` and before `icon` in the defineTool call
- Summaries should be short (under 80 chars), concise, and in imperative form
- Standalone plugins (under `plugins/`) need `npm install` in their directory before builds work in worktrees
- Plugin quality checks: `npm run check` runs build + type-check + lint + format:check
- Strings with apostrophes must use double quotes (Biome formatter rejects escaped single quotes)

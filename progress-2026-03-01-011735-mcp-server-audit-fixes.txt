## Codebase Patterns
- sanitize-error.ts uses chained .replace() calls; order matters â€” more specific patterns (bracket IPv6, bare IPv6) must precede IPv4 to avoid partial matches on IPv4-mapped IPv6 addresses.
- Vitest uses positional pattern args (`npm run test -- sanitize-error`), not `--testPathPattern`.
- lint-staged runs prettier+eslint on staged files automatically; run `npm run lint:fix` to pre-fix before committing.

## 2026-03-01 - US-001
- Added IPv6 address stripping to `sanitizeErrorMessage` in `platform/mcp-server/src/sanitize-error.ts`
- Two new regexes inserted before the IPv4 regex:
  1. Bracket-wrapped form: `/\[[\da-fA-F:]+(?:%[^\]]+)?\](?::\d+)?/g` â€” handles `[::1]:9515`, `[fe80::1%eth0]`
  2. Bare form: `/(?:[\da-fA-F]{0,4}:){2,}[\da-fA-F]{0,4}(?:%\S+)?/g` â€” handles `::1`, `fe80::1`, full 8-group addresses, zone IDs
- Added 7 new tests to `platform/mcp-server/src/sanitize-error.test.ts` covering all IPv6 forms in the acceptance criteria
- **Learnings for future iterations:**
  - The bare IPv6 regex requires at least 2 colons (`{2,}` on the colon-terminated group), which is the key distinguishing feature from other patterns
  - Bracket form must run before bare form to capture `[addr]:port` as a single replacement
  - IPv4-mapped IPv6 (`::ffff:127.0.0.1`) is partially handled â€” the IPv6 prefix is stripped, preventing `::ffff:` from leaking, though the decimal suffix `.0.0.1` may remain (not a listed requirement)
  - `[\da-fA-F]{0,4}` allows 0 hex digits in each group, correctly handling compressed `::` notation

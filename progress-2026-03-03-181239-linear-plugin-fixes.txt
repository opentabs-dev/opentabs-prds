## Codebase Patterns
- Linear plugin tools are in `plugins/linear/src/tools/`, one file per tool
- The `graphql<T>()` helper in `linear-api.ts` returns typed `result.data` as `T`, but inner fields can be null when resources don't exist
- Entity-specific queries (issue, team) should use `ToolError.notFound` for null guards; collection queries (searchIssues, issues, projects, users) should use `ToolError.internal`
- `ToolError` is imported from `@opentabs-dev/plugin-sdk`
- Mutation tools use `ToolError.internal` (not `ToolError.notFound`) since a failed mutation is a server-side issue, not a missing resource
- Use optional chaining (`data.issueCreate?.issue`) for mutation guards since both the wrapper and inner field can be null
- Plugin dependencies must be installed separately (`npm install` in the plugin directory) — they are not part of the root workspace

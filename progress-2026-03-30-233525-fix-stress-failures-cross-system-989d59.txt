## Codebase Patterns
- Permission selects use aria-labels: `Permission for <plugin-name> plugin` (for plugins) and `Permission for browser tools` (for browser tools card)
- The `selectPermission` helper in `e2e/helpers.ts` clicks a Radix Select trigger by aria-label and selects an option by display text
- Permission selects are in the AccordionPrimitive.Header, always visible regardless of accordion expand state
- Browser tools card uses `BrowserToolsCard.tsx`, plugins use `PluginCard.tsx` — both have their permission select in the header
- Toggling a plugin's permission Off fires `plugins.changed` which disrupts in-flight dispatch for that plugin
- When a plugin is removed mid-dispatch, the system can return various errors: "not found", "Invalid arguments", "unknown tool", "disabled", "plugin", etc. — error regexes should be broad
- With 0 plugins, the extension sends no `tab.syncAll` — there are no match patterns, so no tabs are evaluated. Tests starting with 0 plugins must NOT wait for `tab.syncAll`.
- When multiple MCP clients share a server, each client must be warmed up independently with `waitForToolResult` before concurrent testing — one client's warmup does not guarantee others are ready.

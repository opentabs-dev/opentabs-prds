## Codebase Patterns
- E2E tests use fixtures (mcpServer, testServer, extensionContext, mcpClient) for isolated parallel test execution
- `waitFor` from helpers.ts is the standard polling pattern — avoids fixed sleeps
- `plugin_list_tabs` returns `PluginTabsEntry[]` with per-tab readiness info
- Array destructuring with a guard (`if (!tab0 || ...)`) avoids non-null assertions that biome warns about
- Use `test.slow()` for tests that need longer timeouts (3x default)
- `waitForExtensionConnected` + `waitForLog('tab.syncAll received')` + `testServer.reset()` is the manual setup preamble (when not using setupToolTest)
- Adapter files on disk: `<configDir>/extension/adapters/{pluginName}-{sha256first8}.js` — use `fs.readdirSync` + regex to find them
- Embedded adapter hash: 64-char SHA-256 extracted from `__adapterHash="..."` in the IIFE — matches `expectedHash` from `extension_check_adapter`

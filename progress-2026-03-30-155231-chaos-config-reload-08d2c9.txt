## Codebase Patterns
- E2E stress tests go in `e2e/stress-config-reload.e2e.ts` — a single file for all config reload resilience tests
- Use `writeTestConfig` for valid config writes, raw `fs.writeFileSync` for corruption tests
- `waitForLog(server, 'Config watcher: Watching')` ensures the watcher is ready before testing
- `waitForToolList` with a predicate is the correct polling pattern for verifying tool list state changes
- `startMcpServer(configDir, true)` starts in hot mode with isolated config; second param controls hot reload
- Use `authHeaders(server.secret)` and `postReload(port, headers)` for POST /reload calls
- Avoid non-null assertions (`!`) — use optional chaining (`?.`) per Biome lint rules
- For corruption tests, write raw strings via `fs.writeFileSync` and check for `'Reload failed, keeping previous state'` in server logs

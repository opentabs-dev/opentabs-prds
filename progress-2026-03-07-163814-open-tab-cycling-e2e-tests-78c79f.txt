## Codebase Patterns
- The test file `background-message-handlers.test.ts` uses `vi.hoisted()` for all mock declarations, with destructured const at the top. New mocks must be added to BOTH the destructuring and the hoisted return object.
- Module mocks (`vi.mock(...)`) are declared after `vi.hoisted()` and before the chrome API stubs.
- Chrome API stubs are set via `(globalThis as Record<string, unknown>).chrome = {...}` before the dynamic import.
- Handlers are imported via `await import('./background-message-handlers.js')` after all mocks/stubs.
- The `lastFocusedTabId` Map is module-level state that persists across test calls. Use unique plugin names per test to avoid cross-test state leaks.
- `pickNextTab` and `lastFocusedTabId` are NOT exported — test them indirectly through `handleBgOpenPluginTab`.

## Codebase Patterns
- MCP handler patterns: `server.setRequestHandler(Schema, handler)` in `registerMcpHandlers()`. New capabilities must also be added to the `ServerModuleShape` interface and the `createMcpServer` capabilities object.
- Resource pattern mirrors prompts: `RESOURCES` array, `RESOURCE_MAP` for O(1) lookup, `getAllResources(state)` for list, `resolveResource(state, uri)` for read.
- Two tests track handler count: `mcp-setup.test.ts` ("handler count" describe) and `reload.test.ts` ("re-registers MCP handlers"). Both must be updated when adding new handlers.
- Static resource content: `CONTENT_MAP` (Map<string, string>) maps URIs to content constants. Add new entries when writing content for a resource. `resolveResource` checks status URI ŌåÆ CONTENT_MAP ŌåÆ placeholder fallback.

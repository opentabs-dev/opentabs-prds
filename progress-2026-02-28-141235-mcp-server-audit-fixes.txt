## Codebase Patterns
- `getGlobalNodeModulesPaths()` in `platform/mcp-server/src/resolver.ts` caches via `globalThis` using `__opentabs_global_paths__` key; `resetGlobalPathsCache()` is exported for test cleanup
- Test files use `vi.mock('node:child_process')` to mock `spawnSync`; `mockedSpawnSync.mockReset()` in `beforeEach` and `resetGlobalPathsCache()` both needed to isolate tests
- All test state is isolated with `beforeEach`/`afterEach`; temp dirs created via `mkdtempSync` and cleaned with `rmSync`
- `freezeRegistryMap<K, V>(map: Map<K, V>): ReadonlyMap<K, V>` is exported from `platform/mcp-server/src/state.ts`; it overrides `.set/.delete/.clear` to throw TypeError (Object.freeze alone does not block Map mutation)
- ESLint rule `import-x/exports-last` requires no non-exported code after any `export const/interface` â€” avoid placing non-exported helpers after exported ones in state.ts
- Reload serialization uses a promise chain on `globalThis.__opentabs_reload_chain__`; each caller synchronously get-then-sets the chain before any await to guarantee atomicity in JS's single-threaded model
- `FileWatcherEntry.lastSeenMtimes: Map<string, number | null>` â€” null sentinel means file didn't exist at last record; undefined means key not in map (never recorded). Poll condition: `toolsMtime !== null && lastToolsMtime !== undefined && (lastToolsMtime === null || toolsMtime > lastToolsMtime)` handles both file creation and modification
- When `dist/` doesn't exist for a pending plugin, `watch(distDir)` throws, making `entry.watchers.length === 0` â†’ `anyWatcherFailed = true` â†’ fast poll (200ms) is used automatically
- `noUncheckedIndexedAccess` is enabled â€” array[0] returns `T | undefined`; use `expect(entry).toBeDefined(); if (!entry) return;` pattern to narrow before asserting properties
- `state.activeNetworkCaptures` is `Set<number>`; add tabId before calling browser tool handlers in tests
- To intercept named fs imports in tests, use `vi.hoisted` for both the mock fn AND the real fn capture object; plain `let` variables in module scope cannot be assigned inside `vi.mock` factory (TDZ error). Pattern: `const { mockFn, capture } = vi.hoisted(() => ({ mockFn: vi.fn(), capture: { realFn: null as null | FnType } }))`, then in factory: `capture.realFn = actual.fn; return { ...actual, fn: mockFn };`
- `@typescript-eslint/no-non-null-assertion` forbids `!` even in tests â€” use `const fn = capture.realFn; if (!fn) throw new Error(...); return fn(path);` pattern

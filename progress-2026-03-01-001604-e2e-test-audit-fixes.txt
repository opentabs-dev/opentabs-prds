## Codebase Patterns
- E2E test resource cleanup pattern: declare resources as `let X: Type | undefined` before the try block, assign inside try, close/kill in finally with `if (X) await X.cleanup().catch(() => {})`. See e2e/helpers.ts:setupIsolatedIifeTest for the canonical reference.
- McpClient is imported from `./fixtures.js` and has `initialize()` and `close()` methods.
- lint-staged stash hazard: after every commit with unstaged changes, run `git diff --stat && git stash list` to verify unstaged changes survived.
- Race condition pattern for health assertions: `waitForHealth` uses `>=` predicate, so the subsequent assertion must also use `toBeGreaterThanOrEqual` — never `toBe` (exact equality) — because more log entries can arrive between the two health() calls.
- Overly broad assertions (e.g. `toContain('DEBUG')`) that match any log string should be removed if more specific assertions (e.g. `toContain('${prefix} debug-message')`) already cover the same behavior.
- `setupToolTest` returns a `Page` that callers must close — capture it as `const page = await setupToolTest(...)` and call `await page.close()` at the end of the test. Tests that discard the return value leak browser tabs.

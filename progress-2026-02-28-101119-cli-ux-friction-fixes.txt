## Codebase Patterns
- CLI commands live in `platform/cli/src/commands/`. Each command has a test file with the same name.
- Exported pure/testable functions from a command module get tested in the corresponding `.test.ts` file.
- Test files import from vitest, then `@scoped/packages`, then `node:` built-ins — in that order (after the local relative import).
- When testing functions that make HTTP calls, prefer `vi.stubGlobal('fetch', vi.fn()...)` pattern + `vi.unstubAllGlobals()` in `afterEach`.
- `vi.mocked(global.fetch).mock.calls` can have type issues; prefer `let callCount = 0` counter pattern for counting calls.
- The `no-unsafe-assignment` lint rule fires on `vi.fn()` result assigned to a variable; pass the mock directly to `vi.stubGlobal` when possible, or use `vi.fn().mockImplementation(...)` in the stubGlobal call.
- Prettier multi-line export: `export { A, B, C }` with many items needs multi-line formatting with trailing comma.
- `detectRunningServerPort` accepts `secret: string | null` (not calling `readAuthSecret` internally) to keep it testable without filesystem mocks.
- `DEFAULT_PORT` is available from `@opentabs-dev/shared` — already imported in `platform/cli/src/commands/config.ts`.
- `@typescript-eslint/no-dynamic-delete` blocks `delete obj[computedKey]` — use `Object.fromEntries(Object.entries(obj).filter(([k]) => k !== key))` to remove a dynamic key instead.

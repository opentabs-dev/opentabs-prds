## Codebase Patterns
- E2E file watcher tests in `e2e/hot-reload-dynamic.e2e.ts` require a 500ms delay after `waitForLog(server, 'File watcher: Watching', ...)` before any file writes, to guard against the macOS fs.watch()/FSEvents race condition.
- The standard comment for this delay: "On macOS, fs.watch() returns before FSEvents is ready to deliver events. The explicit delay after the log is necessary — without it, the file write can race ahead of FSEvents registration and the change is silently missed."
- `testServer.invocations()` returns `Array<{ ts, method, path, body }>` — use it with `waitFor` to poll for in-flight requests reaching the test server instead of fixed delays.
- `replaceIifeClosing` in `e2e/helpers.ts` matches the first `})();` in the IIFE file — the adapter build appends a freeze block after the main IIFE, so the regex must not anchor to EOF.

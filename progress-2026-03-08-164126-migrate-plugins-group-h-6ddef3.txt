## Codebase Patterns
- Plugin API files (`*-api.ts`) follow a consistent pattern: auth extraction via getAuthCache + localStorage fallback, waitForAuth via waitUntil, and parseRetryAfterMs for rate limit headers
- SDK auth cache stores typed objects (e.g., `SupabaseAuth`), not raw strings — use `getAuthCache<T>` and `setAuthCache<T>` with the full auth object
- `parseRetryAfterMs` handles both seconds and HTTP-date formats, unlike manual `Number() * 1000`
- Null check for `response.headers.get('Retry-After')` should use `!== null` (not truthy check) since the header value could be `"0"`

## Codebase Patterns
- platform/mcp-server/src/server-node.ts: Node.js HTTP adapter — `sendResponse()` converts Web `Response` to Node.js `ServerResponse`. SSE responses pipe a `ReadableStream` via `Readable.fromWeb()`.
- When an error occurs after headers are sent on a streaming response, use `res.destroy()` (drops TCP connection) not `res.end()` (clean close that looks like a normal end-of-stream).
- Always call `nodeStream.destroy()` in the error handler to stop the source from producing more data.
- platform/mcp-server/src/resolver.ts: `getGlobalNodeModulesPaths()` is async — uses `execFile` (callback-based) wrapped in a `Promise`. The `ExecFileException` type is a structural intersection (`Omit<ExecException, "code"> & ...`) that doesn't satisfy `prefer-promise-reject-errors` statically; use `err as Error` cast since it's always an Error at runtime.
- When mocking `node:child_process` in Vitest, mock `execFile` (callback-based). The mock's `mockImplementation` must call the `callback` argument: `(file, args, callback) => callback(null, stdout, '')` for success, `callback(err, '', '')` for failure. `vi.fn(actual.execFile)` does NOT copy `[util.promisify.custom]`, so `promisify(mockedFn)` uses standard callback wrapping (calls mock with `(file, args, callback)`).

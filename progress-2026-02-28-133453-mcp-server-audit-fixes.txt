## Codebase Patterns
- `createHandleWsClose` in `platform/mcp-server/src/http-routes.ts` handles WebSocket disconnect cleanup — the `if (state.extensionWs === ws)` block is where all matching-ws cleanup goes
- Tests in `http-routes.test.ts` use `createTestHandlers()` and `createMockWsHandle()` helpers; existing describe blocks can be extended with new `test()` calls
- `TabMapping` type: `{ state: TabState; tabId: number | null; url: string | null }` — no explicit import needed in tests since TypeScript infers from `Map<string, TabMapping>`
- Browser probe scripts in `analyze-site/index.ts` are plain string templates that run in browser context — they cannot import TS modules. Logic that needs to be unit tested should be extracted into exported TypeScript functions and applied server-side on the script result.
- `detect-framework.ts` is the pure analysis module for framework detection; `deduplicateFrameworkProbes` is exported from there and called in `index.ts` after the browser script returns raw probes.
- `state.ts` uses `import-x/exports-last` ESLint rule — ALL non-exported helper functions/variables must be placed BEFORE the first `export` statement in the file (after imports). Inserting a non-export between export statements triggers violations for all preceding exports.
- `Object.freeze` on a `Map` does NOT prevent `.set()` — Map methods access internal slots, bypassing the frozen property check. To truly prevent mutation, override `.set`/`.delete`/`.clear` via `Object.defineProperty` before freezing.

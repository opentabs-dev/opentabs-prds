## Codebase Patterns
- execSync catch blocks: when a process is killed by signal or timeout, the error's `stdout`, `stderr`, and `status` properties are `null`. Always use optional chaining (`e.stdout?.toString() ?? ''`) and `e.status ?? 1` fallback.
- E2E test files live in `e2e/` at the repo root; helpers/fixtures in `e2e/helpers.ts` and `e2e/fixtures.ts`
- MCP server does NOT log auth failures (no log emitted when `/ws-info` returns 401). `waitForLog` cannot detect auth rejection. Instead, use the side panel "Authentication Failed" text as the polling wait — the side panel only shows this state after the extension has actually attempted and been rejected.
- `waitForLog`, `waitForExtensionConnected`, `waitForExtensionDisconnected`, `waitFor`, `waitForToolResult`, `waitForToolList` are all available in `e2e/helpers.ts` for polling-based waits.
- BrowserToolsCard label is "Browser" (with a CORE badge), NOT "Browser Tools". E2E tests should use `locator('text=Browser')` for this section. The label was renamed in commit 608c01c.
- When adding outer try/finally around a resource, move recovery assertions (waitForExtensionConnected etc.) to AFTER the try/finally so cleanup runs first and recovery code only runs on success path.

## Codebase Patterns
- The `__adapterHash` property name is set by the `hashAndFreeze` snippet in `platform/plugin-tools/src/commands/build.ts` and read by `readAdapterHash` in `iife-injection.ts` and `extension.checkAdapter` in `extension-commands.ts`
- `chrome.scripting.executeScript` `func` callbacks are serialized closures that cannot reference module-scope variables — string literals must be used inside them, with comments referencing the canonical constant
- The canonical constant `ADAPTER_HASH_PROP` lives in `platform/browser-extension/src/constants.ts`
- The test mock for `../constants.js` in `extension-commands.test.ts` must include all constants used by the module under test
- `injectPluginIntoMatchingTabs` supports `skipIfHashMatches` to skip re-injection when the adapter hash matches — used only by handleSyncFull, never by handlePluginUpdate
- The message-router.test.ts mock for `./iife-injection.js` must include `injectPluginIntoMatchingTabs` and `cleanupAdaptersInMatchingTabs` (these are the actual imports used by message-router.ts)
- When testing executeScript-based injection, the mock must track per-tab state (pre-injection vs post-injection) because `verifyAdapterHash` runs after file injection and expects the new hash
- The fixture-based `mcpServer` shares `plugins/e2e-test` — use `copyE2eTestPlugin()` via `setupIsolatedIifeTest` when modifying adapter files on disk
- The file watcher watches `dist/adapter.iife.js` in the local plugin dir, NOT the copies in `configDir/extension/adapters/`
- `triggerHotReload()` = SIGUSR1 → dev proxy restarts worker → sync.full → hash-based skip if adapter hash matches

## Codebase Patterns
- Config version tests: when bumping CURRENT_CONFIG_VERSION, update all version assertions in config.test.ts, config-migrations.test.ts, AND e2e/config-migration.e2e.ts
- The E2E fixtures.ts has its own OpentabsConfig interface that must be kept in sync with the server's config.ts interface
- Config round-trip tests save with a specific version then re-load; if the saved version triggers migration, loaded.version will differ — use CURRENT version to avoid unintended migration
- Migrations operate on raw Record<string, unknown>, not parsed OpentabsConfig — type-check fields before operating on them
- When behavioral changes are made (e.g., rate limiting → coalescing), always check for stale E2E tests that assert the old behavior
- For E2E migration tests, use writeRawConfig/readRawConfig pattern (not writeTestConfig) to control exact config shape

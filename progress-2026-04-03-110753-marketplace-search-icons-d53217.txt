## Codebase Patterns
- PluginSearchResult type is defined independently in two places: `platform/mcp-server/src/plugin-management.ts` and `platform/browser-extension/src/side-panel/bridge.ts`. Both must be updated in lockstep.
- The npm search API (`/-/v1/search`) does NOT return custom package.json fields like `opentabs`. To get custom fields, fetch `/<package>/latest` for each result.
- `probeDirectPackage` fetches the full packument (all versions), so custom fields are in `data.versions[latest]`.
- Story mock factories for `PluginSearchResult` use `Partial<PluginSearchResult>` spread overrides, so new required fields need defaults in the factory.
- Three story files construct `PluginSearchResult` objects: `NpmPluginCard.stories.tsx`, `SearchResults.stories.tsx`, `SidePanel.stories.tsx`.
- PluginIcon selects SVG variant based on readiness: `tabState === 'ready'` uses active SVGs, otherwise inactive SVGs. Components with `tabState="closed"` must pass BOTH active and inactive SVG props for icons to render.
- E2E tests that depend on npm registry state (icons, artifacts) should use pre-check functions and `test.skip()` guards.

## Codebase Patterns
- E2E side panel tests use `pluginAccordionItem()` helper to locate the Accordion.Item wrapper div via XPath ancestor traversal from the trigger button
- Group state (ready vs not-connected) is determined by `opacity-70` CSS class on the Accordion.Item div
- `waitForLog` checks the entire log buffer with `includes()` — for repeated events in loops, use a count-based approach that filters logs and checks `length >= n`
- Tab state transitions are confirmed via server logs: `tab.stateChanged: <plugin> → ready` and `tab.stateChanged: <plugin> → closed`
- Use `sidePanel.on('pageerror', ...)` to collect JS errors for zero-error assertions in stress tests

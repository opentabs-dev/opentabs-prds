## Codebase Patterns
- The side panel design system uses CSS custom properties defined in `styles.css` — add new design-system-level classes (like `.focus-ring`) there, not via Tailwind utilities
- `--radius: 0` means all borders are flat/square — use `outline` (not `box-shadow` or `ring-*`) for focus indicators to match the flat aesthetic
- Accordion triggers in PluginCard and BrowserToolsCard use raw `AccordionPrimitive.Trigger` (not the retro Accordion wrapper) because they need custom header layouts
- Radix Switch renders as `<button role="switch">` — use `getByRole('switch', { name })` in E2E tests, not `input[type="checkbox"]`

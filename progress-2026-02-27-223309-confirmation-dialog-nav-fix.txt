## Codebase Patterns
- ConfirmationDialog component is in platform/browser-extension/src/side-panel/components/ConfirmationDialog.tsx
- Browser extension tests are `.test.ts` files testing pure/exported functions; no React Testing Library is used. For component logic, extract a pure function and test it.
- ESLint rule `import-x/exports-last` requires all exports to appear at the end of files. Use named export syntax at the bottom: `export { Foo, helper }` rather than `export const helper = ...` inline.
- The tsconfig.test.json in browser-extension includes `src/**/*.test.ts` (not .tsx), so test files must be `.test.ts`.
- Vitest config at root includes `platform/**/*.test.ts`.

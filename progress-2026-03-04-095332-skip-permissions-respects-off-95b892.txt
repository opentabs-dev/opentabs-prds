## Codebase Patterns
- `getToolPermission` in `platform/mcp-server/src/state.ts` is the single source of truth for tool permission resolution
- Browser tools use plugin name `'browser'` for permission lookups
- Tests that need browser tools to be auto-permitted should set `state.pluginPermissions = { browser: { permission: 'auto' } }` instead of `state.skipPermissions = true`
- `mcp-setup-dispatch.test.ts` tests browser tool dispatch paths; `mcp-setup.test.ts` tests tool listing with description prefixes
- E2E `writeTestConfig` auto-populates `plugins: { <basename>: { permission: 'auto' } }` from `localPlugins` when `plugins` is not provided
- Side panel components never receive `skipPermissions` — only App.tsx reads it from bridge state to control the banner

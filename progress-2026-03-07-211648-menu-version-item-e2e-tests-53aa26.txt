## Codebase Patterns
- Lucide-react icons render SVGs with CSS class `lucide-<kebab-name>` (e.g., `lucide-folder-open`, `lucide-package`). Use `svg.lucide-folder-open` selectors in E2E tests to verify which icon is rendered.
- E2E test setup pattern: `writeTestConfig()` → `startMcpServer()` → `startTestServer()` → `launchExtensionContext()` → `setupAdapterSymlink()` → `waitForExtensionConnected()` → `waitForLog('tab.syncAll received')` → `openSidePanel()`.
- Plugin three-dot menu: `[aria-label="Plugin options"]`. Browser tools menu: `[aria-label="Browser tools options"]`.
- Menu items are `[role="menuitem"]`. Radix DropdownMenu uses `data-disabled` attribute on disabled items.
- The e2e-test plugin is always local (`source === 'local'`), registered via `localPlugins` in test config.
- Use `getPluginPackageInfo()` to read version from `plugins/e2e-test/package.json`.

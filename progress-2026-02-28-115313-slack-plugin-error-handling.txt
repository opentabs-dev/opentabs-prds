- The SDK's `fetchFromPage` error handling pattern (platform/plugin-sdk/src/fetch.ts lines 62-79) is the reference: DOMException TimeoutError → ToolError.timeout(), signal.aborted → non-retryable ToolError('aborted'), other errors → retryable ToolError with category 'internal'
- The `slackApi` function uses raw `fetch()` (not SDK's fetchFromPage) because it needs form-encoded bodies with internal Slack headers — error handling must follow SDK conventions manually
- `ToolError` static factories: `.timeout()` = retryable, `.internal()` = non-retryable; for retryable internal errors use `new ToolError(msg, code, { category: 'internal', retryable: true })`

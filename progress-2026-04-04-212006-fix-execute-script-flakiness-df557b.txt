- Set `__asyncKey = true` before calling the async function (was set after)

- **Learnings for future iterations:**
  - The `async function` wrapper makes ALL user code async — `(async function() { return 42 })()` always returns a Promise, so the thenable check is always true
  - `.then(fn).catch(fn)` requires TWO microtask hops for rejections (rejected Promise → .then() creates intermediate rejected Promise → .catch() catches it), while `.then(fn, fn2)` handles rejections directly in ONE microtask hop
  - The `try/catch` in the wrapper only catches wrapper-level errors (not user code errors, which become Promise rejections from the async function)
  - E2E tests for browser_execute_script live in both e2e/browser-tools.e2e.ts and e2e/strict-csp.e2e.ts

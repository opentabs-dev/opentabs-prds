## Codebase Patterns
- E2E tests use fixtures from `e2e/fixtures.ts` (mcpServer, testServer, extensionContext, mcpClient) with automatic setup/teardown
- `setupToolTest` helper handles the standard preamble: wait for extension, open tab, poll until plugin is ready
- Use `for...of` with `.entries()` instead of indexed access with `!` non-null assertions to satisfy Biome lint
- `parseToolResult` parses JSON text content from tool call results
- The e2e-test plugin's echo tool returns `{ message: <input> }` matching its input
- `readTestConfig` / `writeTestConfig` are in fixtures.ts (not helpers.ts) — use them for config changes in tests
- Plugin permission `'off'` returns "has not been reviewed yet" for plugin tools (not "currently disabled" which is browser-only)
- `OPENTABS_DANGEROUSLY_SKIP_PERMISSIONS=1` does NOT override `'off'` — it only upgrades `'ask'` to `'auto'`

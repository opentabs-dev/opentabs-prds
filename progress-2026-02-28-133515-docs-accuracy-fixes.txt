## Codebase Patterns
- Docs site is a standalone Next.js project in `docs/` with its own `node_modules` (not part of root npm workspaces). Must run `npm install` inside `docs/` if node_modules are absent before running checks.
- Quality checks for docs: `cd docs && npm run build && npm run type-check && npm run lint && npm run knip && npm run format:check`
- The `lastUpdated` frontmatter field in docs MDX files uses YYYY-MM-DD format

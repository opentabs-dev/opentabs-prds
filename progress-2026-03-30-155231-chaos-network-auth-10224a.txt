## Codebase Patterns
- E2E tests that need custom server lifecycle (kill/restart) use `base` test from `@playwright/test` instead of fixtures, managing their own resources in try/finally
- `startMcpServer(configDir, false)` (hot=false) provides clean kill semantics; hot=true uses dev proxy which complicates kill/restart
- `startMcpServer(configDir, false, savedPort)` reuses a specific port for restart scenarios
- `launchExtensionContext` returns `{ context, cleanupDir, extensionDir, mcpPort }`; use `setupAdapterSymlink(configDir, extensionDir)` to connect adapters
- `createMcpClient` creates a standalone client; after server restart, create a NEW client since old sessions are invalid
- `waitForToolResult` polls a tool call until it succeeds — preferred over fixed sleep for verifying readiness
- `Promise.allSettled` is the correct pattern for in-flight calls during server kill — some will fail, some may succeed
- Auth secret path: `path.join(configDir, 'extension', 'auth.json')` with format `{ secret: "<uuid>" }`; server reads it at startup
- Secret rotation tests don't need extension/test server — only MCP client HTTP auth is tested, keeping them lightweight
- Two-extension tests: symlink ext2's adapters dir to ext1's adapters dir (after `setupAdapterSymlink` on ext1) so both read from the same physical IIFE location

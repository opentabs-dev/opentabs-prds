## Codebase Patterns
- bridge.ts uses `sendRequest(method, params) as Promise<T>` for all JSON-RPC bridge calls — cast return type, no async/await wrapper needed
- bridge.ts exports both types and functions; types use `export type { ... }`, functions use `export { ... }`
- New bridge types follow the interface pattern (not type aliases) for consistency
- `matchesTool` checks displayName, name, description; `matchesPlugin` delegates to `matchesTool` for tools plus checks plugin displayName and name
- Retro card container pattern: `border-border bg-card rounded border-2 p-3 shadow-md transition-all hover:shadow-sm` (note: Tailwind prettier plugin sorts classes alphabetically)
- ESLint `import-x/order` groups: sibling (./) → parent (../) → external → builtin → type (import type, always last)
- Stories follow: `Meta` with `title`, `decorators` wrapping in `w-80` div, named `StoryObj` exports — follow PluginCard.stories.tsx as the canonical pattern
- ConfigStatePlugin.name is a short name like 'slack', NOT the full npm package name — npm search results use full names like '@opentabs-dev/opentabs-plugin-slack'; use `extractShortName` to normalize when deduplicating
- PluginList filters by matchesTool (tool-only), not matchesPlugin (name+tools); SearchResults pre-filters with matchesPlugin and passes empty toolFilter to PluginList to show all matched plugins correctly
- When a component includes a prop in its interface but doesn't use it in the implementation, TypeScript/knip don't complain — fine for API consistency
- TrustTier is 'official' | 'community' | 'local' (NOT 'npm') — use 'community' for npm-sourced plugins in test/story mocks
- PluginState.source is 'npm' | 'local'; PluginState.update is `{ latestVersion: string; updateCommand: string } | undefined`
- Confirmation UX pattern: useState(false) for confirmPending + useRef for 3s timer — clear timer on unmount with useEffect cleanup
- Retro Menu component has a `Menu.Separator` sub-component (added in US-008) — `React.forwardRef` wrapping `DropdownMenuPrimitive.Separator` with class `h-px bg-border -mx-1 my-1`
- `Accordion.Item` accepts `className` merged via `cn()` — apply conditional classes directly (e.g., `pointer-events-none opacity-60`) without extra wrapper divs
- ESLint `react-hooks/set-state-in-effect` forbids calling setState synchronously in useEffect body — use event handler callbacks instead of effects for debounced search patterns
- `import-x/order` sorts `./retro/*` before `./SearchResults` — deeper paths come before shallower siblings alphabetically within the same group

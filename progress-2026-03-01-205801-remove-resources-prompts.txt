## Codebase Patterns
- plugin-sdk (platform/plugin-sdk/src/index.ts) is the source of truth for plugin authoring types — downstream consumers include plugin-tools/build.ts, browser-extension/resource-prompt-dispatch.ts, mcp-server, and plugins
- When removing exported types from plugin-sdk, ALL downstream consumers must be updated in the same change to keep the build green (tsc --build builds all packages in dependency order)
- plugins/e2e-test is used as a real test fixture by platform/plugin-tools/src/commands/build-e2e.test.ts — it symlinks e2e-test's node_modules into temp copies and runs opentabs-plugin build on them, so the pre-built dist/index.js matters
- After changing plugin-sdk exports, rebuild plugins/e2e-test: `cd plugins/e2e-test && npm run build` to update the pre-built dist/index.js used by build-e2e tests
- platform/create-plugin/src/index.test.ts checks the structure of the generated tools.json manifest — update it when manifest fields are removed
- US-004 (inspect), US-005 (state/registry/loader), US-006 (mcp-setup/file-watcher/reload) work was fully completed as part of US-002 to keep the build green — those stories should pass trivially with no changes needed

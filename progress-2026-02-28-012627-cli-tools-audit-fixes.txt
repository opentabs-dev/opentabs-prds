## Codebase Patterns
- Color parsing in validate-icon.ts uses a `parseHex` function (not exported) that feeds into `parseColor`, `isAchromaticColor`, and `convertColorToGray`. All public APIs flow through these internal helpers.
- Tests in validate-icon.test.ts exercise the color parsing pipeline indirectly via `validateInactiveIconColors` and `generateInactiveIcon` (the exported functions).
- The alpha channel in hex colors (#RGBA, #RRGGBBAA) is ignored for both achromatic detection and grayscale conversion — only RGB channels matter.
- SVG color validation and conversion operates on three layers: (1) color-carrying attributes (fill, stroke, etc.), (2) inline style="" attributes, and (3) `<style>` blocks. Each layer uses parallel regex-based extraction with the same color parsing pipeline.
- `formatUptime` in platform/cli/src/commands/status.ts is exported for testing. The hours+ range formats exclude seconds (e.g., "1h 1m", not "1h 1m 1s").

## Codebase Patterns
- To test timer cleanup with `vi.spyOn(globalThis, 'clearTimeout')`, spy before handler call, assert `toHaveBeenCalled()`, then `mockRestore()` after — works cleanly with `vi.useFakeTimers()`
- To use mocked module functions (e.g. `getPluginMeta`, `findAllMatchingTabs`) in tests: add `const { fn } = await import('../module.js')` after `vi.mock(...)`, then call `vi.mocked(fn).mockResolvedValueOnce(...)` in tests
- `import-x/exports-last` rule: non-exported helpers must be placed BEFORE all `export const` declarations in a module, not between or after them
- `handleBrowserExecuteScript` validation helpers (`requireTabId`, `requireStringParam`) do NOT throw — they call `sendToServer` and return `null`. Safe to move outside `try` blocks.
- Test fake timers: use `vi.useFakeTimers()` + `vi.advanceTimersByTimeAsync(n)` for testing timeout behavior; always `vi.useRealTimers()` to restore after the test
- `expect.stringContaining()` triggers `@typescript-eslint/no-unsafe-assignment`; use exact strings or `expect.objectContaining()` cast instead
- `toHaveBeenNthCalledWith(n, ...)` is the idiomatic way to assert specific mock call arguments without accessing `mock.calls` directly (avoids optional chain / unsafe-assignment lint errors)
- The SCRIPT_TIMEOUT_MS mock value is 30000ms in extension-commands.test.ts
- To test the inner `func` passed to `chrome.scripting.executeScript`, use `mockExecuteScript.mockImplementation(async (opts: unknown) => { const scriptOpts = opts as { func: (...a: unknown[]) => Promise<unknown>; args: unknown[] }; const result = await scriptOpts.func(...scriptOpts.args); return [{ result }]; })` — use fake timers and `vi.advanceTimersByTimeAsync` inside the implementation to fire any internal `setInterval` before awaiting the result. Set up `(globalThis as Record<string, unknown>).document` and `.getComputedStyle` as mock globals before calling the handler.

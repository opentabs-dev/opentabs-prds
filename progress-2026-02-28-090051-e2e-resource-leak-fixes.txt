## Codebase Patterns
- E2E test import order: regular imports (relative, then node:), then type-only imports (relative from ./fixtures.js, then external like @playwright/test)
- For manual-setup E2E tests that create resources sequentially, use `let resource: Type | undefined` before the try block, assign inside try, and use optional chaining (`resource?.close()`) or conditionals (`if (cleanupDir) fs.rmSync(...)`) in the finally block
- Capture `let` variable values as `const` before async callbacks/closures to avoid TypeScript narrowing issues (e.g., `const serverPort = server.port; const serverSecret = server.secret;` before `expect.poll` callbacks that use those values)
- The helpers.ts `setupIsolatedIifeTest` pattern (lines 436-485) is the canonical reference for incremental resource tracking in E2E setup

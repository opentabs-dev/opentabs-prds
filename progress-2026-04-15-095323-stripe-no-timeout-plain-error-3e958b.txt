## Codebase Patterns
- Stripe plugin uses raw fetch() with Bearer token auth (not fetchFromPage) because it needs a custom Authorization header from PRELOADED.session_api_key
- Plugin quality checks: `npm run build && npm run type-check && npm run lint && npm run format:check` from the plugin directory
- Plugin node_modules may need `npm install` in the plugin directory if @opentabs-dev packages are missing (worktree setup may not always install plugin deps)
- ToolError is exported from `@opentabs-dev/plugin-sdk` — always use ToolError.internal() instead of plain `new Error()` for recoverable tool errors

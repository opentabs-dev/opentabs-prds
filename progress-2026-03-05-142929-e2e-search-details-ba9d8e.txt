## Codebase Patterns
- E2E side panel tests use `openSidePanel(context)` from `helpers.ts` to open the side panel as a regular extension page
- `waitForExtensionConnected(server)` must be called before opening the side panel
- `setupAdapterSymlink(configDir, extensionDir)` links the server's adapter dir to the extension's
- The `SearchResults` component only renders when `searchQuery` is truthy in `App.tsx`; default view uses `PluginList`
- The 'Installed' section header only appears in search results mode when installed plugins match the filter
- Use `sidePanel.getByPlaceholder('Search plugins and tools...')` to locate the search input
- Cleanup pattern: try/finally with `context.close()`, `server.kill()`, `fs.rmSync(cleanupDir)`, `cleanupTestConfigDir(configDir)`

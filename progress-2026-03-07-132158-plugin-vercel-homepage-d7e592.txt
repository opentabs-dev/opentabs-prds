## Codebase Patterns
- Plugin `homepage` property does NOT use `override` modifier — the published npm SDK (0.0.64) doesn't have `homepage` in the base class yet. The worktree's root node_modules has a newer SDK with `homepage`, but the plugin resolves from its own node_modules after `npm install`.
- Plugin standalone deps need `npm install` in `plugins/<name>/` before running quality checks — the worktree setup only installs the root workspace, not individual plugin node_modules.
- Pre-existing format issues exist in the vercel plugin source files (formatting was already wrong before this PR); auto-fixed via `npm run format`.

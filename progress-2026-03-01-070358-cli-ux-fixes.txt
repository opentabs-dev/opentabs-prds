## Codebase Patterns
- `resolvePort` in parse-port.ts caches the port for the process lifetime — after writing a new port to config, `resolvePort({})` will read from the old cached value if already called, or the new written value if called fresh. To target the currently-running server after a config write, always pass the old port explicitly.
- `notifyServer` in notify-server.ts does a health check + POST /reload. It accepts an optional `successMessage` for caller-controlled output.
- Config tests mock `../notify-server.js` with `vi.fn().mockResolvedValue(undefined)`.
- `handleSetPort` is not exported from config.ts, so it has no direct unit test coverage.
- In `checkServerHealth` (doctor.ts): the "not reachable" case uses `portSuffix` to tell the user the port to start on; the "another service" case must NOT suggest starting on the configured port via `--port` since that port is already occupied.

## Codebase Patterns
- Adapter IIFE files are written via `writeAdapterFile()` in `adapter-files.ts` — it uses content-hashed filenames, making writes idempotent
- `sendSyncFull` handles both file writes and WS notification to the extension; after the refactor, file writes are in `writeAllAdapterFiles` and `sendSyncFull` delegates to it
- `reloadCore` in `reload.ts` is the central reload pipeline: config → discover → state swap → prune → resetStaleReviewedVersions → writeAllAdapterFiles → secret reload → extension sync
- The `isExtensionConnected` guard in `reloadCore` only gates the WS notification (sendSyncFull), not the file writes
- `ensureAdaptersDir`, `cleanupStaleAdapterFiles`, `writeAdapterFile`, `timeoutRace`, `ADAPTER_WRITE_TIMEOUT_MS` are all in `adapter-files.ts`

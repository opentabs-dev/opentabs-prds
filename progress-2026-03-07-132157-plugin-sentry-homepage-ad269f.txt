## Codebase Patterns
- Plugin properties that override base class members require `override readonly` (e.g., `displayName`). Properties added as new (not in published SDK base class) use plain `readonly`.
- Plugins are standalone; run `npm install` first if `node_modules` is absent, then `npm run build && npm run type-check && npm run lint && npm run format:check`.
- The published `@opentabs-dev/plugin-sdk` version may differ from the workspace version. Type errors can flip between `override` required/forbidden depending on which SDK is resolved.

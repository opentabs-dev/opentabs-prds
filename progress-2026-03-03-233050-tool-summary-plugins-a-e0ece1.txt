## Codebase Patterns
- Plugin tools are defined using `defineTool({...})` from `@opentabs-dev/plugin-sdk` in individual files under `plugins/<name>/src/tools/`
- The `summary` field goes after `description` in the tool definition object
- Summaries should be concise (under 80 chars), human-friendly, and avoid platform-specific names (e.g., "a channel" not "a Slack channel")
- Standalone plugins may need `npm install` in the worktree before `opentabs-plugin build` can resolve the SDK version
- `channel-schema.ts` in the Slack plugin is a shared schema file, not a tool — there are 22 tool files + 1 schema file = 23 total .ts files in tools/
- `schemas.ts` in the Discord plugin is the shared schema file — 19 tool files + 1 schema = 20 total .ts files in tools/
- `schemas.ts` in the GitHub plugin is the shared schema file — 21 tool files + 1 schema = 22 total .ts files in tools/

## Codebase Patterns
- Plugin directories are standalone npm packages under plugins/ (not in root workspace)
- Each plugin has its own node_modules/, package.json, and package-lock.json
- scripts/plugins.ts handles batch operations across all plugins (--build, --check flags)
- Biome is pinned as an exact version (no ^ prefix) in plugin devDependencies
- SDK/tools deps use ^semver ranges - update installed versions with npm update
- craigslist and todoist directories do NOT exist despite being listed in some docs
